# 그로버 탐색 (Grover Search)

## 한 줄 요약

그로버(Grover) 알고리즘은 정렬 안 된 N개 항목에서 조건을 만족하는 원소를 찾는데, 고전이 평균 N/2번 봐야 하는 것을 **√N 번**으로 줄인다(제곱 가속, quadratic speedup). 핵심은 **진폭 증폭**(amplitude amplification) - 오라클로 정답에 위상을 표시하고, 확산 연산자(diffusion)로 평균 중심 반사를 반복해 정답 진폭을 키운다. 기하학적으로는 상태 벡터를 정답 방향으로 조금씩 회전시키는 것이며, √N 이 **최적**임이 증명돼 있다.

## 왜 필요한가

- 비정렬 탐색은 데이터베이스·제약 만족·NP 문제 무차별 대입 등 어디에나 있음
- 제곱 가속은 지수(Shor)만큼 극적이진 않지만 훨씬 넓게 적용됨
- 진폭 증폭은 여러 양자 알고리즘의 부품(subroutine)
- 대칭키 암호 키 크기를 실질 절반으로 위협(예: AES-128 → 64비트 안전) → security/[[crypto-basics]]

## 문제 정의

- N = 2ⁿ 개 항목, 그중 M개가 "정답"(f(x)=1)
- 오라클 f 만 주어짐 - 내부 구조 없음(unstructured)
- 목표: 정답 하나 찾기

| 방법 | 질의 횟수 |
|---|---|
| 고전(최악) | N |
| 고전(평균) | N/2 |
| 그로버 | ≈ (π/4)√(N/M) |

## 두 연산자

**오라클 (위상 반전)**: 정답의 진폭 부호만 뒤집음

```
Uf |x⟩ = (−1)^f(x) |x⟩     (위상 킥백, → deutsch-jozsa)
```

**확산 연산자 (평균 중심 반사)**: 모든 진폭을 평균에 대해 반사

```
D = 2|s⟩⟨s| − I ,  |s⟩ = 균등 중첩
```

- 정답은 아래로 뒤집혔다가 평균 반사로 위로 크게 튐 → 진폭 증가
- 오답들은 조금씩 줄어듦

## 진폭 증폭 (그로버 반복)

한 번의 반복 G = D · Uf:

```mermaid
graph LR
    S["균등 중첩 |s⟩"] --> O["Uf: 정답 위상 반전"] --> D["확산: 평균 반사"] --> R["정답 진폭 ↑"]
    R -.반복 √N.-> O
```

- 반복 1회마다 정답 진폭이 일정량 상승
- 약 (π/4)√N 회 반복 후 측정하면 높은 확률로 정답
- 초기화(H⊗n) → G 반복 → 측정 순

## 기하학적 해석

상태를 2차원 평면에서 봄: 정답 축 `|w⟩`, 오답 축 `|w⊥⟩`.

- 초기 상태 `|s⟩`는 `|w⊥⟩`에서 작은 각 θ 만큼 기울어짐 (sin θ = √(M/N))
- Uf = `|w⊥⟩` 축 대칭(반사)
- D = `|s⟩` 축 대칭(반사)
- **두 반사 = 회전** → 매 반복마다 `|w⟩` 쪽으로 각 2θ 회전
- (π/2)/(2θ) ≈ (π/4)√(N/M) 회면 `|w⟩`에 도달

## 과회전 주의

- 반복을 너무 많이 하면 정답을 **지나쳐** 진폭이 다시 감소
- 최적 횟수 근처에서 멈춰야 함 - 무작정 반복은 역효과
- M(정답 수)을 모르면 횟수 추정이 어려움 → 양자 카운팅(quantum counting)으로 M 추정

## 최적성

- Grover는 **점근적으로 최적**: 어떤 양자 알고리즘도 Ω(√N) 질의 필요(BBBV 하한)
- 즉 비정렬 탐색에서 제곱 가속 이상은 불가능
- 지수 가속이 아닌 이유: 구조 없는 오라클에는 착취할 대칭이 없음
- 대조: Shor는 문제에 주기 구조가 있어 지수 가속 → [[shor-algorithm]]

## 응용과 함의

| 분야 | 효과 |
|---|---|
| 대칭키 암호 | 키 탐색 2ⁿ → 2^(n/2), 키 길이 2배 권장 |
| NP 무차별 대입 | 2ⁿ → 2^(n/2) (여전히 지수라 다항 아님) |
| 최솟값 찾기 | 그로버 기반 √N |
| 제약 만족 | 진폭 증폭으로 가속 |

- NP-완전을 다항으로 풀진 못함 - 제곱 가속일 뿐 → complexity-theory/[[beyond-np]]

## 셀프 체크

> [!question]- 그로버의 두 연산자(오라클, 확산)는 각각 무엇을 하는가?
> 오라클 Uf는 정답 기저의 진폭 부호만 뒤집는다(위상 반전, (−1)^f(x)). 확산 연산자 D = 2`|s⟩``⟨s|` − I는 모든 진폭을 평균값에 대해 반사한다. 정답은 아래로 뒤집혔다가 평균 반사로 크게 튀어올라 진폭이 증가하고, 오답들은 조금씩 줄어든다. 이 반복이 진폭 증폭이다.

> [!question]- 왜 반복 횟수가 약 (π/4)√N인가? 기하학적으로 설명하라.
> 상태를 정답 축 `|w⟩`와 오답 축 `|w⊥⟩`의 2차원 평면에서 보면, 오라클과 확산은 각각 축 대칭(반사)이고 두 반사의 합성은 회전이다. 매 반복마다 `|w⟩` 쪽으로 각 2θ(sin θ=√(M/N))씩 회전하므로, (π/2)/(2θ) ≈ (π/4)√(N/M)회면 `|w⟩`에 도달한다.

> [!question]- 그로버에서 반복을 너무 많이 하면 왜 안 되는가?
> 회전이라서 최적 횟수를 지나치면 정답을 넘어 계속 회전하고, 정답 진폭이 다시 감소한다(과회전). 따라서 최적 근처에서 멈춰야 하며, 정답 개수 M을 모르면 횟수 추정이 어려워 양자 카운팅으로 M을 추정해야 한다.

> [!question]- 그로버가 제곱 가속(√N)에 그치고 지수 가속이 아닌 이유는?
> 오라클이 구조 없는(unstructured) 탐색이라 착취할 대칭이 없기 때문이다. BBBV 하한이 어떤 양자 알고리즘도 Ω(√N) 질의가 필요함을 증명해, √N이 최적이다. 반면 Shor는 문제에 주기 구조가 있어 지수 가속이 가능하다.

## 연습문제

> [!example]- 문제: N=4, 정답 1개(M=1)일 때 sin θ와 최적 반복 횟수를 계산하고, 왜 그로버가 단 1회로 정답을 확실히 찾는지 보여라.
> **풀이**
> sin θ = √(M/N) = √(1/4) = 1/2 → θ = π/6 = 30°.
> 초기 상태 `|s⟩`는 `|w⊥⟩`에서 θ=30° 기울어짐.
> 1회 반복 = 2θ = 60° 회전 → `|w⊥⟩`에서 30°+60° = 90° = `|w⟩` 방향에 정확히 도달.
> 따라서 반복 1회 후 측정하면 확률 1로 정답. (π/4)√4 ≈ 1.57도 1회에 부합하며, N=4는 정답 확률 100%인 특수 경우다.

> [!example]- 문제: N=2ⁿ개 항목에서 그로버의 질의 횟수를 고전 평균과 비교하고, AES-128 대칭키에 미치는 영향을 설명하라.
> **풀이**
> 고전 평균 N/2 = 2ⁿ⁻¹, 그로버 ≈ (π/4)√N = (π/4)·2^(n/2).
> 즉 2ⁿ → 2^(n/2)로 지수가 절반이 된다.
> AES-128은 키 공간 2¹²⁸ → 그로버로 유효 2⁶⁴ 탐색.
> 2⁶⁴는 여전히 실용적으로 방어 가능한 수준이지만, 보수적으로는 키 길이를 2배(AES-256 → 유효 2¹²⁸)로 늘리길 권장한다. 지수 가속이 아니라 제곱이므로 키 2배로 견딘다.
