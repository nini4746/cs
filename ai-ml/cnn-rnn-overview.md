# CNN/RNN 개요 (CNN/RNN Overview)

## 한 줄 요약

트랜스포머 이전의 두 주력 구조. CNN은 **지역성+가중치 공유**로 이미지의 공간 패턴을, RNN은 **순환 상태**로 순서 있는 데이터(텍스트·시계열)를 다룬다. 각자가 데이터 구조에 대한 **귀납적 편향**을 내장한 것이 핵심이고, 둘의 한계가 트랜스포머를 낳았다.

## 왜 필요한가

- MLP만으로 왜 부족한가 (이미지·순서)
- CNN/RNN의 핵심 아이디어 (역사적 맥락)
- 왜 트랜스포머로 넘어갔나

## 왜 특화 구조인가: 귀납적 편향

MLP([[neural-networks]])는 만능이지만 비효율 → 데이터 구조에 맞는 **편향을 설계로 넣음**:

```
이미지: 인접 픽셀이 관련 (공간 지역성) → CNN
순서:   앞이 뒤에 영향 (시간 의존) → RNN
```

- **귀납적 편향**(inductive bias): 모델 구조에 담은 가정 → 적은 데이터로 잘 학습
- 좋은 편향 = 문제에 맞는 구조

## CNN (합성곱 신경망)

이미지의 표준 (2010년대):

### 핵심 아이디어

```
작은 필터(커널)를 이미지 위로 슬라이딩 → 지역 패턴 감지
얕은 층: 엣지·색 → 깊은 층: 형태·물체 (계층적 특성)
```

두 원리:
- **지역 수용 영역**: 각 뉴런이 작은 영역만 봄 (전체 아니라) → 공간 지역성 활용
- **가중치 공유**: 같은 필터를 전체에 재사용 → 파라미터 급감 + **위치 불변**(고양이가 어디 있든 감지)

- **풀링**: 다운샘플링 (공간 축소, 불변성↑)
- MLP로 이미지 하면 파라미터 폭발(픽셀마다 가중치) → CNN이 공유로 해결

### 왜 중요

- 이미지·비전의 혁명 (ImageNet 2012 AlexNet)
- 지역 패턴이 있는 모든 것 (음성, 심지어 텍스트도)

## RNN (순환 신경망)

순서 데이터의 표준 (트랜스포머 이전):

### 핵심 아이디어

```
숨은 상태(hidden state)를 시간마다 이어받음:
  hₜ = f(hₜ₋₁, xₜ)   ← 이전 상태 + 현재 입력
"기억"을 상태에 담아 순서 처리
```

- **순환**: 같은 셀을 시퀀스 길이만큼 반복 (가중치 공유 - 시간 축)
- 텍스트·시계열·음성 (순서가 의미)

### 한계 (→ 트랜스포머 동기)

- **기울기 소실**([[training-dynamics]]): 긴 시퀀스에서 앞 정보 잊음 → LSTM/GRU(게이트로 완화)로도 한계
- **순차 처리**: hₜ가 hₜ₋₁ 필요 → **병렬화 불가** → 학습 느림 (GPU 못 살림)
- **장거리 의존**: 멀리 떨어진 단어 관계 포착 약함

## 왜 트랜스포머로 넘어갔나

RNN의 두 한계를 attention이 해결 ([[attention-and-transformers]]):

```
RNN: 순차적(느림) + 장거리 약함
트랜스포머: 병렬 처리(빠름) + attention으로 모든 위치 직접 연결
```

- **"Attention is All You Need"**(2017): 순환 없이 attention만으로
- 병렬화 → 대규모 학습 가능 → LLM 시대 ([[llm-training]])
- CNN도 비전 트랜스포머(ViT)에 상당 부분 대체 (하지만 CNN 편향은 여전히 유효)

## 역사적 위치

```
MLP → CNN(이미지)·RNN(순서) → 트랜스포머(통합, 병렬)
      각자 귀납적 편향 설계    편향 줄이고 데이터·규모로
```

- 트랜스포머는 편향을 줄이고 **데이터·계산으로** 밀어붙임 (스케일링 [[llm-training]])
- 하지만 CNN/RNN의 아이디어(지역성, 상태, 가중치 공유)는 여전히 곳곳에

## 셀프 체크

> [!question]- 귀납적 편향(inductive bias)이란 무엇이고 왜 유용한가?
> 모델 구조에 심어 둔 데이터에 대한 가정이다. CNN은 "인접 픽셀이 관련 있다(공간 지역성)", RNN은 "앞이 뒤에 영향(시간 의존)"을 가정한다. 문제에 맞는 편향은 적은 데이터로도 잘 학습하게 한다.

> [!question]- CNN의 두 핵심 원리는?
> 지역 수용 영역(각 뉴런이 작은 영역만 봄 → 공간 지역성 활용)과 가중치 공유(같은 필터를 전체에 재사용 → 파라미터 급감 + 위치 불변)다. 여기에 풀링으로 다운샘플링한다.

> [!question]- RNN이 트랜스포머로 대체된 두 한계는?
> 순차 처리(hₜ가 hₜ₋₁을 필요로 해 병렬화 불가 → 학습 느림)와 장거리 의존 약함(기울기 소실로 먼 정보 잊음)이다. attention이 병렬 처리와 1홉 연결로 둘 다 해결했다.

> [!question]- MLP로 이미지를 처리하면 왜 비효율적인가?
> 픽셀마다 독립 가중치를 두면 파라미터가 폭발하고, 위치가 조금만 바뀌어도 새로 배워야 한다. CNN은 필터 가중치를 전 이미지에 공유해 파라미터를 줄이고 위치 불변성을 얻는다.

## 연습문제

> [!example]- 문제: 100×100 흑백 이미지를 은닉 뉴런 1000개 완전연결(MLP)로 받을 때 1층 가중치 수와, 3×3 필터 64개 CNN의 가중치 수를 비교하라.
> **풀이**
> MLP: 입력 10,000(=100×100) × 은닉 1,000 = 1,000만 개 가중치.
> CNN: 필터 하나 3×3=9개, 64개면 9×64=576개(+편향 64). 채널 공유로 이미지 크기와 무관.
> 약 1,000만 대 576 - 가중치 공유가 파라미터를 수만 배 줄인다. 이게 CNN이 이미지에서 이기는 핵심 이유다.

> [!example]- 문제: 길이 T=50 문장을 RNN으로 처리할 때 왜 GPU 병렬화 이점을 못 살리나? 트랜스포머와 대비해 설명하라.
> **풀이**
> RNN은 hₜ = f(hₜ₋₁, xₜ)라 시점 t 계산이 t−1 결과를 기다린다. 50 스텝을 순차로 밟아야 해 병렬 불가, 시퀀스가 길수록 시간이 선형으로 늘고 GPU가 놀게 된다.
> 트랜스포머는 모든 토큰의 attention을 한 번의 행렬 연산으로 동시 계산하므로 50개를 병렬 처리해 GPU를 채운다. 이 병렬성이 대규모 학습을 가능케 했다.
