# 프롬프팅 (Prompting)

## 한 줄 요약

LLM은 프롬프트(문맥)만으로 학습 없이 새 작업을 수행한다(in-context learning). 명확한 지시·예시(few-shot)·추론 유도(chain-of-thought)로 성능이 크게 달라진다. 핵심은 모델이 다음 토큰을 예측한다는 사실에서 출발해 **원하는 출력이 자연스럽게 이어지도록 문맥을 설계**하는 것.

## 왜 필요한가

- 파인튜닝 없이 어떻게 새 작업을 시키나
- few-shot, chain-of-thought가 왜 되나
- 좋은 프롬프트 원리

## 왜 프롬프트가 통하나 (in-context learning)

```
LLM = 다음 토큰 예측기 ([[llm-training]])
프롬프트 = 예측의 문맥 → 문맥을 바꾸면 출력이 바뀜
```

- **가중치 변경 없이**(학습 X) 문맥만으로 작업 수행 → **in-context learning**
- 사전학습에서 본 패턴을 프롬프트가 "불러냄"
- 그래서 프롬프트 설계 = 원하는 연속이 나올 문맥 만들기

## Zero-shot vs Few-shot

### Zero-shot

```
"이 리뷰 감정 분류해: '최악이었다' →"
예시 없이 지시만
```

### Few-shot

```
"'좋아요' → 긍정
 '별로예요' → 부정
 '최악이었다' →"
예시 몇 개를 문맥에 → 패턴 파악
```

- **few-shot**: 예시로 형식·기준을 보여줌 → 더 정확·일관
- 예시가 곧 "즉석 학습 데이터" (가중치는 안 변함)
- 형식이 까다롭거나 모호한 작업에 특히 효과

## Chain-of-Thought (CoT)

**추론 과정을 소리 내어** 유도 → 복잡한 문제 성능↑:

```
그냥: "23×17은?" → (바로 답, 자주 틀림)
CoT:  "단계별로 생각해봐: 23×17 = 23×10 + 23×7 = 230 + 161 = 391"
```

- **"단계별로 생각해봐"** 한 줄이 추론 문제 정확도 크게 올림
- 왜 되나: 답을 바로 예측 대신 **중간 단계를 토큰으로 펼침** → 각 단계가 다음 단계의 문맥 (계산을 토큰 공간에 분산)
- [[tokenization]]에서 본 산술 약점도 CoT로 완화 (자리별 전개)
- 변형: **self-consistency**(여러 번 추론 후 다수결), **CoT 예시 few-shot**

## 좋은 프롬프트 원리

- **명확·구체**: 모호하면 모델이 추측 → 원하는 것·형식·제약 명시
- **역할 부여**: "너는 전문 편집자다" - 관련 문맥 활성화
- **출력 형식 지정**: "JSON으로", "3줄 이내" - 파싱·일관성
- **예시 제공**: few-shot으로 기준 전달
- **분해**: 큰 작업을 단계로 (복잡하면 나눠서)
- **문맥 배치**: 중요한 지시를 앞·뒤에 (중간은 놓치기 쉬움 - "lost in the middle")

## 문맥 창 (context window)

프롬프트가 들어갈 **토큰 예산**([[tokenization]]):

- 모델마다 한도 (수천~수백만 토큰)
- 길수록 비용·지연↑, 너무 길면 중간 정보 놓침
- RAG는 관련 부분만 넣어 예산 아낌 ([[rag]])
- 대화가 길어지면 오래된 문맥이 밀려남

## 한계·주의

- **환각(hallucination)**: 모르는 것도 그럴듯하게 지어냄 → 사실은 검증·RAG 필요 ([[rag]])
- **프롬프트 민감성**: 사소한 표현 차이로 결과 요동 (견고하지 않음)
- **프롬프트 인젝션**: 입력에 숨은 악의적 지시 (security 문제) → [[agents]]에서 심각
- **최신 지식 부재**: 학습 시점 이후 모름 → 도구·검색 필요 ([[agents]], [[rag]])
- 프롬프팅으로 안 되면 → RAG(지식)·파인튜닝(행동) 고려 ([[fine-tuning]])

## 언제 프롬프팅 vs 다른 방법

```
프롬프팅: 빠르고 싸다, 먼저 시도 (대부분 여기서 해결)
RAG:      외부·최신 지식 필요 → [[rag]]
파인튜닝:  일관된 형식·스타일·대량 반복 작업 → [[fine-tuning]]
```

## 연결

- 다음 토큰 예측이 근거 → [[llm-training]]
- 토큰 예산·문맥 창 → [[tokenization]]
- 환각·지식 보강 → [[rag]]
- 도구·인젝션 → [[agents]]
- 프롬프팅 vs 파인튜닝 선택 → [[fine-tuning]]
- 출력 평가 → [[llm-evaluation]]
