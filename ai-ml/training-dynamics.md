# 학습 동역학 (Training Dynamics)

## 한 줄 요약

신경망을 실제로 학습시킬 때 부딪히는 현실 - 학습률을 잘못 잡으면 발산하거나 안 움직이고, 기울기가 소실/폭발하고, 초기화·정규화가 수렴을 좌우한다. 옵티마이저(SGD→Adam)와 여러 안정화 기법이 이 문제들을 다룬다. "왜 학습이 안 되나"의 체크리스트.

## 왜 필요한가

- 학습률을 어떻게 정하나 (제일 중요한 하이퍼파라미터)
- 왜 기울기가 소실/폭발하나
- 학습이 안 될 때 뭘 확인하나

## 학습률 (제일 중요)

경사하강 걸음 크기([[linear-models]]) - **하나만 튜닝한다면 이것**:

```
너무 큼:  손실이 발산/진동 (최소점을 건너뜀)
너무 작음: 수렴이 너무 느림 (평생 학습)
적정:     안정적으로 손실 감소
```

- **학습률 스케줄**: 처음 크게 → 점점 작게 (warmup 후 decay)
- **warmup**: 초반엔 작게 시작 (초기 불안정 방지) → 트랜스포머 학습 필수 ([[llm-training]])

## 기울기 소실/폭발

깊은 망에서 역전파([[neural-networks]])는 기울기를 **곱으로 전파** → 문제:

```
소실(vanishing): 작은 값(<1) 곱해짐 → 앞 층 기울기 ≈ 0 → 학습 안 됨
폭발(exploding): 큰 값(>1) 곱해짐 → 기울기 폭발 → 발산(NaN)
```

- **소실 대책**: ReLU 활성화(시그모이드보다 기울기 유지), 잔차 연결(skip), 정규화
- **폭발 대책**: **기울기 클리핑**(gradient clipping - 최대값 제한)
- 시그모이드가 은닉층에서 안 쓰이는 이유가 소실 ([[neural-networks]])

## 초기화 (initialization)

가중치 초기값이 수렴 좌우 (XOR에서 seed 바꾸니 수렴한 이유 [[neural-networks]]):

- **0으로 초기화 금지**: 모든 뉴런 대칭 → 같은 걸 학습 (대칭 깨야)
- **Xavier/He 초기화**: 층 크기에 맞춘 분산 → 기울기 소실/폭발 완화
- 나쁜 초기화 = 학습 실패·local minima

## 옵티마이저

경사하강의 개선판들:

### SGD (확률적 경사하강)

- 전체 데이터 아니라 **미니배치**로 기울기 추정 (빠름·노이즈가 오히려 일반화 도움)
- **모멘텀**: 이전 방향 관성 추가 (골짜기 진동↓, 가속)

### Adam (표준)

- **적응적 학습률**: 파라미터마다 다른 학습률 (기울기 이력 기반)
- 모멘텀 + RMSProp 결합 → 대부분 상황에 잘 됨 (기본 선택)
- 딥러닝·LLM의 사실상 표준 ([[llm-training]])

## 배치 정규화 / 정규화

- **배치 정규화(BatchNorm)**: 층 입력을 정규화(평균0 분산1) → 학습 안정·가속, 큰 학습률 허용
- **LayerNorm**: 배치 아니라 특성 축 정규화 → 트랜스포머 표준 ([[attention-and-transformers]])
- **드롭아웃**: 학습 중 뉴런 랜덤 제거 → 과적합 방지 ([[ml-fundamentals]]의 정규화)

## 배치 크기

- **큼**: 안정적 기울기, GPU 효율↑, 하지만 일반화 약간↓·메모리↑
- **작음**: 노이즈 많지만 일반화·정규화 효과, 메모리↓
- 학습률과 함께 조정 (큰 배치엔 큰 학습률)

## "왜 학습이 안 되나" 체크리스트

실전 디버깅 ([[linux-debugging]]의 계층적 진단처럼):

```
1. 손실이 안 줄어듦 → 학습률 확인 (너무 작나? 너무 커서 발산?)
2. 손실이 NaN → 기울기 폭발 (클리핑), 학습률↓
3. 훈련은 되는데 검증 나쁨 → 과적합 (정규화·데이터↑ [[ml-fundamentals]])
4. 처음부터 안 됨 → 초기화·데이터 전처리·라벨 확인
5. 데이터 정규화했나 (입력 스케일)
6. 작은 데이터로 과적합 되는지 먼저 (모델·코드 검증)
```

- **작은 배치로 과적합 시켜보기**: 모델이 4개 샘플도 못 외우면 버그 (sanity check)

## 왜 이게 중요한가

- 이론상 되는 모델도 **학습 안 되면 무의미**
- 대부분의 실패는 아키텍처 아니라 **학습 설정** (학습률·초기화·데이터)
- 큰 모델일수록 학습 안정성이 관건 ([[llm-training]]의 스케일)
