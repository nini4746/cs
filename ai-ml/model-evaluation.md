# 모델 평가 (Model Evaluation)

## 한 줄 요약

정확도(accuracy)는 불균형 데이터에서 거짓말을 한다 - "전부 음성"만 찍어도 99% 나올 수 있다. 그래서 precision(찾은 게 맞나)/recall(놓치지 않았나)/F1로 나눠 보고, 임계값 전체를 ROC/PR 곡선으로 보며, 교차검증으로 운을 배제한다. **무엇을 최적화할지는 문제에 달렸다.**

## 왜 필요한가

- 왜 정확도만 보면 안 되나 (함정)
- precision/recall/F1의 차이와 언제 무엇을
- 교차검증이 왜 필요한가

## 정확도의 함정

**불균형 데이터**에서 정확도는 무의미할 수 있음:

코드로 확인 (1000명 중 10명만 환자):
```python
# 모델 A: 무조건 "음성" (아무도 병 없다고)
tp, fp, fn, tn = 0, 0, 10, 990
accuracy = (tp+tn)/1000   # = 0.990
```
실행:
```
모델A(전부음성): accuracy=0.990  <- 99% 정확도지만
  recall=0.000 (환자 0명 찾음!)
```

- **99% 정확도인데 환자를 한 명도 못 찾음** - 최악의 모델
- 다수 클래스만 찍으면 정확도 높음 → 소수(중요한) 클래스 무시
- 그래서 **분해해서** 봐야 함

## 혼동 행렬 (confusion matrix)

모든 지표의 토대 - 예측 vs 실제 4칸:

```
              실제 양성     실제 음성
예측 양성   TP(맞은 양성)  FP(오탐, 1종오류)
예측 음성   FN(놓침,2종)   TN(맞은 음성)
```

## precision / recall / F1

```
precision = TP/(TP+FP)   "양성이라 한 것 중 진짜 비율" (오탐 관리)
recall    = TP/(TP+FN)   "진짜 양성 중 찾은 비율"     (놓침 관리)
F1        = 2·P·R/(P+R)  둘의 조화평균 (한쪽만 높으면 낮음)
```

코드로 확인 (모델 B: 8명 잡고 오탐 20명):
```
모델B: accuracy=0.978 precision=0.286 recall=0.800 f1=0.421
```
- 정확도(0.978)는 모델 A(0.990)보다 낮지만 **환자 8명 발견** - 훨씬 유용
- precision 낮음(오탐 많음) vs recall 높음(잘 찾음) → **트레이드오프**가 보임

### 언제 무엇을 (문제가 결정)

- **recall 중요**: 놓치면 큰일 (암 진단, 사기 탐지) - 오탐 감수하고 다 잡기
- **precision 중요**: 오탐이 비쌈 (스팸 분류 - 정상 메일 차단하면 곤란)
- **트레이드오프**: 임계값 낮추면 recall↑ precision↓ (더 많이 양성 판정)

## 임계값과 곡선

로지스틱 출력은 확률([[linear-models]]) → **임계값**으로 양/음 결정. 임계값 전체를 보려면:

- **PR 곡선**(precision-recall): 불균형 데이터에 적합
- **ROC 곡선**: TPR vs FPR, **AUC**(면적)로 요약 (1=완벽, 0.5=랜덤)
- 단일 숫자 말고 **곡선 전체**로 모델 비교

## 교차검증 (cross-validation)

한 번의 훈련/검증 분할은 **운**에 좌우 → k-fold로 안정화:

```
데이터를 k조각 → 각 조각을 한 번씩 검증셋으로, 나머지로 훈련 → k번 평균
```

- 분할 운에 덜 흔들림 (더 정직한 추정)
- 데이터 적을 때 특히 유용
- **주의**: 시계열은 미래 누수 방지 위해 시간 순 분할

## 흔한 함정

- **데이터 누수(leakage)**: 테스트 정보가 훈련에 샘 → 성능 거짓 ([[ml-fundamentals]])
  - 예: 정규화 통계를 전체 데이터로 계산 (테스트 포함) → 누수
- **테스트셋 오염**: 튜닝하며 테스트를 여러 번 봄 → 거기 과적합
- **잘못된 지표**: 불균형인데 정확도 (위)
- **기준선 없음**: "90% 정확도"가 좋은지? 랜덤·다수클래스 기준과 비교해야

## 회귀 지표 (연속값)

분류 아닌 회귀([[linear-models]])는:
- **MAE**(절대오차), **RMSE**(제곱근평균제곱오차 - 큰 오차에 민감)
- **R²**(결정계수): 분산을 얼마나 설명 (1=완벽)

## 왜 평가가 제일 중요한가

- 평가가 틀리면 **나머지 노력이 다 헛됨** (좋은 줄 알았는데 아님)
- "측정할 수 없으면 개선할 수 없다" (software-design/[[testing-strategy]]의 정신)
- LLM 평가는 더 어려움 (정답이 애매) → [[llm-evaluation]]

## 셀프 체크

> [!question]- 불균형 데이터에서 정확도가 왜 거짓말을 하나?
> 다수 클래스만 찍어도 정확도가 높게 나온다. 1000명 중 환자 10명일 때 "전부 음성"이 99% 정확도지만 환자를 하나도 못 찾는다(recall 0). 소수의 중요한 클래스를 무시하므로 분해 지표가 필요하다.

> [!question]- precision과 recall은 각각 무엇을 관리하나?
> precision = TP/(TP+FP), "양성이라 한 것 중 진짜 비율"로 오탐을 관리한다. recall = TP/(TP+FN), "진짜 양성 중 찾은 비율"로 놓침을 관리한다. F1은 둘의 조화평균으로 한쪽만 높으면 낮아진다.

> [!question]- recall이 중요한 문제와 precision이 중요한 문제를 예로 들라.
> recall 중요: 놓치면 큰일 나는 암 진단·사기 탐지(오탐 감수하고 다 잡기). precision 중요: 오탐이 비싼 스팸 분류(정상 메일을 차단하면 곤란). 임계값을 낮추면 recall↑·precision↓로 트레이드오프가 있다.

> [!question]- 교차검증(k-fold)은 무엇을 개선하나?
> 한 번의 훈련/검증 분할은 분할 운에 좌우된다. 데이터를 k조각내 각 조각을 한 번씩 검증셋으로 쓰고 k번 평균 내면 운에 덜 흔들려 더 정직한 추정을 준다. 데이터가 적을 때 특히 유용하며, 시계열은 시간 순 분할로 미래 누수를 막는다.

## 연습문제

> [!example]- 문제: 혼동행렬이 TP=8, FP=20, FN=2, TN=970이다. accuracy, precision, recall, F1을 계산하라.
> **풀이**
> accuracy = (TP+TN)/전체 = (8+970)/1000 = 0.978.
> precision = TP/(TP+FP) = 8/28 ≈ 0.286.
> recall = TP/(TP+FN) = 8/10 = 0.800.
> F1 = 2·P·R/(P+R) = 2·0.286·0.800/(0.286+0.800) ≈ 0.421.
> 정확도는 높지만(0.978) precision이 낮아(오탐 20건) F1이 0.42에 그친다 - 정확도만 봤다면 놓쳤을 약점이다.

> [!example]- 문제: 사기 탐지 모델의 임계값을 0.5에서 0.3으로 낮췄다. precision·recall이 어떻게 변하고 왜 사기 탐지엔 이게 나을 수 있나?
> **풀이**
> 임계값을 낮추면 더 많은 건을 양성으로 판정 → recall↑(더 많은 사기를 잡음), precision↓(오탐도 늘어남).
> 사기 탐지는 놓치면(FN) 손실이 크므로 recall을 우선한다. 오탐은 추가 확인 절차로 걸러낼 수 있어, 사기를 놓치는 것보다 낫다. 단, 오탐 처리 비용과 균형을 봐야 한다.
