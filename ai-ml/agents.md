# 에이전트 (Agents)

## 한 줄 요약

LLM이 한 번 답하고 끝나는 대신 **도구를 호출하고 결과를 보고 다음 행동을 정하는 루프**를 도는 것. LLM이 "무엇을 할지" 계획·결정하고, 도구가 실제 세계와 상호작용(검색·코드 실행·API)한다. MCP는 도구를 표준 방식으로 연결하는 프로토콜. 자율성이 커질수록 신뢰성·안전이 핵심 난제.

## 왜 필요한가

- LLM이 어떻게 행동(검색·계산·실행)하나
- 에이전트 루프가 뭔가
- MCP가 뭔가

## 왜 도구가 필요한가

LLM 혼자의 한계 ([[prompting]]):
```
- 계산 약함([[tokenization]]), 최신 정보 없음, 외부 세계 못 만짐
- 지어냄(환각)
```

**도구 사용**으로 해결:
```
계산 → 계산기/코드 실행    최신 정보 → 검색([[rag]])
데이터 → DB 쿼리·API       행동 → 파일 쓰기, 이메일 전송
```

- LLM은 **뇌**(무엇을 할지 결정), 도구는 **손발**(실제 실행)

## 도구 사용 (function calling)

LLM이 구조화된 도구 호출을 출력:

```
사용자: "서울 날씨 어때?"
LLM → 도구 호출: get_weather(city="서울")   ← JSON으로
시스템이 실행 → 결과 반환: {temp: 28, ...}
LLM → 결과를 문장으로: "서울은 28도입니다"
```

- 모델이 **언제·어떤 도구를·무슨 인자로** 호출할지 결정 (학습·프롬프트로)
- 도구 스키마(이름·설명·인자)를 프롬프트에 제공 → 모델이 선택

## 에이전트 루프 (핵심)

한 번 호출로 안 끝나고 **반복** - 목표 달성까지:

```
while 목표 미달성:
    1. 생각(reasoning): 지금 뭘 해야 하나 ([[prompting]]의 CoT)
    2. 행동(action): 도구 호출
    3. 관찰(observation): 결과 확인
    4. → 결과 보고 다음 행동 결정 (반복)
```

- **ReAct** 패턴: Reasoning + Acting 결합 (생각→행동→관찰 순환)
- 예: "이 버그 고쳐" → 파일 읽기 → 원인 추론 → 수정 → 테스트 실행 → 실패 보고 재수정...
- **계획(planning)**: 큰 목표를 하위 작업으로 분해 후 순차·병렬 실행
- 이 노트 저장소를 만드는 Claude Code도 에이전트 (도구=파일 편집·bash·git)

## MCP (Model Context Protocol)

**도구·데이터를 LLM에 연결하는 표준 프로토콜** (Anthropic):

```
문제: 도구마다 연결 방식 제각각 (N개 앱 × M개 도구 = N×M 통합)
MCP: 표준 인터페이스 → 한 번 만들면 어디서나 (N+M)
```

- **MCP 서버**: 도구·리소스를 표준 방식으로 노출 (DB, 파일시스템, API...)
- **MCP 클라이언트**(LLM 앱): 어떤 MCP 서버든 연결
- USB-C 비유: 표준 포트 하나로 다양한 기기 연결
- 이 세션에도 MCP 서버들 연결됨 (tmap 등) - 표준으로 도구 확장

## 멀티 에이전트

여러 에이전트가 협업:
- **역할 분담**: 계획자·실행자·검증자 (software-design/의 관심사 분리)
- **오케스트레이션**: 상위 에이전트가 하위에 작업 분배
- 복잡한 작업을 나눠 정복 (하지만 조율 비용·오류 전파 주의)

## 난제: 신뢰성·안전

자율성↑ → 위험↑:

- **오류 누적**: 각 단계 작은 오류가 긴 루프에서 증폭 (복리)
- **무한 루프·비용**: 목표 못 찾고 헤맴 → 단계 제한·예산 필요
- **프롬프트 인젝션**: 도구가 읽은 데이터에 악의적 지시 → 에이전트 탈취 ([[prompting]], security/) - **자율 실행이라 특히 위험**
- **권한**: 도구가 실제 행동(파일 삭제·결제) → 최소 권한·확인 (security/[[least-privilege]])
- **검증**: 에이전트 결과를 어떻게 믿나 → [[llm-evaluation]]
- 그래서 위험한 행동은 **사람 확인**(human-in-the-loop) 넣음

## 왜 중요한가

- LLM을 **정적 답변기 → 능동 행위자**로 (코딩·리서치·자동화)
- 실제 세계 작업 자동화의 방향
- 하지만 신뢰성이 실용화의 관건 (자율성 vs 통제)
