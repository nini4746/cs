# 임베딩 (Embeddings)

## 한 줄 요약

단어·문장·이미지를 **의미를 담은 고차원 벡터**로 바꾼 것. 의미가 비슷하면 벡터도 가깝고(코사인 유사도), 방향에 의미가 담겨 벡터 산술(king-man+woman≈queen)도 된다. 유사도 검색·RAG·추천의 토대이며, 검색은 결국 벡터 공간의 최근접 이웃 찾기다.

## 왜 필요한가

- 텍스트를 어떻게 "의미"로 다루나
- 유사도 검색이 어떻게 되나
- RAG·추천의 바탕

## 임베딩이란

```
"cat" → [0.2, -0.5, 0.8, ...]   (예: 768차원 벡터)
의미를 좌표로 → 가까우면 비슷한 의미
```

- 이산 기호(단어)를 **연속 벡터 공간**으로 (신경망이 다룰 수 있게)
- 학습으로 얻음: 비슷한 문맥에 쓰이는 단어는 비슷한 벡터 (분포 가설 - "친구를 보면 그 단어를 안다")
- [[tokenization]]의 토큰이 임베딩 벡터로 → 트랜스포머 입력 ([[attention-and-transformers]])

## 의미 = 거리·방향

### 코사인 유사도

두 벡터의 **방향** 유사도 (크기 무시, -1~1):

```
cos(a,b) = (a·b) / (|a||b|)   1=같은방향, 0=무관, -1=반대
```

### 코드로 확인

장난감 임베딩(축: 동물성/왕족성/크기)으로 유사도·유추:

```python
def cos(a,b): return dot(a,b)/(norm(a)*norm(b))
```

실행:
```
코사인 유사도:
  king: queen=0.988, woman=0.744, man=0.723
  cat:  dog=0.980, woman=0.615, man=0.508

king - man + woman 에 가장 가까운: queen (1.000)
```

- **king ↔ queen**(0.988), **cat ↔ dog**(0.980)이 최고 유사도 (의미 가까움)
- **벡터 산술**: `king - man + woman ≈ queen` - 방향에 "성별"·"왕족성" 의미가 담김
- 이게 임베딩의 마법: 의미 관계가 기하 구조로

## 차원과 SVD

- 고차원(수백~수천) 벡터. 각 차원이 잠재 특성 (해석 어려움)
- **차원축소**로 시각화·압축: PCA/SVD(math/[[svd-basics]]) → 주요 축만
- [[ml-fundamentals]]의 비지도 차원축소와 직결

## 유사도 검색 (최근접 이웃)

임베딩의 핵심 용도 - **쿼리와 가장 가까운 벡터 찾기**:

```
쿼리 "강아지 키우기" → 임베딩 → DB에서 가장 가까운 문서들
```

- **완전 탐색**: 모든 벡터와 거리 계산 (정확하지만 느림, O(n))
- **ANN**(근사 최근접 이웃): 정확도 살짝 포기하고 빠르게 (HNSW, IVF)
  - data-structures/의 공간 분할·그래프 인덱스 아이디어
- **벡터 DB**(Pinecone, Faiss, pgvector): ANN 인덱스로 대규모 유사도 검색 → [[rag]]

## 용도

- **의미 검색**: 키워드 아니라 의미로 (database/[[full-text-search]]의 어휘 매칭을 넘어)
- **RAG**: 관련 문서 검색해 LLM에 제공 ([[rag]]의 핵심)
- **추천**: 비슷한 아이템·사용자 (벡터 근접)
- **클러스터링·분류**: 임베딩 위에서 ([[ml-fundamentals]])
- **중복 탐지**: 의미 유사 문서

## 종류

- **단어 임베딩**: word2vec, GloVe (고정 - 문맥 무시, "bank"가 은행/강둑 구분 못 함)
- **문맥 임베딩**: BERT 등 (문맥 따라 벡터 달라짐 - 트랜스포머 [[attention-and-transformers]])
- **문장/문서 임베딩**: 문장 전체를 벡터 하나로 (검색·RAG용)
- **멀티모달**: 이미지·텍스트를 같은 공간에 (CLIP - "고양이 사진"과 "cat" 텍스트가 가까움)

## 왜 중요한가

- **의미를 계산 가능하게**: 검색·유사도·추천의 수학적 토대
- **RAG의 심장**: 관련 지식 검색 ([[rag]])
- LLM 안팎 어디서나 (입력 임베딩부터 외부 검색까지)

## 연결

- 토큰 → 임베딩 → 트랜스포머 → [[tokenization]], [[attention-and-transformers]]
- 차원축소·SVD → math/[[svd-basics]]
- 비지도·클러스터링 → [[ml-fundamentals]]
- 검색 활용 → [[rag]]
- 의미 vs 어휘 검색 → database/[[full-text-search]]
- ANN 인덱스 → data-structures/
