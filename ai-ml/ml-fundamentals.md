# 머신러닝 기초 (ML Fundamentals)

## 한 줄 요약

데이터에서 규칙을 명시적으로 짜지 않고 **자동으로 함수를 학습**하는 것. 지도(정답 있음)/비지도(정답 없음)로 갈리고, 모든 것의 중심 문제는 **일반화** - 본 적 없는 데이터에서도 잘하기. 이를 위해 데이터를 훈련/검증/테스트로 나누고 과적합을 정규화로 막는다.

## 왜 필요한가

- ML이 전통 프로그래밍과 뭐가 다른가
- 왜 데이터를 나누나 (훈련/검증/테스트)
- 과적합이 뭐고 왜 근본 문제인가

## ML vs 전통 프로그래밍

```
전통: 규칙(코드) + 입력 → 출력      (사람이 규칙 작성)
ML:   입력 + 출력(데이터) → 규칙    (기계가 규칙 학습)
```

- "고양이 판별 규칙"을 손으로 못 짬 → 예시(사진+라벨) 주고 **함수를 학습**시킴
- 본질: 데이터에 맞는 함수 f의 **파라미터를 찾기** (최적화, [[linear-models]]의 경사하강법)

## 학습 종류

### 지도학습 (supervised)

- **입력+정답(label)** 쌍으로 학습
- **분류**(classification): 이산 라벨 (고양이/개, 스팸/정상)
- **회귀**(regression): 연속 값 (집값, 온도)
- 대부분의 실용 ML ([[linear-models]], [[neural-networks]])

### 비지도학습 (unsupervised)

- **정답 없이** 구조 발견
- **군집화**(clustering): 비슷한 것끼리 (k-means)
- **차원축소**: 중요 축 찾기 (PCA, math/[[svd-basics]]) → [[embeddings]]와 연결
- 라벨 비용이 없음, 대신 평가 어려움

### 기타

- **자기지도**(self-supervised): 데이터 자체에서 정답 생성 (다음 단어 맞히기) → LLM 사전학습의 핵심 ([[llm-training]])
- **강화학습**(RL): 보상으로 학습 (RLHF [[llm-training]])

## 핵심 문제: 일반화

ML의 **근본 목표** - 훈련 데이터가 아니라 **새 데이터**에서 잘하기:

```
훈련 데이터 암기 ≠ 학습
목표: 본 적 없는 데이터에 대한 예측
```

### 과적합 vs 과소적합

```
과소적합(underfit): 너무 단순 → 훈련도 못 맞힘 (편향↑)
적정:              패턴 학습, 노이즈 무시
과적합(overfit):   너무 복잡 → 훈련 노이즈까지 암기 → 새 데이터 못함 (분산↑)
```

- **편향-분산 트레이드오프**: 단순하면 편향↑, 복잡하면 분산↑. 균형이 관건
- 과적합 신호: **훈련 정확도 높은데 검증 정확도 낮음**

## 데이터 분리 (왜 3분할)

일반화를 **정직하게 측정**하려면 데이터를 나눔:

```
훈련(train, ~60-80%):   파라미터 학습
검증(validation, ~10-20%): 하이퍼파라미터 조정·모델 선택
테스트(test, ~10-20%):    최종 성능 (딱 한 번, 절대 안 봄)
```

- **테스트를 미리 보면**: 거기 맞춰 튜닝 → 부풀린 성능 (자기기만)
- 검증으로 튜닝하고, 테스트는 **마지막 정직한 시험**
- **데이터 누수(leakage)**: 테스트 정보가 훈련에 새면 성능 거짓 → [[model-evaluation]]

## 정규화 (과적합 방지)

모델 복잡도를 억제:

- **L1/L2 정규화**: 큰 가중치에 페널티 (단순한 모델 선호)
- **드롭아웃**: 학습 중 뉴런 일부 끔 ([[training-dynamics]])
- **조기 종료**(early stopping): 검증 성능 나빠지면 멈춤
- **데이터 늘리기**: 더 많은/증강 데이터 (과적합 최선의 약)

## 왜 이게 토대인가

- 지도/비지도, 일반화, 과적합, 데이터 분리는 **모든 ML의 공통 언어**
- 딥러닝·LLM도 결국 이 틀 위 ([[neural-networks]], [[llm-training]])
- 평가를 제대로 못 하면 나머지가 무의미 ([[model-evaluation]])

## 연결

- 파라미터 찾기 = 최적화 → [[linear-models]]
- 일반화 측정 → [[model-evaluation]]
- 차원축소·비지도 → math/[[svd-basics]], [[embeddings]]
- 자기지도·RL → [[llm-training]]
- 정규화 (드롭아웃 등) → [[training-dynamics]]
- 신경망으로 확장 → [[neural-networks]]
