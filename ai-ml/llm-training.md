# LLM 학습 (LLM Training)

## 한 줄 요약

대형 언어 모델은 3단계로 만들어진다 - 사전학습(방대한 텍스트로 다음 토큰 예측, 지식·언어능력 습득) → SFT(지시-응답 예시로 지시 따르기 학습) → 선호 정렬(RLHF/DPO로 사람 선호에 맞춤). 사전학습이 능력의 대부분을 만들고, 뒤 단계는 그 능력을 유용·안전하게 다듬는다. 스케일링 법칙이 "크게=좋게"를 예측한다.

## 왜 필요한가

- LLM이 어떻게 만들어지나 (파이프라인)
- 사전학습 vs 파인튜닝 vs RLHF의 역할 차이
- 스케일링 법칙이 뭔가

## 1단계: 사전학습 (pretraining)

**방대한 텍스트로 다음 토큰 예측** - 능력의 원천:

```
목표: "The cat sat on the ___" → "mat" 예측
자기지도([[ml-fundamentals]]): 라벨 불필요, 텍스트 자체가 정답
```

- 인터넷·책 규모 텍스트(수조 토큰)로 학습
- **다음 토큰 예측만으로** 문법·사실·추론·번역 능력이 창발 (emergent)
- 트랜스포머 디코더([[attention-and-transformers]])로 구현
- **제일 비쌈**: 수개월·수천 GPU·막대한 비용 (능력의 90%가 여기서)
- 결과: **베이스 모델** - 아는 건 많지만 지시를 안 따름 (그냥 이어쓰기)

## 2단계: SFT (지도 파인튜닝)

**지시-응답 예시**로 "지시 따르기"를 가르침:

```
입력: "파이썬으로 정렬 코드 짜줘"
정답: (사람이 쓴 좋은 응답)
→ 이런 쌍 수만~수십만 개로 파인튜닝
```

- 베이스 모델을 **대화·지시 형식**에 맞춤 ([[fine-tuning]])
- 새 지식 주입 아니라 **행동 형성** (있는 능력을 유용하게)
- 상대적으로 적은 데이터·비용 (사전학습 대비)

## 3단계: 선호 정렬 (RLHF / DPO)

**사람 선호**에 맞춤 - "좋은 응답"의 미묘함을 학습:

### RLHF (인간 피드백 강화학습)

```
1. 같은 질문에 여러 응답 생성
2. 사람이 순위 매김 (A > B > C)
3. 보상 모델 학습 (사람 선호 예측)
4. 강화학습으로 LLM이 보상↑ 방향 (PPO)
```

- SFT로 안 되는 것: 유용성·안전성·톤의 미세 조정
- **정렬(alignment)**: 사람 의도·가치에 맞추기

### DPO (직접 선호 최적화)

- RLHF의 복잡한 RL 없이 **선호쌍으로 직접 최적화** (더 간단·안정)
- 최근 널리 쓰임 (보상 모델·PPO 생략)

## 스케일링 법칙 (scaling laws)

**모델·데이터·계산을 키우면 성능이 예측 가능하게 향상**:

```
손실 ∝ 규모의 거듭제곱 (부드러운 곡선)
→ 얼마나 키우면 얼마나 좋아질지 미리 예측
```

- **Chinchilla**(2022): 주어진 계산 예산에서 **모델 크기와 데이터를 균형** 있게 (당시 모델들은 데이터 부족)
  - 대략 파라미터 : 토큰 = 1 : 20이 최적
- **창발 능력**: 일정 규모 넘으면 갑자기 생기는 능력 (추론, in-context learning) - 논쟁 있음
- 이 예측성이 대규모 투자의 근거

## 학습 규모의 현실

- **분산 학습**: 모델·데이터를 수천 GPU에 쪼갬 (data/model/pipeline 병렬) → distributed-systems/
- **혼합정밀도·최적화**: 메모리·속도 ([[training-dynamics]])
- **warmup + decay** 학습률 필수 ([[training-dynamics]])
- **데이터 품질**: 양만큼 질 중요 (중복 제거, 필터링) - garbage in garbage out

## 전체 그림

```
방대한 텍스트 → [사전학습] → 베이스 모델 (지식·능력)
                                ↓ [SFT] 지시 따르기
                             지시 모델
                                ↓ [RLHF/DPO] 선호 정렬
                             정렬된 어시스턴트 (Claude 등)
```

- 뒤로 갈수록 데이터·비용↓, 하지만 사용성엔 필수
- 추론 시엔 프롬프트([[prompting]])·도구([[agents]])·검색([[rag]])으로 확장

## 셀프 체크

> [!question]- LLM 제작 3단계와 각 역할은?
> (1) 사전학습 - 방대한 텍스트로 다음 토큰 예측, 지식·언어능력의 대부분이 여기서 생김 (2) SFT - 지시-응답 예시로 지시 따르기(행동) 학습 (3) 선호 정렬(RLHF/DPO) - 사람 선호에 맞춰 유용성·안전성·톤을 다듬음. 뒤로 갈수록 비용은 낮지만 사용성엔 필수다.

> [!question]- 사전학습이 왜 자기지도학습인가?
> "The cat sat on the ___"의 정답 "mat"이 텍스트 안에 이미 있어, 별도 라벨 없이 텍스트 자체가 정답이 된다. 다음 토큰 예측만으로 문법·사실·추론 능력이 창발한다.

> [!question]- RLHF와 DPO의 차이는?
> RLHF는 사람 순위로 보상 모델을 학습하고 PPO 강화학습으로 LLM이 보상을 높이게 한다(복잡). DPO는 보상 모델·RL 없이 선호쌍으로 직접 최적화해 더 간단·안정하며 최근 널리 쓰인다.

> [!question]- Chinchilla가 밝힌 스케일링의 교훈은?
> 주어진 계산 예산에서 모델 크기와 데이터를 균형 있게(대략 파라미터:토큰 = 1:20) 써야 한다는 것이다. 당시 모델들은 파라미터에 비해 데이터가 부족해 계산을 비효율적으로 썼다.

## 연습문제

> [!example]- 문제: Chinchilla의 파라미터:토큰 ≈ 1:20 지침을 따를 때, 70억(7B) 파라미터 모델의 대략적 최적 학습 토큰 수는?
> **풀이**
> 7×10⁹ × 20 = 1.4×10¹¹ = 약 1,400억(140B) 토큰.
> 즉 7B 모델은 대략 140B 토큰으로 학습해야 계산 예산 대비 효율적이다. 파라미터만 키우고 데이터가 이보다 적으면 데이터 부족(under-trained)으로 잠재력을 못 살린다.

> [!example]- 문제: 베이스 모델(사전학습만 마친)에 "파이썬 정렬 코드 짜줘"라고 하면 왜 지시를 안 따르고 엉뚱하게 이어쓰나? 어느 단계가 이를 고치나?
> **풀이**
> 베이스 모델은 다음 토큰 예측기라 "그럴듯한 이어쓰기"만 한다. 지시를 명령으로 해석해 응답하는 행동을 배운 적이 없어, 질문을 그냥 텍스트로 이어갈 수 있다.
> SFT(2단계)가 지시-응답 예시로 "지시 따르기"라는 행동을 형성해 이를 고친다. 이후 RLHF/DPO가 응답의 유용성·톤을 더 다듬는다.

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 사전학습→SFT→선호 정렬 각 단계가 무엇을 만드는지 (2) 다음 토큰 예측(자기지도)이 능력의 원천인 이유 (3) 스케일링 법칙과 Chinchilla의 파라미터·데이터 균형.
