# DFA와 NFA (유한 오토마타)

## 한 줄 요약

유한 상태 기계로 문자열을 인식하는 두 모델. DFA는 각 상태·입력에 전이가 하나(결정적), NFA는 여러 개(비결정적)지만 표현력은 같다. NFA→DFA 부분집합 구성으로 변환된다.

## 왜 필요한가

- 계산 이론의 출발점 - 가장 단순한 계산 모델
- 정규식([[regex-and-finite-automata]])·렉서·KMP의 기반
- "비결정성이 능력을 더 주나?"의 첫 답 (안 줌)

## DFA (결정적 유한 오토마타)

5요소 (Q, Σ, δ, q₀, F):
- **Q**: 유한 상태 집합
- **Σ**: 입력 알파벳
- **δ**: 전이 함수 Q×Σ → Q (각 상태·입력에 **정확히 하나**의 다음 상태)
- **q₀**: 시작 상태
- **F**: 수락 상태 집합

동작: 시작 상태에서 입력을 한 글자씩 읽으며 전이. 입력 끝에 수락 상태면 **수락**.

```
"짝수 개의 0을 포함하는 문자열" DFA:
  상태 짝수 ⇄(0) 상태 홀수,  1은 자기 루프
  시작=짝수, 수락=짝수
```

- 메모리 = 현재 상태 하나 (유한). 카운터 없음 → 셀 수 없음
- 각 입력에 전이가 유일 → 실행이 결정적, 한 경로

## NFA (비결정적 유한 오토마타)

DFA와 같지만 전이가 **여러 개 가능**:
- δ: Q×Σ → **2^Q** (다음 상태 집합, 0개~여러 개)
- **ε-전이**: 입력 안 읽고 상태 이동 가능
- 수락: 가능한 경로 **하나라도** 수락 상태로 끝나면 수락

비결정성 = "동시에 여러 상태에 있을 수 있음" 또는 "운 좋은 추측". 어떤 패턴은 NFA로 훨씬 간결하게 표현.

```
"뒤에서 3번째가 1인 문자열" - NFA는 간단, DFA는 상태 많음
```

## DFA = NFA (표현력 동일)

**핵심 정리**: NFA로 인식하는 언어 = DFA로 인식하는 언어 = **정규 언어**.

비결정성이 편의는 주지만 **능력은 안 더함**. 증명 = 변환 구성:

### 부분집합 구성 (subset construction)

NFA를 DFA로 변환. **DFA의 각 상태 = NFA의 상태 집합**:

```
DFA 상태 = "NFA가 현재 있을 수 있는 상태들의 집합"
전이: 그 집합의 각 상태에서 갈 수 있는 곳들의 합집합
수락: 집합에 NFA 수락 상태가 하나라도 포함
```

- NFA 상태 n개 → DFA 상태 최대 **2ⁿ개** (집합의 개수). 상태 폭발 가능
- 하지만 항상 변환 가능 → 표현력 같음 증명

## DFA vs NFA

| | DFA | NFA |
|---|---|---|
| 전이 | 유일 | 여럿 가능 + ε |
| 실행 | 한 경로 | 여러 경로 (병렬) |
| 표현력 | 정규 언어 | 정규 언어 (같음) |
| 상태 수 | 많을 수 있음 | 간결 |
| 실행 속도 | O(n) 확정 | 시뮬레이션 비용 |

**실용**: 정규식→NFA(간결하게 구성)→DFA(빠른 실행)로 변환하는 게 정규식 엔진의 표준 → [[regex-and-finite-automata]], [[regex-engines]].

## 정규 언어의 정의

**정규 언어** = DFA(=NFA=정규식)로 인식 가능한 언어. 유한 오토마타가 인식할 수 있는 것의 한계:

- 할 수 있음: 패턴 매칭, 키워드 인식, 짝/홀 세기(유한), 나눗셈 나머지
- 못 함: 균형 괄호, `aⁿbⁿ`(개수 세기) → 무한 메모리 필요, 유한 상태로 불가 ([[pumping-lemma]])

## 실용 응용

- **렉서/토크나이저**: 프로그래밍 언어 토큰 = 정규 언어 → compilers/[[lexing]]
- **정규식 엔진**: grep, RE2 → [[regex-engines]]
- **KMP 문자열 매칭**: 실패 함수 = DFA → algorithms/[[string-matching]]
- **프로토콜 상태 머신**: TCP 상태 → network/[[tcp-basics]]

## 셀프 체크

> [!question]- DFA의 전이 함수 δ와 NFA의 전이 함수 δ는 치역이 어떻게 다른가?
> DFA는 δ: Q×Σ → Q로 각 상태·입력에 다음 상태가 정확히 하나다. NFA는 δ: Q×Σ → 2^Q로 다음 상태의 집합(0개~여러 개)을 주고, ε-전이로 입력을 안 읽고 이동할 수도 있다.

> [!question]- 부분집합 구성에서 DFA의 한 상태는 무엇에 대응하며, 상태 수는 최대 몇 개인가?
> DFA의 한 상태는 "NFA가 현재 있을 수 있는 상태들의 집합"에 대응한다. NFA 상태가 n개면 부분집합이 2ⁿ개이므로 DFA 상태는 최대 2ⁿ개까지 폭발할 수 있다.

> [!question]- 비결정성은 유한 오토마타의 표현력을 늘리는가?
> 아니다. NFA로 인식하는 언어와 DFA로 인식하는 언어는 정확히 같은 정규 언어다. 부분집합 구성으로 항상 DFA로 변환되므로 비결정성은 간결함만 줄 뿐 능력을 더하지 않는다.

> [!question]- 유한 오토마타가 aⁿbⁿ을 인식하지 못하는 근본 이유는?
> 메모리가 현재 상태 하나(유한)뿐이라 "지금까지 a를 몇 개 봤나"를 무한히 셀 수 없기 때문이다. 카운터가 없어 n이 상태 수를 넘으면 개수를 기억하지 못한다.

## 연습문제

> [!example]- 문제: "뒤에서 2번째 글자가 1인 이진 문자열" 언어의 NFA를 만들고, 부분집합 구성으로 DFA로 변환하라.
> **풀이**
> NFA: 상태 {q0, q1, q2}, 시작 q0, 수락 q2.
> - q0 --0,1--> q0 (자기 루프로 아무 접두사나 소비)
> - q0 --1--> q1 (뒤에서 2번째 후보인 1을 추측)
> - q1 --0,1--> q2 (마지막 한 글자)
> - q2는 수락, 나가는 전이 없음
> 부분집합 구성 (시작 {q0}):
> - {q0} --0--> {q0}, --1--> {q0,q1}
> - {q0,q1} --0--> {q0,q2}, --1--> {q0,q1,q2}
> - {q0,q2} --0--> {q0}, --1--> {q0,q1}
> - {q0,q1,q2} --0--> {q0,q2}, --1--> {q0,q1,q2}
> 수락 상태 = q2를 포함하는 집합, 즉 {q0,q2}와 {q0,q1,q2}. 총 4개 DFA 상태로 "마지막 두 글자"를 기억하는 구조가 된다.

> [!example]- 문제: "0의 개수가 짝수이고 동시에 1의 개수가 짝수인 이진 문자열" DFA를 구성하라.
> **풀이**
> 상태를 (0의 패리티, 1의 패리티) 쌍으로 잡으면 4개: EE, EO, OE, OO. 시작·수락 모두 EE.
> - 0을 읽으면 첫 성분 토글: EE⇄OE, EO⇄OO
> - 1을 읽으면 둘째 성분 토글: EE⇄EO, OE⇄OO
> 유한한 패리티 2개는 상태로 기억 가능하지만, 이는 "개수 자체"가 아니라 나머지(mod 2)만 세는 것이라 유한 상태로 가능하다.

## 파인만

> [!note]- 백지에 DFA와 NFA의 5요소를 각각 적고, 왜 둘이 같은 표현력인지를 부분집합 구성으로 설명하라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) δ의 치역이 Q인가 2^Q인가로 두 모델을 구분할 수 있는가, (2) DFA 상태 = NFA 상태 집합이라는 대응을 말할 수 있는가, (3) 상태 폭발(2ⁿ)이 표현력이 아니라 상태 수의 문제임을 설명할 수 있는가.
