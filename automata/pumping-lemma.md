# 펌핑 보조정리 (Pumping Lemma)

## 한 줄 요약

정규 언어라면 충분히 긴 문자열은 "펌핑"(반복)해도 여전히 그 언어에 속한다. 이 성질을 위반하는 언어는 정규가 아니다. aⁿbⁿ이 정규가 아님을 증명하는 도구.

## 왜 필요한가

- "이 언어는 정규식/DFA로 표현 불가"를 증명하는 법
- 유한 오토마타의 근본 한계
- 왜 균형 괄호를 정규식으로 못 하나

## 직관: 유한 상태의 한계

DFA는 **유한한 상태**만 가짐 ([[dfa-nfa]]). 상태가 p개인 DFA에 길이 ≥ p인 문자열을 넣으면:

- 비둘기집 원리(math/[[combinatorics]]): p+1개 상태를 방문하는데 상태는 p개 → **어떤 상태를 두 번 방문**
- 두 번 방문 = **사이클** → 그 사이클 부분을 반복(펌핑)해도 DFA는 여전히 수락

이 "반복해도 수락"이 정규 언어의 필연적 성질. 이걸 어기면 정규 아님.

## 펌핑 보조정리 (형식)

L이 정규 언어면, **어떤 펌핑 길이 p가 존재**해서 길이 ≥ p인 모든 문자열 s를 s = xyz로 나눌 수 있고:

1. **|y| > 0** (y는 비어있지 않음 - 펌핑할 게 있음)
2. **|xy| ≤ p** (y는 앞쪽 p글자 안)
3. **모든 i ≥ 0에 대해 xyⁱz ∈ L** (y를 0번 이상 반복해도 L에 속함)

y가 사이클 부분. 이걸 반복(i번)해도 언어에 남아야 함.

## 사용법: 비정규성 증명 (반증)

**대우로 증명**: L이 위 성질을 위반하면 정규가 아님. 게임 형식:

1. L이 정규라고 **가정** (모순 유도)
2. 펌핑 길이 p가 존재 (상대가 줌, 값은 모름)
3. 길이 ≥ p인 **영리한 문자열 s를 선택** (내가 고름)
4. 상대의 모든 분할 xyz에 대해 (조건 1,2 만족하는)
5. **어떤 i에서 xyⁱz ∉ L**임을 보임 → 조건 3 위반 → 모순
6. → L은 정규가 아님

## 예: aⁿbⁿ이 정규 아님

L = {aⁿbⁿ : n ≥ 0} (a가 n개 뒤 b가 n개, 개수 같음):

1. 정규라 가정, 펌핑 길이 p
2. **s = aᵖbᵖ 선택** (길이 2p ≥ p)
3. 조건 2(|xy| ≤ p)에 의해 xy는 앞쪽 p글자 = **전부 a**. 따라서 y = a^k (k>0)
4. **i=2로 펌핑**: xy²z = a^(p+k)b^p → a가 b보다 많음
5. a 개수 ≠ b 개수 → aⁿbⁿ 형태 아님 → ∉ L → 모순
6. → **L은 정규가 아님** ∎

직관: DFA는 유한 상태라 "지금까지 a를 몇 개 봤나"를 무한히 셀 수 없음. n이 커지면 못 셈.

## 무엇이 정규가 아닌가

펌핑 보조정리로 증명되는 비정규 언어들 - 공통점은 **무한히 세거나 기억해야** 함:

- **aⁿbⁿ**: 개수 맞추기
- **균형 괄호**: 여는/닫는 짝 (중첩 깊이 셈) → 프로그래밍 언어가 정규가 아닌 이유
- **회문(palindrome)**: 앞뒤 대칭 기억
- **aⁿbⁿcⁿ**: 더 강함 (문맥 자유도 아님 → [[cfl-pumping]])

이들은 문맥 자유 문법([[context-free-grammars]])이나 그 이상이 필요.

## 주의: 필요조건이지 충분조건 아님

- 펌핑 보조정리 **위반 → 비정규** (증명 도구로 유효)
- 펌핑 성질 **만족 ≠ 정규** (일부 비정규 언어도 만족). 정규성을 **증명**하는 데는 못 씀
- 정규 증명은 DFA/정규식을 직접 제시 ([[dfa-nfa]])

Myhill-Nerode 정리가 더 강력한 판별법 (필요충분) → [[dfa-minimization]].

## 연결

- 균형 괄호는 CFG로 → [[context-free-grammars]]
