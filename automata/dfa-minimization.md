# DFA 최소화 (DFA Minimization)

## 한 줄 요약

같은 언어를 인식하는 가장 작은 DFA를 구한다. 구별 불가능한 상태를 병합하며, Myhill-Nerode 정리가 최소 DFA의 유일성과 정규성 판별을 함께 준다.

## 왜 필요한가

- 부분집합 구성([[dfa-nfa]])이 만든 큰 DFA를 줄이기
- 정규식 엔진의 상태 수 최적화 → [[regex-engines]]
- Myhill-Nerode로 비정규성 증명 (펌핑보다 강력)

## 왜 최소화하나

NFA→DFA 변환(부분집합 구성)은 상태가 2ⁿ까지 폭발 ([[dfa-nfa]]). 그중 많은 상태가 **사실 같은 역할** → 병합하면 작아짐:

- 메모리 절약 (임베디드, 대량 패턴)
- 실행은 이미 O(n)이지만 상태 테이블이 작아짐
- **최소 DFA는 유일** (같은 언어면 정확히 하나의 최소형)

## 구별 가능한 상태

두 상태 p, q가 **구별 가능(distinguishable)**: 어떤 문자열 w를 넣었을 때 하나는 수락, 하나는 거절로 갈림. 즉 이후 행동이 다름.

- **구별 불가능**: 모든 w에 대해 둘 다 수락하거나 둘 다 거절 → 같은 역할 → 병합 가능
- 최소화 = 구별 불가능한 상태들을 하나로 합침

## 최소화 알고리즘

### 분할 정제 (partition refinement)

```
1. 상태를 두 그룹으로: 수락 상태 F vs 비수락 Q-F
   (수락/비수락은 빈 문자열로 구별됨 - 시작부터 다름)
2. 각 그룹에서, 어떤 입력에 대해 다른 그룹으로 가는 상태들을 분리
3. 더 이상 나눌 수 없을 때까지 반복
4. 최종 각 그룹 = 최소 DFA의 한 상태
```

- 도달 불가능한 상태 먼저 제거
- O(n²) 또는 Hopcroft 알고리즘으로 O(n log n)

## Myhill-Nerode 정리

최소화의 이론적 기반이자 강력한 판별 도구.

**동치 관계**: 두 문자열 x, y가 동치 ⟺ 모든 z에 대해 xz ∈ L ⟺ yz ∈ L (뒤에 뭘 붙여도 같은 운명).

**정리**: L이 정규 ⟺ 이 동치 관계의 **동치류가 유한개**. 그리고 그 개수 = 최소 DFA의 상태 수.

### 두 가지 힘

1. **최소 DFA 상태 수 = 동치류 개수** → 최소성·유일성 증명
2. **정규성 판별**: 동치류가 무한이면 **비정규**

### 비정규 증명 (펌핑보다 강력)

aⁿbⁿ ([[pumping-lemma]])을 Myhill-Nerode로:

- a, aa, aaa, ... 는 서로 다른 동치류 (aⁱ 뒤에 bⁱ를 붙여야만 수락 → 각 i가 구별됨)
- 무한개의 동치류 → **비정규** ∎

펌핑 보조정리는 필요조건만(만족해도 정규 아닐 수 있음), Myhill-Nerode는 **필요충분** → 더 강력. 정규성을 양방향으로 판별.

## 응용

- **정규식 엔진**: 컴파일된 DFA 최소화로 상태 절감 → [[regex-engines]]
- **하드웨어**: 유한 상태 기계(FSM) 회로 최소화 → 게이트 절약
- **프로토콜/컴파일러**: 상태 머신 단순화
- **이론**: 정규성 판별, 최소성 증명

## 셀프 체크

> [!question]- 두 상태 p, q가 "구별 가능(distinguishable)"하다는 것은 무슨 뜻인가?
> 어떤 문자열 w가 존재해서, p에서 w를 읽으면 수락이고 q에서 w를 읽으면 거절(또는 반대)로 갈리는 것이다. 모든 w에 대해 둘의 운명이 같으면 구별 불가능해서 병합할 수 있다.

> [!question]- Myhill-Nerode 동치 관계는 무엇이며, 어떤 두 가지 힘을 주는가?
> 문자열 x, y가 동치 ⟺ 모든 z에 대해 xz∈L ⟺ yz∈L (뒤에 뭘 붙여도 운명이 같음). 힘: (1) 동치류 개수 = 최소 DFA 상태 수라 최소성·유일성을 주고, (2) 동치류가 무한이면 비정규라는 정규성 판별을 준다.

> [!question]- Myhill-Nerode가 펌핑 보조정리보다 강력한 이유는?
> 펌핑 보조정리는 필요조건만 준다(성질을 만족해도 비정규일 수 있음). Myhill-Nerode는 필요충분이라 정규성을 양방향으로 판별한다 - 동치류 유한이면 정규, 무한이면 비정규.

> [!question]- 왜 최소 DFA는 유일한가?
> 최소 DFA의 상태는 Myhill-Nerode 동치류와 일대일 대응하는데, 동치류는 언어 L만으로 정해진다. 따라서 같은 언어면 (동형을 무시하면) 정확히 하나의 최소형만 존재한다.

## 연습문제

> [!example]- 문제: 분할 정제로 다음 DFA를 최소화하라. 상태 {A,B,C,D}, 시작 A, 수락 {D}. 전이: A-0→B, A-1→C, B-0→B, B-1→D, C-0→C, C-1→D, D-0→D, D-1→D.
> **풀이**
> 1. 초기 분할: 비수락 {A,B,C} | 수락 {D}.
> 2. {A,B,C}를 입력으로 정제. 1을 읽으면 A→C(비수락 그룹), B→D(수락 그룹), C→D(수락 그룹). B와 C는 1에서 수락 그룹으로 가고 A는 비수락 그룹으로 감 → A를 분리: {A} | {B,C} | {D}.
> 3. {B,C} 정제: 0에서 B→B, C→C (둘 다 {B,C}); 1에서 B→D, C→D (둘 다 {D}). 구별 안 됨 → 유지.
> 4. 더 못 나눔. 최소 DFA 상태 = {A}, {B,C}, {D} 3개. B와 C가 병합된다.

> [!example]- 문제: Myhill-Nerode로 L = {aⁿbⁿ : n ≥ 0}이 비정규임을 보여라.
> **풀이**
> 문자열 ε, a, aa, aaa, ... 를 보자. 서로 다른 i, j (i≠j)에 대해 aⁱ와 aʲ는 동치가 아니다: 뒤에 bⁱ를 붙이면 aⁱbⁱ∈L이지만 aʲbⁱ∉L (개수 불일치). 즉 각 aⁱ가 서로 다른 동치류에 속한다.
> 따라서 동치류가 무한개 → Myhill-Nerode에 의해 L은 비정규. ∎

## 파인만

> [!note]- 백지에 분할 정제 알고리즘의 네 단계를 적고, 각 단계가 Myhill-Nerode 동치류와 어떻게 대응하는지 설명하라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 초기 분할이 수락/비수락인 이유(빈 문자열로 구별)를 말할 수 있는가, (2) 최종 각 그룹이 하나의 동치류 = 최소 DFA 한 상태임을 연결할 수 있는가, (3) "동치류 유한 ⟺ 정규"로 정규성을 판별할 수 있는가.
