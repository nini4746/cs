# 튜링 머신 (Turing Machines)

## 한 줄 요약

무한 테이프에 읽고 쓸 수 있는 유한 상태 기계. 가장 강력한 계산 모델이며, "계산 가능"의 정의 자체(처치-튜링 논제)다. 현대 컴퓨터가 할 수 있는 모든 것을 표현한다.

## 왜 필요한가

- "계산 가능하다"의 정확한 정의
- 무엇이 원리적으로 계산 불가능한가의 기준 ([[decidability]])
- 촘스키 위계의 정점

## 정의

튜링 머신(TM):
- **무한 테이프**: 양방향 무한, 칸마다 기호. 읽고 **쓸 수 있음**
- **읽기/쓰기 헤드**: 현재 칸을 읽고, 쓰고, 좌/우 한 칸 이동
- **유한 상태 제어**: DFA처럼 유한 상태
- **전이**: (상태, 읽은 기호) → (새 상태, 쓸 기호, 이동 방향 L/R)
- **수락/거절 상태**: 도달 시 정지

동작: 테이프에 입력을 놓고, 규칙대로 읽기/쓰기/이동 반복. 수락/거절 상태 도달 시 정지 (또는 **영원히 안 멈출 수도** - 무한 루프).

## PDA를 넘어서: 자유로운 메모리

메모리 모델의 진화 ([[dfa-nfa]], [[pushdown-automata]]):

| 모델 | 메모리 | 인식 언어 |
|---|---|---|
| DFA | 없음 (상태만) | 정규 |
| PDA | 스택 (LIFO) | 문맥 자유 |
| **TM** | **무한 테이프 (자유 접근)** | **재귀 열거** |

TM의 테이프는 **아무 데나 읽고 쓸 수 있음** (스택의 LIFO 제약 없음):
- aⁿbⁿcⁿ([[cfl-pumping]]에서 CFL 아니라던 것) 가능
- 사실상 **아무 알고리즘이나** 표현 (충분한 시간/테이프면)

## 처치-튜링 논제 (Church-Turing Thesis)

**"직관적으로 계산 가능한 모든 함수는 튜링 머신으로 계산 가능하다."**

- 정리가 아니라 **논제**(정의에 가까운 믿음) - 증명 대상 아님
- 여러 독립적 계산 모델(람다 계산, 재귀 함수, TM)이 **전부 같은 능력**으로 판명 → 강한 근거
- 함의: TM = 계산 가능성의 기준. "TM이 못 하면 어떤 컴퓨터도 못 함"
- 현대 컴퓨터, 모든 프로그래밍 언어가 TM과 동등 (튜링 완전)

## 변형들은 다 동등

TM을 강화해도 능력이 안 늘어남 (효율만):

- **다중 테이프 TM**: 테이프 여러 개 → 단일 테이프로 시뮬레이션 가능 (같은 능력)
- **비결정적 TM(NTM)**: 여러 경로 → 결정적 TM으로 시뮬레이션 (지수 느리지만 같은 능력). NP의 정의에 사용 → algorithms/[[p-vs-np]]
- **2-스택 PDA**: 스택 2개 = TM
- 이 견고함(robustness)이 처치-튜링 논제를 뒷받침

## 재귀 열거 vs 재귀

TM으로 정의되는 두 언어 클래스 ([[decidability]]):

- **재귀(recursive/decidable)**: TM이 **항상 정지**하며 수락/거절. "결정 가능"
- **재귀 열거(recursively enumerable)**: TM이 수락하는 건 정지하지만, 거절은 **영원히 안 멈출 수도**. "인식 가능"

이 차이(정지 보장 여부)가 결정 가능성 이론의 핵심 → [[decidability]].

## 보편 튜링 머신 (Universal TM)

**다른 TM을 입력으로 받아 시뮬레이션하는 TM**:

- TM의 설명(코드)을 테이프에 놓으면, 그 TM을 실행
- = **저장된 프로그램 컴퓨터**의 이론적 원형 (폰 노이만 구조)
- "프로그램도 데이터"라는 현대 컴퓨팅의 핵심 아이디어. 컴파일러·인터프리터가 이것 → programming-languages/[[compiled-vs-interpreted]]

## 촘스키 위계 정점

```
정규 ⊊ 문맥자유 ⊊ 문맥민감 ⊊ 재귀열거 (튜링)
DFA    PDA        LBA         TM
```

TM이 최상위. 그 위는 없음 (계산 가능성의 한계). 하지만 TM도 **못 하는 게 있음** → [[decidability]].

## 셀프 체크

> [!question]- TM의 전이는 무엇을 받고 무엇을 내는가? PDA 스택과 달리 테이프가 주는 힘은?
> (상태, 읽은 기호) → (새 상태, 쓸 기호, 이동 방향 L/R)이다. 테이프는 아무 칸이나 읽고 쓸 수 있어(LIFO 제약 없음) aⁿbⁿcⁿ 같은 CFL 밖 언어도 인식하며 사실상 임의 알고리즘을 표현한다.

> [!question]- 처치-튜링 논제는 정리인가? 그 근거는?
> 정리가 아니라 논제(정의에 가까운 믿음)라 증명 대상이 아니다. 람다 계산·재귀 함수·TM 등 독립적으로 제안된 여러 계산 모델이 전부 같은 능력으로 판명된 것이 강한 근거다.

> [!question]- 재귀(decidable)와 재귀 열거(recognizable)의 차이는?
> 재귀는 TM이 항상 정지하며 수락/거절한다(결정 가능). 재귀 열거는 수락하는 입력에선 정지하지만 거절할 입력에선 영원히 안 멈출 수도 있다(인식 가능). 정지 보장 여부가 갈림점이다.

> [!question]- 보편 튜링 머신(UTM)이란 무엇이며 왜 중요한가?
> 다른 TM의 설명(코드)을 입력으로 받아 그 TM을 시뮬레이션하는 TM이다. "프로그램도 데이터"라는 저장된 프로그램 컴퓨터(폰 노이만 구조)의 이론적 원형이며, 컴파일러·인터프리터가 이 아이디어다.

## 연습문제

> [!example]- 문제: L = {aⁿbⁿcⁿ : n ≥ 0}을 결정하는 TM의 고수준 알고리즘을 설계하라 (CFL이 아니므로 PDA로는 불가).
> **풀이**
> 테이프에 입력을 놓고 반복한다.
> 1. 왼쪽부터 스캔해 표시 안 된 첫 a를 X로 표시.
> 2. 오른쪽으로 이동해 표시 안 된 첫 b를 Y로 표시.
> 3. 이어서 표시 안 된 첫 c를 Z로 표시. (도중 순서가 a..b..c가 아니면 거절)
> 4. 테이프 왼쪽으로 되돌아가 1로 반복.
> 5. a가 더 없을 때, 남은 b나 c가 있으면 거절, 전부 표시됐으면 수락.
> 테이프를 여러 번 왕복하며 각 a·b·c를 하나씩 짝지어 지운다. 스택 하나로는 못 하던 세 개수 일치를 테이프의 자유 접근으로 해결한다.

> [!example]- 문제: 이진수를 1 증가시키는 TM을 설계하라 (입력: 테이프에 이진수, LSB가 오른쪽 끝).
> **풀이**
> 1. 헤드를 오른쪽 끝(LSB)으로 이동.
> 2. 오른쪽에서 왼쪽으로: 현재 칸이 1이면 0으로 쓰고 계속 왼쪽(자리올림 전파), 0이면 1로 쓰고 정지(수락).
> 3. 모든 자리가 1이었으면(전부 0이 됨) 맨 앞 왼쪽 빈칸에 1을 써서 자리 확장 후 정지.
> 예: 0111 → 1000. 자리올림을 헤드 이동으로 처리하는 게 핵심이다.

## 파인만

> [!note]- 백지에 DFA·PDA·TM의 메모리 모델과 인식 언어 클래스를 표로 그리고, "TM = 계산 가능성의 기준"이 무슨 뜻인지 처치-튜링 논제로 설명하라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 테이프의 자유 접근이 스택 LIFO보다 강한 이유를 말할 수 있는가, (2) 처치-튜링이 논제(증명 불가)임을 설명할 수 있는가, (3) 재귀 vs 재귀 열거를 정지 보장으로 구분할 수 있는가.
