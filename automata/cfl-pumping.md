# 문맥 자유 펌핑 보조정리 (CFL Pumping Lemma)

## 한 줄 요약

문맥 자유 언어도 충분히 긴 문자열은 두 부분을 동시에 펌핑할 수 있다. 이 성질을 위반하면 문맥 자유가 아니다. aⁿbⁿcⁿ이 CFL이 아님을 증명하는 도구.

## 왜 필요한가

- "이 언어는 CFG/PDA로 표현 불가"를 증명
- 스택 하나의 근본 한계 ([[pushdown-automata]])
- 촘스키 위계의 다음 경계 (CFL → 문맥 민감)

## 정규 펌핑과의 차이

[[pumping-lemma]]는 **한 부분**(y)을 펌핑. CFL 펌핑은 **두 부분**(v, y)을 동시에:

- 이유: 파스 트리([[context-free-grammars]])가 충분히 크면 **같은 변수가 경로에 두 번** 나타남 (비둘기집)
- 그 변수 사이를 반복 → 트리의 두 위치가 동시에 펌핑됨
- 그래서 s = uvxyz로 나뉘고 v, y가 함께 반복

## 형식

L이 CFL이면 펌핑 길이 p가 존재해 길이 ≥ p인 모든 s = uvxyz로 나눌 수 있고:

1. **|vy| > 0** (둘 중 하나는 비지 않음)
2. **|vxy| ≤ p** (v,x,y가 가까이)
3. **모든 i ≥ 0: uvⁱxyⁱz ∈ L** (v와 y를 같은 횟수 i로 반복)

v, y가 파스 트리에서 반복 변수의 좌우 확장 부분.

## 예: aⁿbⁿcⁿ이 CFL 아님

L = {aⁿbⁿcⁿ : n ≥ 0} (세 글자 개수 모두 같음):

1. CFL이라 가정, 펌핑 길이 p
2. **s = aᵖbᵖcᵖ 선택**
3. 조건 2(|vxy| ≤ p)에 의해 vxy는 **최대 두 종류 글자만** 걸침 (a와 b, 또는 b와 c - 셋을 다 걸칠 만큼 안 넓음)
4. i=2로 펌핑하면:
   - vy가 a,b만 포함 → a,b는 늘지만 **c는 그대로** → 개수 안 맞음
   - 어느 경우든 세 개수가 깨짐
5. → uv²xy²z ∉ L → 모순
6. → **aⁿbⁿcⁿ는 CFL이 아님** ∎

직관: 스택 하나([[pushdown-automata]])로 a-b는 맞추지만, 그러면 스택이 소진돼 c를 또 맞출 수 없음. **두 개의 독립적 개수 관계**를 스택 하나로 못 함.

## 무엇이 CFL이 아닌가

CFL 펌핑으로 증명되는 것들 - **두 개 이상의 독립 개수/매칭**:

- **aⁿbⁿcⁿ**: 세 개수 일치
- **ww** (같은 문자열 두 번, w wᴿ 회문과 달리): 순서까지 매칭
- **{aⁱbʲcᵏ : i=j=k}** 등
- 프로그래밍 언어의 **변수 선언-사용 일치**: 같은 이름 매칭이 이 형태 → 그래서 파싱(CFG) 후 의미 분석 별도 → programming-languages/[[semantic-analysis]]

이들은 **문맥 민감 문법**(스택 이상)이 필요.

## 필요조건 주의

정규 펌핑과 같이:
- CFL 펌핑 **위반 → CFL 아님** (증명 도구)
- 펌핑 성질 **만족 ≠ CFL** (일부 비CFL도 만족)
- CFL 증명은 CFG/PDA를 직접 제시

## 촘스키 위계 위치

```
정규 ⊊ 문맥자유 ⊊ 문맥민감 ⊊ 재귀열거
       (aⁿbⁿ 가능)  (aⁿbⁿcⁿ 가능)
```

- 정규 펌핑: 정규 vs 문맥자유 경계 (aⁿbⁿ)
- CFL 펌핑: 문맥자유 vs 문맥민감 경계 (aⁿbⁿcⁿ)
- 각 계층의 한계를 각 펌핑 보조정리가 증명 → [[turing-machines]]

## 왜 CFL이 닫혀있지 않나 (곁가지)

CFL은 **교집합에 닫혀있지 않음**: aⁿbⁿcᵐ (CFL) ∩ aᵐbⁿcⁿ (CFL) = aⁿbⁿcⁿ (CFL 아님). 정규 언어와 다른 성질.

## 연결

- 정규 펌핑 (한 부분) → [[pumping-lemma]]
- 스택 하나의 한계 → [[pushdown-automata]]
- CFG 파스 트리 (반복 변수) → [[context-free-grammars]]
- 문맥 민감이 필요한 것 → [[turing-machines]]
- 변수 선언-사용은 의미분석 → programming-languages/[[semantic-analysis]]
