# 정규식과 유한 오토마타 (Regex ↔ Automata)

## 한 줄 요약

정규 표현식과 유한 오토마타(DFA/NFA)는 정확히 같은 언어 집합을 표현한다(Kleene 정리). 정규식 → NFA → DFA로 상호 변환되며, 이것이 실제 정규식 엔진의 기반이다.

## 왜 필요한가

- 정규식이 이론적으로 무엇인가
- 왜 정규식 엔진이 오토마타로 구현되나
- "정규식으로 표현 가능"의 정확한 경계

## 정규 표현식

문자열 패턴을 대수적으로 표현. 세 기본 연산:

- **연결(concatenation)**: `ab` = a 다음 b
- **합집합(union/alternation)**: `a|b` = a 또는 b
- **클레이니 스타(Kleene star)**: `a*` = a를 0번 이상

이것들의 조합. `(a|b)*abb` = a나 b 아무거나 온 뒤 abb로 끝. 이론적 정규식은 이 세 연산만 (실용 정규식의 `+`, `?`, `[...]`는 이들의 축약).

## Kleene 정리: 정규식 = 유한 오토마타

**핵심 정리**: 정규식으로 표현 가능한 언어 = 유한 오토마타로 인식 가능한 언어 = **정규 언어**.

세 표현이 정확히 동등 ([[dfa-nfa]]):

```
정규식 ⟺ NFA ⟺ DFA   (전부 정규 언어)
```

어느 것으로 표현되면 나머지로도 변환 가능. 상호 변환:

## 정규식 → NFA (Thompson 구성)

정규식의 각 연산을 작은 NFA 조각으로 조립:

- **문자 a**: 두 상태 + a 전이
- **연결 RS**: R의 끝을 S의 시작에 ε-전이로 연결
- **합집합 R|S**: 새 시작에서 R,S로 ε-분기
- **스타 R\***: R을 ε-루프로 감쌈

재귀적으로 조립 → 정규식 하나가 NFA 하나로. ε-전이를 활용해 간결하게. 이것이 정규식 컴파일의 첫 단계.

## NFA → DFA → 최소 DFA

1. **NFA → DFA**: 부분집합 구성 ([[dfa-nfa]]) - 각 DFA 상태 = NFA 상태 집합
2. **DFA 최소화**: 동등한 상태 병합 → 최소 상태 DFA ([[dfa-minimization]])

결과: 정규식 → 실행 가능한 최소 DFA. **입력을 O(n)에 매칭** (각 글자당 전이 하나).

## DFA → 정규식

역방향도 가능 (state elimination): DFA 상태를 하나씩 제거하며 간선에 정규식을 붙임 → 최종적으로 하나의 정규식. 세 표현의 완전한 동등성 확인.

## 왜 이게 중요한가

이 변환 체인이 **실제 정규식 엔진**의 기반:

```
정규식 소스 → NFA (Thompson) → DFA → 매칭 실행
```

- **RE2, grep, Go regexp**: 이 오토마타 기반 → 입력 길이에 **선형 시간 보장** ([[regex-engines]])
- 오토마타로 컴파일하면 백트래킹 없이 O(n) 매칭
- 반면 PCRE 계열은 백트래킹 → 이론적 정규식을 넘어서지만 지수 시간 위험 (ReDoS) → [[regex-engines]]

## 실용 정규식의 확장

이론적 정규식은 세 연산뿐. 실용 정규식의 편의 문법:

| 문법 | 의미 | 정규? |
|---|---|---|
| `a+` | a 1번 이상 = `aa*` | ✓ |
| `a?` | a 0 또는 1번 = `(a\|ε)` | ✓ |
| `[a-z]` | 문자 클래스 = `a\|b\|...` | ✓ |
| `.` | 임의 문자 | ✓ |
| **`\1` (역참조)** | 앞 그룹 재등장 | **✗ 정규 아님!** |
| **lookbehind** | | ✗ |

역참조가 붙은 "정규식"은 이론상 **정규 언어를 넘어섬** - 이름만 정규식. 이게 백트래킹이 필요한 이유이자 성능 함정 → [[regex-engines]].

## 셀프 체크

> [!question]- Kleene 정리는 무엇을 말하는가?
> 정규 표현식으로 표현 가능한 언어 = 유한 오토마타(NFA/DFA)로 인식 가능한 언어 = 정규 언어. 세 표현이 정확히 동등하며, 어느 하나로 표현되면 나머지로도 변환 가능하다.

> [!question]- 이론적 정규식의 기본 연산 세 가지는? 실용 정규식의 `+`, `?`, `[a-z]`는 정규인가?
> 연결, 합집합(alternation), 클레이니 스타 세 가지다. `+`, `?`, `[a-z]`, `.`는 모두 이 세 연산의 축약이라 여전히 정규다. 하지만 역참조(`\1`)는 정규 언어를 넘어선다.

> [!question]- Thompson 구성이 하는 일은?
> 정규식의 각 연산(문자, 연결, 합집합, 스타)을 작은 NFA 조각으로 만들어 ε-전이로 재귀적으로 조립한다. 결과는 정규식 하나에 대응하는 NFA 하나로, 정규식 컴파일의 첫 단계다.

> [!question]- 역참조가 붙은 "정규식"이 이론적 정규식을 넘어서는 이유는?
> 역참조는 앞서 매칭한 임의 길이의 그룹을 그대로 다시 요구한다(예: 같은 문자열 반복). 이는 유한 오토마타로 인식 못 하는 언어(예: ww 꼴)를 표현하므로 정규 언어 밖이며, 그래서 백트래킹이 필요하다.

## 연습문제

> [!example]- 문제: 정규식 `(a|b)*abb`를 Thompson 구성으로 NFA로 조립하는 과정을 서술하라.
> **풀이**
> 안쪽부터 조각을 만들어 조립한다.
> 1. `a`: 상태 s1 --a--> s2. `b`: s3 --b--> s4.
> 2. `a|b`: 새 시작 s0에서 ε로 s1, s3에 분기, s2·s4에서 ε로 공통 끝 s5로 합류.
> 3. `(a|b)*`: 위 조각을 ε-루프로 감쌈 - 새 시작 s6에서 ε로 조각 시작과 최종 수락 후보로, 조각 끝에서 ε로 조각 시작(반복)과 밖으로.
> 4. 뒤의 `abb`: 세 문자 조각을 연결(각 끝을 다음 시작에 ε 또는 직접 연결).
> 5. 전체 연결: `(a|b)*` 끝을 `a`의 시작에 ε로 이어 붙임. 최종 상태는 마지막 b의 도착 상태.
> 결과 NFA는 부분집합 구성으로 DFA로 만들면 O(n) 매칭이 된다.

> [!example]- 문제: `a?` = `(a|ε)`, `a+` = `aa*` 임을 이용해 `a+b?`와 동치인, 세 기본 연산만 쓴 정규식을 작성하라.
> **풀이**
> `a+` = `aa*` (a 하나 뒤 a를 0번 이상). `b?` = `(b|ε)` (b 0 또는 1번). 연결하면:
> `aa*(b|ε)`
> 이는 편의 문법 `+`, `?`가 세 기본 연산의 축약일 뿐 표현력을 넘지 않음을 보여준다.

## 파인만

> [!note]- 백지에 "정규식 ⟺ NFA ⟺ DFA" 삼각형을 그리고 각 변환(Thompson, 부분집합 구성, state elimination)의 방향과 역할을 채워라. 막히면 그 변환만 다시.
> **점검 포인트**: (1) 세 표현이 모두 정규 언어를 나타냄을 말할 수 있는가, (2) 정규식→NFA→DFA가 실제 엔진 컴파일 파이프라인임을 연결할 수 있는가, (3) 역참조가 왜 이 삼각형 밖인지 설명할 수 있는가.
