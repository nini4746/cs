# 결정 가능성 (Decidability)

## 한 줄 요약

어떤 문제는 알고리즘으로 절대 풀 수 없다. 정지 문제가 대표 - "이 프로그램이 멈추는가"를 판정하는 프로그램은 존재할 수 없다. 대각선 논법으로 증명된다.

## 왜 필요한가

- 계산에 원리적 한계가 있다는 충격적 사실
- 왜 완벽한 버그 검출기/바이러스 검출기가 불가능한가
- 컴퓨터 과학의 가장 깊은 결과 중 하나

## 결정 가능 vs 인식 가능

튜링 머신([[turing-machines]])으로 정의:

- **결정 가능(decidable)**: TM이 **항상 정지**하며 예/아니오. 완전한 알고리즘
- **인식 가능(recognizable, r.e.)**: 예인 경우는 정지해 수락, **아니오면 영원히 안 멈출 수도**
- **결정 불가**: 어떤 TM도 항상 정지하며 못 푸는 문제

핵심 질문: 모든 문제가 결정 가능한가? **아니다.**

## 정지 문제 (Halting Problem)

**"프로그램 P와 입력 x가 주어졌을 때, P(x)가 정지하는가?"**

이걸 판정하는 알고리즘 HALT(P, x)가 존재할 것 같지만 - **존재할 수 없음**. 튜링의 증명 (1936):

### 대각선 논법 증명

1. HALT(P, x)가 존재한다고 **가정** (P가 x에서 정지하면 true)
2. 이걸로 이상한 프로그램 D를 만듦:

```
D(P):
   if HALT(P, P) == true:   # P가 자기 자신 입력에 정지하면
       무한 루프            # D는 안 멈춤
   else:
       정지
```

3. **D(D)를 실행**하면?
   - D(D)가 정지한다면 → HALT(D,D)=true → D는 무한 루프 → **정지 안 함** (모순)
   - D(D)가 안 멈춘다면 → HALT(D,D)=false → D는 정지 → **정지함** (모순)
4. 어느 쪽도 모순 → **HALT는 존재할 수 없음** ∎

자기 참조(D가 자신을 입력으로)가 모순을 만듦 - 칸토어의 대각선 논법([[sets-relations-functions]])과 같은 구조. "거짓말쟁이 역설"의 계산 버전.

## 정지 문제의 함의

정지 문제가 결정 불가 → 많은 실용 문제도 결정 불가 (환원 → [[reductions]]):

- **완벽한 버그 검출**: "이 프로그램이 크래시하나"를 항상 판정 불가
- **완벽한 바이러스 검출**: "이 코드가 악성인가" 일반적으로 불가
- **코드 최적화 한계**: "이 두 프로그램이 같은가" 불가
- **완벽한 데드락 검출**: 일반적으로 불가

그래서 실무 도구는 **근사**: 보수적(거짓 양성 허용), 특정 케이스만, 휴리스틱. 완벽은 원리적으로 불가능.

## Rice 정리

정지 문제를 일반화한 강력한 결과:

**"프로그램의 의미적 성질(무엇을 계산하나) 중 자명하지 않은 것은 전부 결정 불가."**

- 자명하지 않음 = 어떤 프로그램은 만족, 어떤 건 안 함
- 예: "이 프로그램이 항상 0을 출력하나", "이 함수가 상수 함수인가" 등 전부 결정 불가
- 함의: 프로그램의 **행동**에 관한 흥미로운 질문은 대개 결정 불가. 구문(syntax)은 판정 가능하지만 의미(semantics)는 아님

## 결정 가능한 것들 (대조)

전부 불가능한 건 아님. 결정 가능한 문제:

- 정규 언어 소속 (DFA 실행 → [[dfa-nfa]])
- CFL 소속 (CYK 파싱 → [[cfg-to-parsing]])
- 소수 판정, 정렬 검증 등 대부분 일상 알고리즘
- 유한한 탐색으로 끝나는 것

경계: **유한 시간 보장**이 있으면 결정 가능. 무한 탐색 가능성(정지 여부, 프로그램 의미)이면 위험.

## 계층 정리

```
결정 가능 ⊊ 인식 가능(r.e.) ⊊ 모든 언어
정렬 등     정지 문제(수락측)   대부분(비가산 무한)
```

- 정지 문제는 인식 가능(멈추면 수락)하지만 결정 불가
- 정지 문제의 **여집합**은 인식조차 불가
- 대부분의 언어는 인식조차 불가 (언어는 비가산, TM은 가산 → 셀 수 없이 많은 언어가 어떤 TM에도 대응 안 됨)

## 셀프 체크

> [!question]- 결정 가능(decidable), 인식 가능(recognizable), 결정 불가는 각각 무엇인가?
> 결정 가능: TM이 항상 정지하며 예/아니오를 낸다(완전한 알고리즘). 인식 가능(r.e.): 예인 경우는 정지해 수락하지만 아니오면 영원히 안 멈출 수도 있다. 결정 불가: 어떤 TM도 항상 정지하며 풀 수 없다.

> [!question]- 정지 문제의 대각선 논법 증명에서 모순을 만드는 핵심 장치는?
> 자기 참조다. HALT를 이용해 만든 D가 자기 자신 D를 입력으로 받아 실행되면(D(D)), "정지하면 무한루프, 안 멈추면 정지"라는 양쪽 모두 모순이 나온다. 거짓말쟁이 역설의 계산 버전이다.

> [!question]- Rice 정리는 무엇을 말하는가?
> 프로그램의 의미적(무엇을 계산하나) 성질 중 자명하지 않은 것은 전부 결정 불가라는 것이다. "항상 0을 출력하나", "상수 함수인가" 같은 행동 질문은 대개 결정 불가이며, 구문(syntax)은 판정 가능해도 의미(semantics)는 아니다.

> [!question]- 결정 가능한 문제와 그렇지 않은 문제를 가르는 경계는?
> 유한 시간 보장이 있으면(유한 탐색으로 끝나면) 결정 가능하다. DFA 소속·CYK·소수 판정 등이 그 예다. 무한 탐색 가능성(정지 여부, 프로그램 의미)이 개입하면 결정 불가일 수 있다.

## 연습문제

> [!example]- 문제: 정지 문제 HALT(P,x)가 존재한다는 가정에서 모순을 유도하는 대각선 논법을 처음부터 재구성하라.
> **풀이**
> 1. HALT(P,x)가 존재한다고 가정 (P가 x에서 정지하면 true).
> 2. 프로그램 D 정의:
>    D(P): if HALT(P,P)==true → 무한 루프; else → 정지.
> 3. D(D)를 실행하면?
>    - D(D)가 정지한다 가정 → HALT(D,D)=true → 정의상 D는 무한 루프 → 정지 안 함 (모순).
>    - D(D)가 안 멈춘다 가정 → HALT(D,D)=false → 정의상 D는 정지 → 정지함 (모순).
> 4. 어느 쪽도 모순이므로 HALT는 존재할 수 없다. ∎

> [!example]- 문제: Rice 정리로 "주어진 프로그램이 항상 42를 출력하는가?"가 결정 불가임을 (환원 아이디어로) 논증하라.
> **풀이**
> 이 성질은 의미적(프로그램이 무엇을 계산하나)이고 자명하지 않다: 항상 42를 출력하는 프로그램도 있고(예: `print(42)`) 아닌 것도 있으므로. Rice 정리의 전제(의미적 + 비자명)를 만족하므로 곧바로 결정 불가다.
> 환원 관점: 만약 이 성질을 판정할 수 있다면, 임의의 P(x)에 대해 "P(x)를 실행하고 나서 42를 출력하는" P'를 만들어 물어보면 P(x)의 정지 여부를 알 수 있다 → 정지 문제 판정 → 모순. ∎

## 파인만

> [!note]- 백지에 정지 문제 증명의 D 구성과 D(D) 두 경우 분석을 적고, 왜 이것이 대각선 논법(칸토어)과 같은 구조인지 설명하라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 결정 가능 vs 인식 가능을 정지 보장으로 구분할 수 있는가, (2) 자기 참조 D(D)가 모순을 만드는 메커니즘을 재현할 수 있는가, (3) Rice 정리로 "행동에 관한 질문은 대개 불가"를 일반화할 수 있는가.
