# 환원 (Reductions)

## 한 줄 요약

문제 A를 문제 B로 변환해 "B를 풀면 A도 풀린다"를 보이는 기법. 결정 불가능성을 전파하고(A가 불가능하면 B도), NP-완전성을 증명하는(A만큼 B도 어렵다) 핵심 도구.

## 왜 필요한가

- 정지 문제([[decidability]]) 하나로 수많은 문제의 불가능성 증명
- NP-완전성 증명의 메커니즘 → algorithms/[[p-vs-np]]
- "이 문제가 저 문제만큼 어렵다"의 형식화

## 환원의 아이디어

A를 B로 **환원(reduce)**: A의 인스턴스를 B의 인스턴스로 변환하는 함수 f가 존재해:

```
A의 답(x) = B의 답(f(x))
```

즉 "A를 풀려면 f로 변환 후 B를 풀면 됨". 이게 되면:

- **B를 풀 수 있으면 → A도 풀 수 있음** (변환 후 B 사용)
- 대우: **A를 풀 수 없으면 → B도 풀 수 없음** (B가 풀리면 A도 풀릴 테니 모순)

**A ≤ B** 표기: "A가 B로 환원됨" = "B가 A만큼은 어렵다".

## 방향 주의 (핵심)

환원 방향이 결론을 정함:

- **불가능성 전파**: A가 불가능하고 A ≤ B면 → B도 불가능. **어려운 A를 새 문제 B로 환원**
- **가능성 전파**: B가 가능하고 A ≤ B면 → A도 가능

증명하려는 것에 따라 방향이 반대. 헷갈리면 "무엇을 이미 알고, 무엇을 보이려 하나"로 정리.

## 결정 불가능성 증명

정지 문제([[decidability]])가 불가능 → 다른 문제도 불가능을 환원으로:

**예: "프로그램이 특정 입력에 특정 출력을 내나?"** 가 결정 불가:

```
정지 문제를 이 문제로 환원:
  P(x)가 정지하나? 를 판정하려면
  → P를 변형: P(x) 실행 후 "42" 출력하는 P' 만듦
  → "P'(x)가 42를 출력하나?"를 물음
  → 이게 판정되면 정지 문제도 판정됨 (모순, 정지문제 불가)
  → 이 출력 문제도 결정 불가 ∎
```

정지 문제 ≤ 출력 문제. 정지가 불가능하니 출력 문제도 불가능. 이렇게 **하나의 불가능성**(정지)에서 수많은 불가능성이 파생 → Rice 정리([[decidability]])의 일반화.

## NP-완전성 증명

환원의 또 다른 용도 (다항 시간 환원) → algorithms/[[p-vs-np]]:

- **A ≤ₚ B** (다항 시간 환원): A를 다항 시간에 B로 변환
- A가 NP-완전이고 A ≤ₚ B면 → B도 NP-hard
- **NP-완전 증명법**: 알려진 NP-완전 문제(SAT 등)를 새 문제로 환원

```
3-SAT ≤ₚ 클리크 ≤ₚ 정점커버 ≤ₚ ...
Cook-Levin이 SAT을 첫 NP-완전으로 → 이후 전부 환원 체인
```

여기선 방향이 "어려운 문제를 새 문제로" - 새 문제도 최소한 그만큼 어렵다.

## 환원의 종류

| 종류 | 용도 |
|---|---|
| **매핑 환원(many-one)** | 결정 가능성 (인스턴스→인스턴스) |
| **튜링 환원** | 더 일반 (B를 서브루틴으로 여러 번) |
| **다항 시간 환원(≤ₚ)** | NP-완전성 (변환이 다항 시간) |
| **로그 공간 환원** | 더 세밀한 복잡도 클래스 |

변환의 **자원 제한**이 어떤 결론을 얻는지 정함 (계산 가능성 vs 복잡도).

## 왜 강력한가

환원은 **한 번의 어려움을 무한히 재사용**:

- 정지 문제 하나 → 수천 개 결정 불가 문제
- SAT 하나 → 수천 개 NP-완전 문제
- "새 문제가 어렵나?"를 알려진 어려운 문제로의 환원으로 즉시 답
- 문제들 사이의 **상대적 난이도 지도**를 그림

## 셀프 체크

> [!question]- "A ≤ B" (A를 B로 환원)는 무엇을 뜻하며, 어느 문제가 더 어렵다는 것인가?
> A의 인스턴스를 B의 인스턴스로 변환하는 함수 f가 있어 A의 답(x) = B의 답(f(x))이 성립하는 것이다. B를 풀면 A도 풀리므로 "B가 A만큼은 어렵다".

> [!question]- 불가능성을 전파하려면 어느 방향으로 환원해야 하는가?
> 이미 불가능한 A를 새 문제 B로 환원한다(A ≤ B). 그러면 B가 풀린다면 A도 풀릴 테니 모순 → B도 불가능. 방향을 반대로 하면 결론이 달라지므로 "무엇을 이미 알고 무엇을 보이려 하나"로 정리한다.

> [!question]- NP-완전성 증명은 어느 방향으로 환원하는가?
> 알려진 NP-완전 문제 A를 새 문제 B로 다항 시간 환원한다(A ≤ₚ B). A가 NP-완전이고 A ≤ₚ B면 B도 NP-hard다 - 새 문제가 최소한 A만큼 어렵다.

> [!question]- 환원의 종류가 다른 결론을 주는 이유는?
> 변환에 걸린 자원 제한이 결론을 정한다. 매핑 환원은 계산 가능성(결정 가능성), 다항 시간 환원(≤ₚ)은 NP-완전성, 로그 공간 환원은 더 세밀한 복잡도 클래스를 다룬다.

## 연습문제

> [!example]- 문제: 정지 문제를 이용해 "프로그램 P가 입력 x에서 값 42를 출력하는가?"(OUTPUT 문제)가 결정 불가임을 환원으로 증명하라.
> **풀이**
> 정지 문제 HALT를 OUTPUT으로 환원한다(HALT ≤ OUTPUT).
> 1. HALT(P,x)를 판정하고 싶다.
> 2. 변환: P를 변형해 "P(x)를 실행한 뒤 42를 출력하는" 프로그램 P'를 만든다(다른 출력은 하지 않게).
> 3. 이제 OUTPUT(P', x)를 묻는다. P(x)가 정지하면 P'는 42를 출력, 정지 안 하면 P'는 42에 도달 못 함.
> 4. 즉 OUTPUT(P',x)=true ⟺ HALT(P,x)=true. OUTPUT을 판정할 수 있다면 HALT도 판정됨 → 정지 문제 결정 불가에 모순.
> 5. → OUTPUT 문제도 결정 불가. ∎

> [!example]- 문제: 새 결정 문제 B가 NP-완전임을 증명하려는 표준 절차(골격)를 서술하라.
> **풀이**
> 두 가지를 보여야 한다.
> 1. B ∈ NP: B의 답(yes 인스턴스)에 대한 증거(certificate)를 다항 시간에 검증할 수 있음을 보인다.
> 2. B가 NP-hard: 알려진 NP-완전 문제 A(예: 3-SAT)를 골라 A ≤ₚ B 환원을 구성한다. 즉 A의 임의 인스턴스를 다항 시간에 B의 인스턴스로 변환하되, A가 yes ⟺ B가 yes가 되도록 한다.
> 두 조건이 성립하면 B는 NP-완전이다. Cook-Levin이 SAT을 첫 NP-완전으로 세운 뒤로는 이 환원 체인으로 확장된다.

## 파인만

> [!note]- 백지에 "A ≤ B"의 정의와 두 쓰임(불가능성 전파 / NP-완전성)을 적고, 각각 환원 방향이 왜 그렇게 되는지 설명하라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) A ≤ B가 "B가 더 어렵다"임을 말할 수 있는가, (2) 불가능성 전파는 "어려운 것을 새 문제로" 환원함을 설명할 수 있는가, (3) 하나의 어려움(정지·SAT)을 무한히 재사용하는 아이디어를 짚을 수 있는가.
