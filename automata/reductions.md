# 환원 (Reductions)

## 한 줄 요약

문제 A를 문제 B로 변환해 "B를 풀면 A도 풀린다"를 보이는 기법. 결정 불가능성을 전파하고(A가 불가능하면 B도), NP-완전성을 증명하는(A만큼 B도 어렵다) 핵심 도구.

## 왜 필요한가

- 정지 문제([[decidability]]) 하나로 수많은 문제의 불가능성 증명
- NP-완전성 증명의 메커니즘 → algorithms/[[p-vs-np]]
- "이 문제가 저 문제만큼 어렵다"의 형식화

## 환원의 아이디어

A를 B로 **환원(reduce)**: A의 인스턴스를 B의 인스턴스로 변환하는 함수 f가 존재해:

```
A의 답(x) = B의 답(f(x))
```

즉 "A를 풀려면 f로 변환 후 B를 풀면 됨". 이게 되면:

- **B를 풀 수 있으면 → A도 풀 수 있음** (변환 후 B 사용)
- 대우: **A를 풀 수 없으면 → B도 풀 수 없음** (B가 풀리면 A도 풀릴 테니 모순)

**A ≤ B** 표기: "A가 B로 환원됨" = "B가 A만큼은 어렵다".

## 방향 주의 (핵심)

환원 방향이 결론을 정함:

- **불가능성 전파**: A가 불가능하고 A ≤ B면 → B도 불가능. **어려운 A를 새 문제 B로 환원**
- **가능성 전파**: B가 가능하고 A ≤ B면 → A도 가능

증명하려는 것에 따라 방향이 반대. 헷갈리면 "무엇을 이미 알고, 무엇을 보이려 하나"로 정리.

## 결정 불가능성 증명

정지 문제([[decidability]])가 불가능 → 다른 문제도 불가능을 환원으로:

**예: "프로그램이 특정 입력에 특정 출력을 내나?"** 가 결정 불가:

```
정지 문제를 이 문제로 환원:
  P(x)가 정지하나? 를 판정하려면
  → P를 변형: P(x) 실행 후 "42" 출력하는 P' 만듦
  → "P'(x)가 42를 출력하나?"를 물음
  → 이게 판정되면 정지 문제도 판정됨 (모순, 정지문제 불가)
  → 이 출력 문제도 결정 불가 ∎
```

정지 문제 ≤ 출력 문제. 정지가 불가능하니 출력 문제도 불가능. 이렇게 **하나의 불가능성**(정지)에서 수많은 불가능성이 파생 → Rice 정리([[decidability]])의 일반화.

## NP-완전성 증명

환원의 또 다른 용도 (다항 시간 환원) → algorithms/[[p-vs-np]]:

- **A ≤ₚ B** (다항 시간 환원): A를 다항 시간에 B로 변환
- A가 NP-완전이고 A ≤ₚ B면 → B도 NP-hard
- **NP-완전 증명법**: 알려진 NP-완전 문제(SAT 등)를 새 문제로 환원

```
3-SAT ≤ₚ 클리크 ≤ₚ 정점커버 ≤ₚ ...
Cook-Levin이 SAT을 첫 NP-완전으로 → 이후 전부 환원 체인
```

여기선 방향이 "어려운 문제를 새 문제로" - 새 문제도 최소한 그만큼 어렵다.

## 환원의 종류

| 종류 | 용도 |
|---|---|
| **매핑 환원(many-one)** | 결정 가능성 (인스턴스→인스턴스) |
| **튜링 환원** | 더 일반 (B를 서브루틴으로 여러 번) |
| **다항 시간 환원(≤ₚ)** | NP-완전성 (변환이 다항 시간) |
| **로그 공간 환원** | 더 세밀한 복잡도 클래스 |

변환의 **자원 제한**이 어떤 결론을 얻는지 정함 (계산 가능성 vs 복잡도).

## 왜 강력한가

환원은 **한 번의 어려움을 무한히 재사용**:

- 정지 문제 하나 → 수천 개 결정 불가 문제
- SAT 하나 → 수천 개 NP-완전 문제
- "새 문제가 어렵나?"를 알려진 어려운 문제로의 환원으로 즉시 답
- 문제들 사이의 **상대적 난이도 지도**를 그림

## 연결
