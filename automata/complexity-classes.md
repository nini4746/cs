# 복잡도 클래스 (Complexity Classes)

## 한 줄 요약

결정 가능한 문제들을 필요한 자원(시간·공간)으로 분류한다. P, NP, PSPACE 등의 클래스와 그 관계, NP-완전성의 이론적 기반. 계산 가능성([[decidability]])이 "되나"라면 복잡도는 "얼마나 효율적으로"를 묻는다.

## 왜 필요한가

- 결정 가능한 문제도 "쉬운/어려운"으로 나뉨
- P vs NP의 이론적 틀 → algorithms/[[p-vs-np]]
- 문제의 본질적 난이도 분류

## 계산 가능성 vs 복잡도

- **계산 가능성**([[decidability]]): 알고리즘이 **존재하나** (예/아니오)
- **복잡도**: 존재한다면 **얼마나 많은 자원**(시간/공간)이 필요한가

결정 가능한 문제들 안에서 다시 난이도를 나눔. 튜링 머신([[turing-machines]])의 시간(단계 수)·공간(테이프 칸)으로 측정.

## 시간 복잡도 클래스

| 클래스 | 정의 |
|---|---|
| **P** | 결정적 TM이 **다항 시간**에 푸는 문제 |
| **NP** | 비결정적 TM이 다항 시간 = 답을 다항 시간에 **검증** |
| **EXPTIME** | 지수 시간 |

- **P**: "효율적으로 풀 수 있다"의 표준 (정렬, 최단경로 등)
- **NP**: "답을 빠르게 검증" (SAT, TSP 결정판) → algorithms/[[p-vs-np]]에서 상세
- P ⊆ NP (풀 수 있으면 검증도 가능)

## 공간 복잡도 클래스

| 클래스 | 정의 |
|---|---|
| **L** | 로그 공간 (결정적) |
| **PSPACE** | 다항 **공간** (시간 무제한) |
| **NPSPACE** | 비결정 다항 공간 |

**Savitch 정리**: PSPACE = NPSPACE (공간에선 비결정성이 능력을 거의 안 더함, 제곱만). 시간의 P vs NP와 대조적 - 공간은 결판났는데 시간은 미해결.

## 클래스 관계

알려진 포함 관계:

```
L ⊆ P ⊆ NP ⊆ PSPACE ⊆ EXPTIME
```

- 대부분 **등호인지 진포함인지 미해결** (P vs NP가 대표)
- 확실한 것: **P ⊊ EXPTIME** (시간 계층 정리로 증명됨 - 지수는 다항보다 진짜 강함)
- 그 사이 어디서 갈라지는지는 대부분 모름

## NP-완전과 PSPACE-완전

각 클래스의 "가장 어려운" 문제 (환원 → [[reductions]]):

- **NP-완전**: NP에서 가장 어려움. SAT, TSP, 배낭... 하나 풀면 NP 전부 → algorithms/[[p-vs-np]]
- **PSPACE-완전**: PSPACE에서 가장 어려움. **양화 불리언 식(QBF)**, 많은 게임(체스 일반화, 바둑)의 최적 수 판정
- 게임이 PSPACE-완전인 이유: "내가 이기는 수가 있나 = 모든 상대 응수에 대해 내 응수가 있나" (교대 양화 = 공간 소모)

## 왜 P를 "효율적"으로 보나

- 다항 시간 = 입력 2배에 상수배 증가 (관리 가능)
- 지수 = 입력 조금만 늘어도 폭발 → 실용 불가
- 다항의 차수(n² vs n¹⁰⁰)는 실전 차이나지만, 이론적 경계선은 다항/지수
- 처치-튜링의 복잡도 버전: "합리적 계산 모델은 다항 시간 차이만" (양자 제외 논쟁)

## 다른 클래스들

- **co-NP**: "아니오"를 다항 검증 (NP의 여집합 클래스). NP=co-NP? 미해결
- **BPP**: 확률적 다항 시간 (랜덤 허용, 오류 확률 낮음)
- **BQP**: 양자 다항 시간 (양자 컴퓨터). 인수분해가 여기 (Shor) → security/[[crypto-basics]]
- **#P**: 세는 문제 (해가 몇 개인가)

## 실전 함의

- 문제가 어느 클래스인지 알면 접근법 결정
- NP-hard/PSPACE-hard → 최적 다항 알고리즘 포기, 근사/휴리스틱 → algorithms/[[approximation-and-heuristics]]
- 암호학은 특정 문제의 어려움에 기댐 (인수분해, 이산로그) → security/[[crypto-basics]]

## 연결

- 결정 가능성 (전제) → [[decidability]]
- 튜링 머신 자원 측정 → [[turing-machines]]
- P vs NP 상세 → algorithms/[[p-vs-np]]
- 환원과 완전성 → [[reductions]]
- 공간 복잡도 → [[space-complexity]]
- 양자와 암호 → security/[[crypto-basics]]
