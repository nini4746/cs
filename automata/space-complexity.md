# 공간 복잡도 (Space Complexity)

## 한 줄 요약

시간이 아니라 메모리(테이프 칸) 사용을 기준으로 문제를 분류한다. 공간은 재사용 가능해 시간과 다르게 행동하며, PSPACE와 로그 공간(L)이 핵심 클래스다.

## 왜 필요한가

- 시간만이 자원이 아니다 - 메모리도 제약
- 공간이 시간과 어떻게 다르게 행동하나
- 복잡도 이론의 다른 축

## 공간 = 재사용 가능

시간과 공간의 근본 차이:

- **시간**: 한 번 쓰면 사라짐 (지나간 사이클은 못 돌림)
- **공간**: **재사용 가능** (같은 메모리 칸을 여러 번 덮어씀)

그래서 공간이 시간보다 "귀하지 않음". 다항 공간으로 지수 시간을 쓸 수 있음 (같은 공간을 재활용하며 오래 계산). 이 재사용성이 공간 복잡도를 시간과 다르게 만듦.

## 주요 클래스

| 클래스 | 정의 |
|---|---|
| **L** (LOGSPACE) | 로그 공간 (결정적) - 입력 외 O(log n) 작업 공간 |
| **NL** | 비결정적 로그 공간 |
| **PSPACE** | 다항 공간 (시간 무제한) |

### L (로그 공간)

입력을 읽되(읽기 전용), **작업 메모리는 O(log n)**만:

- log n 비트 = 입력 인덱스 몇 개를 저장할 정도 (포인터 몇 개)
- 예: 입력이 정렬됐나 검사 (인덱스 하나로 순회), 그래프 연결성(무방향, Reingold 정리)
- 매우 제한적이지만 실용적 (스트리밍, 인덱스 기반)

### PSPACE

**다항 공간, 시간은 무제한**:

- 게임 (체스·바둑 일반화의 최적 수), QBF (양화 불리언 식)
- P ⊆ NP ⊆ PSPACE (다항 시간이면 다항 공간, 검증도)
- PSPACE-완전 문제가 "공간의 가장 어려움" → [[complexity-classes]]

## Savitch 정리

**PSPACE = NPSPACE** (비결정 다항 공간 = 결정 다항 공간):

- 더 정확히: NSPACE(f) ⊆ SPACE(f²) - 비결정을 결정으로 바꿀 때 공간은 제곱만 늘어남
- 시간의 P vs NP(지수 격차 의심)와 **극명한 대조**: 공간에선 비결정성이 겨우 제곱만 이득
- 이유: 공간 재사용 덕에 비결정 경로들을 순차적으로 시도 가능 (시간을 희생하고 공간은 재활용)

## 시간-공간 관계

```
L ⊆ NL ⊆ P ⊆ NP ⊆ PSPACE ⊆ EXPTIME
```

- **시간 t → 공간 t 이하**: t 시간에 t칸 넘게 못 씀 (한 단계에 한 칸)
- **공간 s → 시간 2^O(s)**: s 공간의 서로 다른 상태가 2^s개 → 그 이상 돌면 반복(무한루프) → 정지하려면 2^s 시간 내
- 그래서 PSPACE ⊆ EXPTIME (다항 공간 → 지수 시간 상한)

## 왜 공간이 흥미로운가

- **메모리 제약 계산**: 임베디드, 스트리밍(로그 공간에 가까운 sublinear)
- **게임·논리**: PSPACE-완전 문제의 자연스러운 원천
- **이론적 대조**: Savitch로 공간의 비결정성 문제는 (거의) 해결됐는데 시간(P vs NP)은 미해결 - 두 자원이 근본적으로 다르게 행동
- **L vs P**: "로그 공간으로 다항 시간 문제를 다 풀 수 있나" 도 미해결

## 실용 관점

이론이 주로지만 실무 연결:

- 알고리즘의 공간 복잡도도 중요 (인메모리 처리, [[dp-optimization]]의 공간 압축)
- 스트리밍 알고리즘 = sublinear 공간 (전체를 못 담음) → data-structures/[[hyperloglog]], [[bloom-filter]]
- 외부 정렬·DB = 메모리 제약하 계산 → database/[[buffer-pool]]
