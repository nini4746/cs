# 푸시다운 오토마타 (Pushdown Automata)

## 한 줄 요약

유한 오토마타에 스택을 추가한 모델. 스택이 무한 중첩을 기억할 수 있어 문맥 자유 언어를 인식한다. PDA와 CFG는 정확히 같은 언어 클래스를 표현한다.

## 왜 필요한가

- 문맥 자유 언어([[context-free-grammars]])를 인식하는 기계
- 왜 스택이 aⁿbⁿ을 가능케 하나
- 파서가 왜 스택을 쓰나 → compilers/[[parsing]]

## DFA + 스택

DFA([[dfa-nfa]])는 유한 상태만 → 무한히 못 셈([[pumping-lemma]]). PDA는 **스택**을 추가:

```
PDA = 유한 상태 + 무한 스택
전이: (현재 상태, 입력 글자, 스택 top) → (새 상태, 스택 연산)
스택 연산: push / pop / 그대로
```

스택이 **무한 메모리**(단, LIFO 제약) → 중첩 깊이를 기억. 이게 정규를 넘어서는 힘.

## 스택이 aⁿbⁿ을 푸는 법

L = {aⁿbⁿ} ([[pumping-lemma]]에서 정규 아님):

```
1. a를 읽을 때마다 스택에 push (a 개수를 스택에 쌓음)
2. b를 읽을 때마다 스택에서 pop (하나씩 소진)
3. 입력 끝에 스택이 비었으면 수락 (a 개수 == b 개수)
```

스택이 "지금까지 a를 몇 개 봤나"를 기억 → 유한 상태로 못 하던 걸 함. 균형 괄호도 같은 방식 (여는 괄호 push, 닫는 괄호 pop).

## PDA = CFG (동치)

**핵심 정리**: PDA가 인식하는 언어 = CFG가 생성하는 언어 = **문맥 자유 언어(CFL)**.

- CFG → PDA: 문법 규칙을 스택으로 시뮬레이션 (변수를 스택에 놓고 규칙으로 확장)
- PDA → CFG: PDA의 스택 동작을 문법 규칙으로
- [[dfa-nfa]]에서 정규식=DFA였듯, 여기선 CFG=PDA. 문법(생성)과 오토마타(인식)의 동등성

## 비결정성이 여기선 중요하다

DFA=NFA(비결정성 무의미)였지만([[dfa-nfa]]), **PDA는 다름**:

- **비결정적 PDA(NPDA)** = 문맥 자유 언어 전체
- **결정적 PDA(DPDA)** = 결정적 문맥 자유 언어 (**더 작음!**)
- 예: 회문 {w wᴿ}은 NPDA만 (중간 지점을 추측해야 - 비결정)

비결정성이 유한 오토마타엔 능력을 안 줬지만 PDA엔 **더 큰 능력**을 줌. 계층마다 비결정성의 역할이 다름.

## 실용: DPDA와 파싱

프로그래밍 언어 파서는 **결정적 PDA**에 대응:

- LL/LR 파서 = 결정적으로 동작 (한 번 읽고 확정) → compilers/[[parsing]]
- 결정적이라 효율적 O(n) 파싱
- 그래서 언어 문법을 결정적으로 파싱 가능하게 설계 (LL(1), LR(1))
- 파서가 스택을 쓰는 이유 = PDA의 스택 (중첩 구조 추적)

## 스택 하나의 한계

PDA는 스택이 **하나** → 여전히 한계:

- **aⁿbⁿcⁿ 불가**: 스택으로 a-b는 맞추는데 그 다음 c를 또 맞추려면 정보가 이미 소진 → [[cfl-pumping]]
- 스택 **두 개**면 튜링 머신과 동등 (무한 메모리 자유롭게) → [[turing-machines]]
- 스택 하나 = 딱 문맥 자유만큼의 힘

## 촘스키 위계

```
정규 (유한 상태) ⊂ 문맥자유 (유한상태 + 스택 1개) ⊂ ... ⊂ 튜링 (무한 테이프)
```

메모리 모델이 계층을 정함: 없음 → 스택 → 테이프. → [[turing-machines]]

## 셀프 체크

> [!question]- PDA의 전이는 무엇을 입력으로 받고 무엇을 출력하는가?
> (현재 상태, 입력 글자, 스택 top)을 받아 (새 상태, 스택 연산)을 낸다. 스택 연산은 push / pop / 그대로 중 하나다. 유한 상태에 무한 스택(LIFO)을 더한 모델이다.

> [!question]- 스택이 aⁿbⁿ을 인식하게 하는 원리는?
> a를 읽을 때마다 push, b를 읽을 때마다 pop 하고, 입력 끝에 스택이 비었으면 수락한다. 스택이 "지금까지 본 a 개수"를 기억하므로 유한 상태로 못 하던 개수 맞추기를 할 수 있다.

> [!question]- 유한 오토마타와 달리 PDA에서 비결정성이 중요한 이유는?
> DFA=NFA로 비결정성이 능력을 안 줬지만, NPDA(비결정 PDA)는 문맥 자유 언어 전체를 인식하는 반면 DPDA(결정 PDA)는 더 작은 결정적 CFL만 인식한다. 예로 회문 {wwᴿ}은 중간 지점을 추측해야 해서 NPDA만 가능하다.

> [!question]- 스택이 하나뿐이라 PDA가 못 하는 것과, 스택 2개면 무엇이 되는가?
> aⁿbⁿcⁿ 같은 두 개의 독립 개수 관계는 스택 하나가 a-b 맞추며 소진돼 못 한다. 스택 2개면 무한 메모리를 자유롭게 써 튜링 머신과 동등해진다.

## 연습문제

> [!example]- 문제: L = {aⁿbⁿ : n ≥ 0}을 인식하는 PDA를 상태·전이·스택 연산으로 명시하라.
> **풀이**
> 상태 {q0(a 읽는 중), q1(b 읽는 중), qacc}. 스택 바닥 기호 $.
> - 시작: q0, 스택에 $ push. (n=0을 위해 q0도 수락으로 두거나 ε로 qacc)
> - q0에서 a 읽음: A를 push (top 무시). q0 유지.
> - q0에서 b 읽음(top이 A): pop, q1로. (첫 b에서 a→b 전환)
> - q1에서 b 읽음(top이 A): pop, q1 유지.
> - q1에서 입력 끝, top이 $: qacc로(수락). 스택이 A 없이 $만 남음 = a와 b 개수 일치.
> a보다 b가 많으면 A가 먼저 바닥나 pop 실패, 적으면 끝에 A가 남아 $ 아님 → 거절.

> [!example]- 문제: 균형 괄호 언어(여는 `(`와 닫는 `)`가 올바르게 중첩)를 인식하는 PDA를 설계하고, 왜 결정적(DPDA)으로 가능한지 설명하라.
> **풀이**
> 단일 상태 q(=수락), 스택 바닥 $.
> - `(` 읽음: L을 push.
> - `)` 읽음(top이 L): pop.
> - `)` 인데 top이 $(짝 없음): 거절.
> - 입력 끝에 top이 $: 수락(모든 여는 괄호가 닫힘).
> 각 입력 글자마다 top을 보고 행동이 유일하게 정해지므로 추측이 필요 없다 → DPDA. 그래서 실제 파서가 O(n)으로 괄호를 검사할 수 있다.

## 파인만

> [!note]- 백지에 DFA→PDA→TM의 메모리 모델(없음 / 스택 / 테이프)을 표로 그리고, 각 모델이 인식하는 언어 클래스와 왜 그 경계가 생기는지 설명하라. 막히면 그 칸만 다시.
> **점검 포인트**: (1) 스택이 무한하지만 LIFO 제약이 있음을 말할 수 있는가, (2) NPDA≠DPDA인데 NFA=DFA인 대조를 설명할 수 있는가, (3) 스택 1개 vs 2개가 CFL vs 튜링을 가름을 짚을 수 있는가.

## 연결

- CFG와의 동치 → [[context-free-grammars]]
- 유한 오토마타 (스택 없음) → [[dfa-nfa]]
- CFL 한계 → [[cfl-pumping]]
- 스택 2개 = 튜링 → [[turing-machines]]
- 파서의 스택 → compilers/[[parsing]]
- PDA의 무한 스택 = 스택 자료구조(LIFO) → data-structures/[[stacks-and-queues]]
