# 푸시다운 오토마타 (Pushdown Automata)

## 한 줄 요약

유한 오토마타에 스택을 추가한 모델. 스택이 무한 중첩을 기억할 수 있어 문맥 자유 언어를 인식한다. PDA와 CFG는 정확히 같은 언어 클래스를 표현한다.

## 왜 필요한가

- 문맥 자유 언어([[context-free-grammars]])를 인식하는 기계
- 왜 스택이 aⁿbⁿ을 가능케 하나
- 파서가 왜 스택을 쓰나 → compilers/[[parsing]]

## DFA + 스택

DFA([[dfa-nfa]])는 유한 상태만 → 무한히 못 셈([[pumping-lemma]]). PDA는 **스택**을 추가:

```
PDA = 유한 상태 + 무한 스택
전이: (현재 상태, 입력 글자, 스택 top) → (새 상태, 스택 연산)
스택 연산: push / pop / 그대로
```

스택이 **무한 메모리**(단, LIFO 제약) → 중첩 깊이를 기억. 이게 정규를 넘어서는 힘.

## 스택이 aⁿbⁿ을 푸는 법

L = {aⁿbⁿ} ([[pumping-lemma]]에서 정규 아님):

```
1. a를 읽을 때마다 스택에 push (a 개수를 스택에 쌓음)
2. b를 읽을 때마다 스택에서 pop (하나씩 소진)
3. 입력 끝에 스택이 비었으면 수락 (a 개수 == b 개수)
```

스택이 "지금까지 a를 몇 개 봤나"를 기억 → 유한 상태로 못 하던 걸 함. 균형 괄호도 같은 방식 (여는 괄호 push, 닫는 괄호 pop).

## PDA = CFG (동치)

**핵심 정리**: PDA가 인식하는 언어 = CFG가 생성하는 언어 = **문맥 자유 언어(CFL)**.

- CFG → PDA: 문법 규칙을 스택으로 시뮬레이션 (변수를 스택에 놓고 규칙으로 확장)
- PDA → CFG: PDA의 스택 동작을 문법 규칙으로
- [[dfa-nfa]]에서 정규식=DFA였듯, 여기선 CFG=PDA. 문법(생성)과 오토마타(인식)의 동등성

## 비결정성이 여기선 중요하다

DFA=NFA(비결정성 무의미)였지만([[dfa-nfa]]), **PDA는 다름**:

- **비결정적 PDA(NPDA)** = 문맥 자유 언어 전체
- **결정적 PDA(DPDA)** = 결정적 문맥 자유 언어 (**더 작음!**)
- 예: 회문 {w wᴿ}은 NPDA만 (중간 지점을 추측해야 - 비결정)

비결정성이 유한 오토마타엔 능력을 안 줬지만 PDA엔 **더 큰 능력**을 줌. 계층마다 비결정성의 역할이 다름.

## 실용: DPDA와 파싱

프로그래밍 언어 파서는 **결정적 PDA**에 대응:

- LL/LR 파서 = 결정적으로 동작 (한 번 읽고 확정) → compilers/[[parsing]]
- 결정적이라 효율적 O(n) 파싱
- 그래서 언어 문법을 결정적으로 파싱 가능하게 설계 (LL(1), LR(1))
- 파서가 스택을 쓰는 이유 = PDA의 스택 (중첩 구조 추적)

## 스택 하나의 한계

PDA는 스택이 **하나** → 여전히 한계:

- **aⁿbⁿcⁿ 불가**: 스택으로 a-b는 맞추는데 그 다음 c를 또 맞추려면 정보가 이미 소진 → [[cfl-pumping]]
- 스택 **두 개**면 튜링 머신과 동등 (무한 메모리 자유롭게) → [[turing-machines]]
- 스택 하나 = 딱 문맥 자유만큼의 힘

## 촘스키 위계

```
정규 (유한 상태) ⊂ 문맥자유 (유한상태 + 스택 1개) ⊂ ... ⊂ 튜링 (무한 테이프)
```

메모리 모델이 계층을 정함: 없음 → 스택 → 테이프. → [[turing-machines]]

## 연결

- CFG와의 동치 → [[context-free-grammars]]
- 유한 오토마타 (스택 없음) → [[dfa-nfa]]
- CFL 한계 → [[cfl-pumping]]
- 스택 2개 = 튜링 → [[turing-machines]]
- 파서의 스택 → compilers/[[parsing]]
