# SVD 기초 (Singular Value Decomposition)

## 한 줄 요약

임의의 행렬을 세 조각(회전-스케일-회전)으로 분해하는 것. 고유값의 일반화이며, 저차원 근사로 압축·추천·차원 축소·임베딩의 기반이 된다.

## 왜 필요한가

- 고유값([[eigenvalues]])을 정사각 아닌 행렬로 일반화
- 저차원 근사(압축·추천)의 수학
- ai-ml/[[embeddings]]의 기반

## SVD = 임의 행렬 분해

**어떤 행렬 A든** 세 조각으로:

```
A = U Σ Vᵀ
  V (회전) → Σ (스케일) → U (회전)
```

- **U, V**: 직교 행렬 (회전/반사, 방향만 바꿈)
- **Σ**: 대각 행렬, **특이값(singular value)** - 각 축의 스케일 (크기)
- 기하: 어떤 선형 변환([[vectors-and-matrices]])도 "회전 → 축별 스케일 → 회전"으로 분해됨

**고유값의 일반화**([[eigenvalues]]):
- 고유값은 정사각 행렬만, SVD는 **모든 행렬** (직사각도)
- 특이값 = 항상 양수 (고유값은 음수·복소 가능)

## 특이값의 의미

- **특이값 크기 = 그 방향의 중요도** (분산, 정보량)
- 큰 특이값 = 데이터의 주요 방향
- 작은 특이값 = 노이즈·덜 중요한 방향
- 순서대로 정렬 (큰 것부터)

## 저차원 근사 (low-rank approximation)

SVD의 킬러 앱 - **큰 특이값 몇 개만 유지**:

```
A ≈ (큰 특이값 k개만) → rank-k 근사
전체 대신 k개 성분으로 A를 근사 → 압축
```

- **최적 근사**: rank-k 중 오차 최소 (Eckart-Young 정리)
- 데이터의 본질을 소수 성분으로 → 압축·잡음 제거

## CS 응용

### 차원 축소 / PCA

- PCA([[eigenvalues]])가 SVD로 계산됨
- 고차원 데이터 → 주요 방향 몇 개로 (시각화, ML 전처리)

### 추천 시스템

- **행렬 분해**: 사용자-아이템 평점 행렬을 SVD로 분해 → 잠재 요인(latent factor)
- 안 본 항목 평점 예측 (넷플릭스 추천)
- 사용자·아이템을 저차원 벡터로 (임베딩 유사)

### 임베딩

- 단어·항목을 저차원 벡터로 (ai-ml/[[embeddings]])
- 잠재 의미 분석(LSA): 문서-단어 행렬 SVD → 의미 공간
- 현대 임베딩(신경망)의 선형대수 조상

### 이미지 압축

- 이미지 행렬을 rank-k 근사 → 적은 데이터로 근사 (특이값 몇 개)

### 노이즈 제거

- 작은 특이값(노이즈) 버리고 큰 것만 → 신호 복원

## 왜 강력한가

- **보편성**: 모든 행렬에 적용 (정사각 아니어도)
- **최적성**: 저차원 근사가 최적 (Eckart-Young)
- **해석 가능**: 특이값이 중요도 순 → 몇 개 볼지 선택
- 데이터 = 행렬로 보면 SVD가 그 본질 구조 추출

## 셀프 체크

> [!question]- SVD가 고유분해(eigendecomposition)보다 더 일반적인 이유는?
> 고유값·고유분해는 정사각 행렬에만 정의되지만 SVD는 직사각을 포함한 모든 행렬 A에 적용된다. 또한 특이값(singular value)은 항상 0 이상의 실수인 반면 고유값은 음수나 복소수가 될 수 있다. 그래서 A = UΣVᵀ는 어떤 선형 변환이든 회전 → 축별 스케일 → 회전으로 항상 분해할 수 있다.

> [!question]- Σ의 특이값 크기는 무엇을 나타내는가?
> 각 방향(축)의 스케일, 즉 그 방향이 담은 정보량·분산의 크기다. 큰 특이값은 데이터의 주요 방향, 작은 특이값은 노이즈나 덜 중요한 방향에 해당한다. 특이값은 큰 것부터 정렬되므로 앞의 몇 개만 봐도 본질 구조를 잡을 수 있다.

> [!question]- rank-k 근사에서 Eckart-Young 정리가 보장하는 것은?
> 큰 특이값 k개만 남겨 만든 rank-k 행렬이, rank가 k 이하인 모든 행렬 중에서 원본 A와의 오차(Frobenius/스펙트럴 노름)를 최소화하는 최적 근사라는 것이다. 즉 "큰 특이값만 유지"라는 단순한 전략이 수학적으로 최선임을 보장한다. 이것이 압축·잡음 제거·차원 축소가 SVD로 정당화되는 근거다.

> [!question]- 추천 시스템의 행렬 분해에서 SVD가 만들어내는 "잠재 요인"은 무엇인가?
> 사용자-아이템 평점 행렬을 SVD로 분해하면 사용자와 아이템 각각이 저차원 벡터(latent factor)로 표현된다. 이 저차원 공간에서 아직 평가하지 않은 항목의 평점을 예측할 수 있다(넷플릭스식 추천). 임베딩과 같은 원리로, 큰 특이값 방향이 취향의 주요 축을 이룬다.

## 연습문제

> [!example]- 계산하라: A = [[3, 0], [0, -2]]의 특이값과 Σ를 구하라. 고유값과 어떻게 다른지 설명하라.
> **풀이**
> 특이값은 AᵀA의 고유값의 제곱근이다. AᵀA = [[9, 0], [0, 4]]이므로 고유값은 9, 4이고 특이값은 √9 = 3, √4 = 2.
> Σ = diag(3, 2)로 항상 양수, 큰 것부터 정렬한다.
> 반면 A 자체의 고유값은 3과 −2로 음수를 포함한다. 특이값은 크기(스케일)만 담고 부호·방향은 U, V가 담당한다는 점이 대비된다.

> [!example]- 증명하라(스케치): rank-k 근사 A_k = Σ_{i=1}^{k} σᵢ uᵢ vᵢᵀ 가 원본과의 제곱오차를 남은 특이값들로 표현할 수 있음을 보여라.
> **풀이**
> A = Σ_{i=1}^{r} σᵢ uᵢ vᵢᵀ (r = rank)로 쓰면, 앞 k개만 남긴 A_k와의 차이는 A − A_k = Σ_{i=k+1}^{r} σᵢ uᵢ vᵢᵀ.
> uᵢ, vᵢ가 각각 정규직교이므로 Frobenius 노름 제곱은 ‖A − A_k‖_F² = Σ_{i=k+1}^{r} σᵢ².
> 즉 근사 오차는 버린 특이값들의 제곱합이다. 특이값이 급격히 작아지면 k를 작게 잡아도 오차가 작아 압축이 잘 된다. Eckart-Young은 이 A_k가 rank-k 중 이 오차를 최소로 함을 말한다.

> [!example]- 계산하라: 특이값이 [10, 8, 0.5, 0.1]인 행렬을 rank-2로 근사할 때 남는 오차(Frobenius 제곱)와 보존된 "에너지" 비율을 구하라.
> **풀이**
> rank-2 근사는 앞의 10, 8을 유지하고 0.5, 0.1을 버린다.
> 남는 오차(제곱합) = 0.5² + 0.1² = 0.25 + 0.01 = 0.26.
> 전체 에너지 = 10² + 8² + 0.5² + 0.1² = 100 + 64 + 0.25 + 0.01 = 164.26.
> 보존 비율 = (100 + 64)/164.26 = 164/164.26 ≈ 0.9984, 즉 약 99.8%. 성분 2개만으로 정보의 99.8%를 담을 수 있다.

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: 이해했다면 답할 수 있어야 하는 핵심 3가지.
> 1. A = UΣVᵀ의 기하적 의미(회전 → 축별 스케일 → 회전)와 U·V가 직교, Σ가 양의 특이값 대각이라는 것.
> 2. 특이값 = 방향의 중요도(분산)이고, 큰 것 k개만 남기는 rank-k 근사가 Eckart-Young에 의해 최적이라는 것.
> 3. 이 저차원 근사가 PCA·추천(행렬 분해)·임베딩(LSA)·이미지 압축·노이즈 제거로 이어진다는 응용 사슬.
