# 통계 기초 (Statistics Basics)

## 한 줄 요약

데이터로부터 결론을 추론하는 법. 추정(표본→모집단), 신뢰구간, 가설검정이 도구이며, p-value의 정확한 의미를 아는 게 중요하다. A/B 테스트·성능 측정·ML 평가의 기반.

## 왜 필요한가

- 표본으로 전체를 어떻게 추론하나
- p-value가 정확히 뭔가 (흔한 오해)
- A/B 테스트·벤치마크 해석

## 기술 통계 vs 추론 통계

- **기술(descriptive)**: 데이터 요약 (평균, 중앙값, 분산 [[random-variables]])
- **추론(inferential)**: 표본으로 모집단 추론 (이 노트)

**핵심 문제**: 전체(모집단)를 다 못 보고 **표본**만 봄 → 표본에서 전체를 추론 (불확실성 동반).

## 요약 통계

- **평균(mean)**: 산술 평균 - 이상치에 민감
- **중앙값(median)**: 가운데 값 - 이상치에 강건 (algorithms/[[selection]]의 그것). 지연시간은 중앙값·p99가 평균보다 유용
- **분산/표준편차**: 흩어짐 [[random-variables]]
- **백분위수(percentile)**: p50, p95, p99 - 꼬리 (distributed-systems/[[observability-basics]])

## 추정 (estimation)

표본 통계로 모집단 파라미터 추정:

- **점 추정**: 하나의 값 (표본 평균 → 모평균 추정)
- **구간 추정**: 범위 + 신뢰도

### 신뢰구간 (confidence interval)

```
95% 신뢰구간: "이 구간이 참값을 포함한다고 95% 확신"
```

- **정확한 의미**: 같은 방법으로 반복 표본하면 **95%의 구간이 참값 포함** (참값이 이 구간에 있을 확률 95%가 아님 - 미묘!)
- 표본이 클수록 좁아짐 (√n에 반비례, 집중 [[concentration]])
- 중심극한정리([[random-variables]])가 기반

## 가설 검정 (hypothesis testing)

"효과가 있나?"를 통계적으로:

```
귀무가설(H0): 효과 없음 (기본)
대립가설(H1): 효과 있음
→ 데이터가 H0 하에서 얼마나 이상한가? → p-value
```

### p-value의 정확한 의미 (핵심)

**p-value = 귀무가설이 참일 때, 관측된 것만큼(또는 더) 극단적인 결과가 나올 확률**:

- **작은 p (< 0.05)**: "H0가 참이면 이런 결과가 드물다" → H0 기각 (효과 있다고 봄)
- **흔한 오해들** (전부 틀림):
  - ❌ "H0가 참일 확률" (아님 - 반대 조건)
  - ❌ "효과 크기" (아님 - 큰 표본이면 작은 효과도 유의)
  - ❌ "결과가 재현될 확률"
- **정확히**: P(데이터 | H0), H0를 가정한 데이터의 이상함

### 유의성의 함정

- **p-hacking**: 여러 검정 후 유의한 것만 보고 (다중 비교 → 우연히 유의)
- **유의 ≠ 중요**: 큰 표본은 사소한 효과도 유의 (실용적 중요성 별개)
- **효과 크기**도 봐야 (p-value만으로 부족)

## A/B 테스트

CS 실무의 핵심 응용:

```
버전 A vs B → 사용자 나눠 노출 → 지표 차이가 유의한가?
```

- 가설 검정으로 "B가 정말 나은가, 우연인가"
- 표본 크기 계산 (충분히 커야 검출)
- 다중 비교 주의 (여러 지표·변형)
- 웹 최적화(web/[[web-performance]]), 추천, 기능 출시

## CS 응용 (종합)

- **A/B 테스트**: 기능·UX 평가
- **성능 벤치마크**: 여러 번 측정 + 신뢰구간 (한 번 측정은 노이즈 - 이 노트 시리즈의 실측도 여러 번!)
- **ML 평가**: 모델 성능 비교 (ai-ml/[[model-evaluation]])
- **p99 지연**: 백분위수 (distributed-systems/[[observability-basics]])
- **이상 탐지**: 통계적 이상 (모니터링)
