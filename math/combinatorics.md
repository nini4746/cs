# 조합론 (Combinatorics)

## 한 줄 요약

"몇 가지 방법이 있나"를 세는 수학. 순열·조합, 포함-배제, 비둘기집 원리가 도구. 알고리즘 복잡도 분석, 확률 계산, 해시 충돌의 기반.

## 왜 필요한가

- 경우의 수를 어떻게 세나 (복잡도·확률)
- 비둘기집 원리가 왜 강력한가
- 조합론이 CS 어디에

## 기본 계수 원리

- **곱의 법칙**: A가 m가지, B가 n가지 → (A,B)는 m×n가지 (순차 선택)
- **합의 법칙**: A 또는 B (배타적) → m+n가지
- 이게 복잡도 계산의 기본 (중첩 루프 = 곱)

## 순열과 조합

n개에서 k개 선택:

### 순열 (permutation) - 순서 있음

```
P(n,k) = n!/(n-k)!    (순서 구별)
n개 전체 순열 = n!
```

- 정렬의 경우의 수 = n! → 비교 정렬 하한 log(n!) (algorithms/[[comparison-sorts]])
- TSP 완전탐색 = n! (algorithms/[[p-vs-np]])

### 조합 (combination) - 순서 없음

```
C(n,k) = n!/(k!(n-k)!) = "n choose k"    (순서 무시)
```

- 부분집합 개수, "n개 중 k개 고르기"
- 이항 계수 (파스칼 삼각형)
- 예: 팀 구성, 로또

## 포함-배제 (inclusion-exclusion)

합집합의 크기를 정확히 (중복 보정):

```
|A ∪ B| = |A| + |B| − |A ∩ B|      (교집합 한 번 빼기)
|A ∪ B ∪ C| = |A|+|B|+|C| − |A∩B|−|A∩C|−|B∩C| + |A∩B∩C|
```

- 겹치는 걸 더했다 뺐다 (홀수 개 교집합 +, 짝수 −)
- 여러 조건 만족하는 개수 계산
- 확률(math/[[probability-basics]])에도

## 비둘기집 원리 (pigeonhole)

단순하지만 강력 - **n+1개를 n개 상자에 넣으면 어떤 상자에 2개 이상**:

- 자명해 보이지만 강력한 증명 도구
- **CS 응용**:
  - **해시 충돌**(data-structures/[[hash-tables]]): 키가 버킷보다 많으면 충돌 불가피
  - **펌핑 보조정리**(automata/[[pumping-lemma]]): 상태 p개인 DFA에 길이 p 이상 → 상태 반복 (사이클)
  - **압축 불가**: 모든 파일을 압축할 수 없음 (2^n 파일을 2^n 미만으로 못 매핑)
  - **생일 역설**(math/[[probability-basics]]): 23명이면 생일 겹칠 확률 50%

## CS 응용

조합론이 CS 곳곳:

- **복잡도 분석**: 경우의 수 = 시간 (algorithms/[[asymptotic-analysis]])
- **정렬 하한**: log(n!) (algorithms/[[comparison-sorts]])
- **완전탐색**: 순열·조합 열거 (백트래킹)
- **확률 알고리즘**: 경우의 수로 확률 (math/[[probability-basics]])
- **비둘기집**: 충돌·펌핑 (data-structures/[[hash-tables]], automata/[[pumping-lemma]])
- **DP**: 경우의 수 세기 (algorithms/[[dp-patterns]] - 경로 수 등)

## 카탈란 수 등 특수 수열

- **카탈란 수**: 균형 괄호, 이진 트리 모양, 다각형 삼각분할의 개수
- **피보나치**: 재귀 계수 (algorithms/[[dp-fundamentals]])
- 조합론적 구조를 세는 수열들

## 셀프 체크

> [!question]- 순열 P(n,k)와 조합 C(n,k)의 차이는 무엇이고, 둘 사이 관계식은?
> 순열은 순서를 구별하고 조합은 순서를 무시한다. P(n,k)=n!/(n-k)!, C(n,k)=n!/(k!(n-k)!)이며, 같은 k개 선택에 대해 순서를 매기는 방법이 k!가지이므로 P(n,k)=C(n,k)·k! 관계가 성립한다.

> [!question]- 비교 정렬의 하한이 왜 log(n!)인가? 조합론과 어떻게 연결되나?
> n개 원소의 가능한 정렬 결과(순열)가 n!가지이고, 비교 하나가 결과를 두 갈래로 나누므로 결정 트리의 잎이 최소 n!개 필요하다. 따라서 트리 높이(비교 횟수)는 최소 log₂(n!) ≈ n log n이 된다. 경우의 수(순열)가 곧 정보량 하한이 되는 예다.

> [!question]- 비둘기집 원리가 해시 충돌이 불가피함을 어떻게 보장하는가?
> n+1개를 n개 상자에 넣으면 반드시 어떤 상자에 2개 이상이 들어간다. 키의 개수가 버킷 수보다 많으면(키=비둘기, 버킷=상자) 서로 다른 키가 같은 버킷에 매핑될 수밖에 없다. 그래서 해시 테이블은 충돌 처리(체이닝/개방 주소법)가 항상 필요하다.

> [!question]- 포함-배제에서 교집합 항의 부호는 어떻게 정해지는가?
> 교집합에 포함된 집합 개수가 홀수면 더하고(+), 짝수면 뺀다(−). |A∪B|=|A|+|B|−|A∩B|처럼 2개 교집합은 빼고, |A∩B∩C|처럼 3개 교집합은 다시 더한다. 겹쳐서 중복 계산된 부분을 번갈아 보정하는 것이다.

## 연습문제

> [!example]- 계산하라: 8명 중 3명으로 위원회를 만드는 경우의 수와, 그중 회장·부회장·총무를 정하는 경우의 수
> **풀이**
> 위원회(순서 무시)는 조합: C(8,3)=8!/(3!·5!)=(8·7·6)/(3·2·1)=336/6=56가지.
> 직책 배정(순서 구별)은 순열: P(8,3)=8·7·6=336가지.
> 확인: P(8,3)=C(8,3)·3!=56·6=336으로 일치한다.

> [!example]- 계산하라: 1부터 100까지 정수 중 2 또는 3 또는 5로 나누어떨어지는 수의 개수 (포함-배제)
> **풀이**
> |2|=⌊100/2⌋=50, |3|=⌊100/3⌋=33, |5|=⌊100/5⌋=20.
> |2∩3|=⌊100/6⌋=16, |2∩5|=⌊100/10⌋=10, |3∩5|=⌊100/15⌋=6.
> |2∩3∩5|=⌊100/30⌋=3.
> 합=50+33+20−16−10−6+3=74. 따라서 74개다.

> [!example]- 증명하라: 임의의 6명 중에는 서로 아는 3명 또는 서로 모르는 3명이 반드시 존재한다 (비둘기집)
> **풀이**
> 한 사람 A를 고정한다. 나머지 5명 각각과 A는 "안다/모른다" 두 관계 중 하나다. 비둘기집(5명을 2종류로 분류)에 의해 최소 ⌈5/2⌉=3명이 A와 같은 관계다. 그 3명이 A와 모두 "안다"라고 하자. 이 3명 사이에 아는 쌍이 하나라도 있으면 그 둘과 A가 서로 아는 3명이 된다. 없으면 이 3명이 서로 모르는 3명이 된다. "모른다" 쪽도 대칭으로 동일. 따라서 항상 존재한다.
