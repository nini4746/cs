# 콜모고로프 복잡도 (Kolmogorov Complexity)

## 한 줄 요약

한 문자열의 복잡도를 "그걸 출력하는 가장 짧은 프로그램의 길이"로 정의한 것 - 확률 분포가 아니라 **개별 객체**의 정보량. 규칙적이면 짧게 압축되고, 무작위면 자기 자신이 최단(압축 불가). 근본적으로 **계산 불가능**하지만, 압축·학습·무작위성·괴델 불완전성을 잇는 깊은 개념이다.

## 왜 필요한가

- 섀넌 엔트로피는 분포의 정보량. 그럼 **문자열 하나**의 정보량은?
- "무작위"가 뭔지 어떻게 정의하나
- 왜 "학습 = 압축"인가 (MDL의 뿌리)

## 정의

**대상을 출력하는 최단 프로그램의 길이**:

```
K(x) = min { |p| : U(p) = x }
       U = 범용 튜링 기계, p = 프로그램 (automata/[[turing-machines]])
```

- 섀넌 엔트로피([[entropy-and-information]])와 대비:
  - 엔트로피 = **확률 분포**의 평균 정보량 (앙상블)
  - K(x) = **개별 문자열** 하나의 정보량 (알고리즘적)
- 기계 U 선택에 따라 상수만큼 차이 (불변성 정리) → 점근적으로 잘 정의

## 규칙성 = 압축 가능성

```
"010101...01" (10^6개): 짧은 프로그램 "01을 50만 번" → K 작다
무작위 비트열 10^6개:    줄일 규칙 없음 → K ≈ 문자열 길이 (압축 불가)
π의 처음 10^6자리:       π 계산 프로그램 (짧다!) → K 작다 (겉보기 무작위지만)
```

- **K(x)가 작다 = 규칙적 = 압축 가능**
- **K(x) ≈ |x| = 무작위 = 압축 불가** (알고리즘적 무작위성의 정의)
- π는 통계적으론 무작위처럼 보여도 K 작음 → 엔트로피가 못 잡는 구조를 K는 잡음

## 계산 불가능 (핵심 한계)

**K(x)는 알고리즘으로 계산할 수 없다**:

```
증명 스케치 (베리 역설): "50단어 미만으로 기술 못 하는 가장 작은 수"
→ 방금 그걸 15단어로 기술함 (모순)
정식: K 계산 가능하면 "K(x)>n인 최초 x"를 짧은 프로그램으로 찾음 → 모순
```

- automata/[[decidability]]의 정지 문제와 같은 종류의 계산 불가능성
- 최단 프로그램을 찾으려면 모든 프로그램을 돌려봐야 하는데 정지 여부를 모름 ([[turing-machines]])
- **하한 증명도 불가**: "이 문자열은 압축 불가"를 대부분 증명할 수 없음

## 엔트로피와의 관계

계산 불가능하지만 섀넌과 연결됨:

```
분포 p에서 뽑은 x에 대해:  E[K(x)] ≈ H(p) + 상수
```

- 확률적 소스라면 **평균 콜모고로프 복잡도 ≈ 섀넌 엔트로피** ([[entropy-and-information]])
- 두 정보 개념(알고리즘적/확률적)이 평균적으로 만남
- 실용 압축(gzip 등 [[lempel-ziv]])은 K(x)의 **상한 근사** (실제 압축 길이 ≥ K)

## 학습 = 압축 (MDL의 근거)

머신러닝의 깊은 관점:

```
데이터의 규칙을 찾는 것 = 짧게 기술하는 것 = 압축하는 것
좋은 모델 = 데이터를 가장 압축하는 가설 (MDL, [[information-theory-in-ml]])
```

- **일반화의 본질**: 짧은 설명(단순한 모델)이 새 데이터에 잘 맞음 (오컴의 면도날의 형식화)
- 과적합 = 데이터를 규칙 없이 통째 암기 = 압축 실패 (ai-ml/[[ml-fundamentals]])
- "이해 = 압축": 지능을 압축 능력으로 보는 관점 (Solomonoff 귀납, AIXI)

## 응용·연결 개념

- **Solomonoff 귀납**: 모든 가능한 프로그램에 사전확률 → 이상적 예측 (계산 불가, 이론적 이상)
- **정규화 압축 거리(NCD)**: `≈ K(x,y)` 근사로 유사도 측정 → 실제 압축기로 언어·유전자 분류
- **괴델 불완전성 연결**: Chaitin의 Ω(정지 확률)은 압축 불가능한 수 → 수학의 한계
- **최소 기술 길이(MDL)**: K의 계산 가능한 실용 버전 → [[information-theory-in-ml]]

## 왜 중요한가

- **정보의 궁극 정의**: 확률 없이 개별 객체의 정보량 (섀넌을 넘어)
- **무작위성의 형식화**: "압축 불가 = 무작위" (통계적 검정을 넘어선 정의)
- **학습·지능의 이론**: 압축이 곧 이해라는 관점의 수학적 기반
- 계산 불가능이지만 **사고의 틀**로 강력 (실용은 근사로)

## 연결

- 확률적 정보량 (대비·평균 일치) → [[entropy-and-information]]

- MDL·학습=압축 → [[information-theory-in-ml]]
- 계산 불가능성 → automata/[[decidability]], [[turing-machines]]
- 일반화·오컴 → ai-ml/[[ml-fundamentals]]
