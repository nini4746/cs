# 소스 코딩 정리 (Source Coding Theorem)

## 한 줄 요약

무손실 압축의 근본 한계를 못 박는 정리 - 어떤 부호도 평균 길이가 엔트로피 H(X)보다 짧을 수 없고, H(X)+1 비트 안에는 항상 도달할 수 있다. 유일 복호 가능한 부호는 Kraft 부등식 `Σ2^(-lᵢ)≤1`을 만족해야 하고, 최적 길이는 `lᵢ=-log₂p(x)`다.

## 왜 필요한가

- 압축이 얼마나 짧아질 수 있나 (이론적 바닥)
- 왜 엔트로피가 그 바닥인가
- 접두부호와 Kraft 부등식이 뭔가

## 부호화 문제

기호 열을 비트 열로 - **유일 복호 가능**해야:

```
기호 {a,b,c,d} → 비트열, 받는 쪽이 애매함 없이 되돌릴 수 있어야
```

- **접두부호(prefix code)**: 어떤 부호도 다른 부호의 접두사가 아님 → 구분자 없이 즉시 복호
- 예: a=0, b=10, c=110, d=111 (0을 보면 즉시 a, 10이면 b...)
- 접두부호면 유일 복호 가능 (역은 아니지만 접두부호로 최적 달성 가능)

## Kraft 부등식

**접두부호가 존재할 조건**:

```
Σ 2^(-lᵢ) ≤ 1        (lᵢ = i번째 기호의 부호 길이)
```

- 짧은 부호를 쓰면 이진 트리에서 많은 자리를 차지 → 다 합쳐 1 넘으면 불가능
- 등호(=1)면 트리를 꽉 채운 완전 부호
- 이 부등식이 "길이를 마음대로 짧게 못 한다"는 제약

### 코드로 확인

`p={a:0.5, b:0.25, c:0.125, d:0.125}`, 최적 길이 `-log₂p = {1,2,3,3}`:

```
Kraft 합 Σ2^(-l) = 1.0000  (<=1 이어야 접두부호 존재)
엔트로피 H       = 1.7500 bits/기호
평균 부호 길이 L = 1.7500 bits/기호
L >= H ?  1.7500 >= 1.7500  (소스 코딩 정리 하한)
```

- Kraft 합 = 1.0 → 접두부호 존재 ✓
- **L = H = 1.75**: 분포가 2의 거듭제곱이라 `lᵢ=-log₂pᵢ`가 정수로 딱 떨어져 **완벽 도달**

## 소스 코딩 정리 (핵심)

```
H(X) ≤ L* < H(X) + 1
```

- **하한 `L ≥ H`**: 어떤 유일 복호 부호도 엔트로피보다 짧을 수 없음 (Kraft + Gibbs 부등식으로 증명, [[mutual-information]]의 KL≥0)
- **상한 `L < H+1`**: `lᵢ=⌈-log₂pᵢ⌉`(올림)로 부호 만들면 항상 H+1 미만 달성 (Shannon 부호)
- **블록화로 1비트 갭 제거**: n개 기호를 묶어 부호화하면 기호당 갭이 1/n → n→∞에서 **L→H** (정리의 점근 형태)

### 왜 최적 길이가 -log p인가

- 자주 나오는 기호(p 큼)엔 짧은 부호, 드문 기호엔 긴 부호 (모스 부호의 직관)
- `L = Σ pᵢlᵢ`를 Kraft 제약 하에 최소화 → 라그랑주로 `lᵢ=-log₂pᵢ` (자기정보 [[entropy-and-information]])

## 엔트로피 코딩이 이득인 이유

```
고정 2비트 부호 L=2.0 > H=1.7500  (엔트로피 코딩이 0.250bit 절약)
```

- 분포를 무시한 고정 길이 부호는 낭비
- 편향된 분포일수록 엔트로피 코딩([[huffman-and-entropy-coding]]) 이득 큼
- H가 낮을수록(예측 가능할수록) 더 압축 (편향 동전 0.47비트, [[entropy-and-information]])

## 한계와 확장

- **정리는 도달 가능성만**: 실제 부호는 Huffman(기호 단위, 정수 길이 갭)·arithmetic(갭 거의 제거)이 구현 → [[huffman-and-entropy-coding]]
- **분포를 알아야**: 소스 통계를 모르면? → 적응형·범용 압축(LZ, 분포 학습) [[lempel-ziv]]
- **무손실 전용**: 손실 허용 시 더 줄임 → [[rate-distortion]]

## 셀프 체크

> [!question]- Kraft 부등식 `Σ2^(-lᵢ)≤1`은 무엇에 대한 조건인가?
> 주어진 길이 집합 {lᵢ}로 접두부호를 만들 수 있는 필요충분조건이다. 짧은 부호는 이진 트리에서 많은 자리를 차지하므로, 합이 1을 넘으면 겹치지 않게 배치할 수 없다. 등호=1이면 트리를 꽉 채운 완전 부호.

> [!question]- 소스 코딩 정리 `H(X) ≤ L* < H(X)+1`에서 하한과 상한은 각각 어떻게 보장되나?
> 하한 L≥H는 유일 복호 가능한 어떤 부호에도 성립(Kraft+Gibbs). 상한은 `lᵢ=⌈-log₂pᵢ⌉`로 만든 Shannon 부호가 항상 H+1 미만을 달성한다는 구성적 증명. n개를 블록으로 묶으면 기호당 갭이 1/n→0으로 사라져 L→H.

> [!question]- 최적 부호 길이가 `lᵢ = -log₂pᵢ`인 이유는?
> 평균 길이 `L=Σpᵢlᵢ`를 Kraft 제약 하에 최소화하면(라그랑주) `lᵢ=-log₂pᵢ`가 나온다. 즉 자주 나오는 기호엔 짧은 부호, 드문 기호엔 긴 부호로, 자기정보와 정확히 일치한다.

## 연습문제

> [!example]- 문제: `p={a:0.4, b:0.3, c:0.2, d:0.1}`에 대해 Shannon 부호 길이 `lᵢ=⌈-log₂pᵢ⌉`를 구하고 Kraft 합과 평균 길이 L을 계산하라.
> **풀이**
> `-log₂0.4=1.32→l=2`, `-log₂0.3=1.74→l=2`, `-log₂0.2=2.32→l=3`, `-log₂0.1=3.32→l=4`.
> Kraft 합 `= 2⁻²+2⁻²+2⁻³+2⁻⁴ = 0.25+0.25+0.125+0.0625 = 0.6875 ≤ 1` ✓ (접두부호 존재).
> `L = 0.4·2+0.3·2+0.2·3+0.1·4 = 0.8+0.6+0.6+0.4 = 2.4 비트/기호`.
> 참고로 `H≈1.846`이라 `H ≤ L < H+1` 만족.

> [!example]- 문제: 유일 복호 가능한 부호는 `L ≥ H(X)`임을 증명하라.
> **풀이**
> `L - H = Σpᵢlᵢ + Σpᵢlog₂pᵢ = Σpᵢlog₂(pᵢ·2^{lᵢ})`. `qᵢ = 2^{-lᵢ}/Z` (Z=Σ2^{-lⱼ}≤1)로 두면
> `L - H = Σpᵢlog₂(pᵢ/qᵢ) - log₂Z = D(p‖q) - log₂Z`.
> `D(p‖q) ≥ 0`이고 `Z ≤ 1`이라 `-log₂Z ≥ 0`이므로 `L - H ≥ 0`. ∎

## 파인만

> [!note]- 백지에 "왜 엔트로피가 압축의 바닥인가"를 Kraft와 함께 설명하라.
> **점검 포인트**: (1) 접두부호 ↔ Kraft 부등식의 대응, (2) 최적 길이 -log p가 라그랑주 최소화의 결과라는 것, (3) H≤L<H+1의 하한(모든 부호)·상한(Shannon 부호)·블록화로 갭 소멸.
