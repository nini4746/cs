# 오류 정정 부호 (Error-Correcting Codes)

## 한 줄 요약

중복을 계산된 방식으로 추가해 잡음으로 뒤집힌 비트를 **재전송 없이 복원**하는 것. 핵심은 부호어 사이의 **해밍 거리** - 최소 거리 d면 `⌊(d-1)/2⌋`개 오류를 정정한다. Hamming 부호는 패리티 비트의 신드롬으로 단일 오류의 위치를 짚어낸다. 채널 코딩 정리를 실제로 구현하는 방법.

## 왜 필요한가

- 잡음 채널([[channel-capacity]])에서 오류를 실제로 어떻게 정정하나
- 왜 재전송 대신 정정인가
- 해밍 거리와 정정 능력의 관계

## 정정 vs 검출 vs 재전송

```
검출(detection): 오류 있음만 앎 (패리티 1비트, CRC) → 재전송 요청
정정(correction): 오류를 스스로 고침 (재전송 불필요)
```

- **왜 정정(FEC, forward error correction)**: 재전송이 비싸거나 불가능할 때
  - 위성·우주 통신(왕복 지연 큼), 저장매체(SSD/CD - 되읽어도 같은 오류), 방송(수신자 다수)
- 재전송(ARQ)은 network/[[tcp-reliability]], 정정은 물리·링크 계층에서

## 해밍 거리와 정정 능력 (핵심)

```
해밍 거리 d(a,b) = 두 비트열이 다른 자리 수
최소 거리 d_min = 부호어들 사이 최소 거리
```

정정·검출 능력:
```
검출: d_min - 1 개 오류
정정: ⌊(d_min - 1) / 2⌋ 개 오류
```

- 직관: 부호어들을 공간에 **멀리 떨어뜨리면**, 오류로 조금 어긋나도 **가장 가까운 부호어**로 복원
- d_min=3이면 1개 정정 (뒤집혀도 여전히 원본이 최근접), d_min=5면 2개 정정
- 중복을 늘려 거리를 벌림 = 부호율(정보/전체) 희생 ↔ 정정력 (트레이드오프)

## Hamming(7,4) 부호

4 데이터 비트 → 7 비트 (패리티 3개 추가), **단일 오류 정정** (d_min=3):

```
패리티를 겹치게 배치 → 오류 위치가 신드롬(패리티 검사 결과)에 이진수로 나타남
```

### 코드로 확인

```python
# 인코딩: 3개 패리티가 서로 다른 데이터 비트 조합을 커버
# 디코딩: 3개 신드롬 비트 s1,s2,s3 → pos = s1+2·s2+4·s3 = 오류 위치
```

실행:
```
데이터 [1, 0, 1, 1] -> 부호 [0, 1, 1, 0, 0, 1, 1]
수신 (3번 비트 손상): [0, 1, 0, 0, 0, 1, 1]
신드롬이 가리킨 오류 위치: 3
정정 후: [0, 1, 1, 0, 0, 1, 1]  == 원본?  True
복원된 데이터: [1, 0, 1, 1]
```

- 3번 비트가 뒤집혔는데 **신드롬이 정확히 "3"을 가리킴** → 그 자리만 되뒤집어 복원
- 마법의 핵심: 패리티를 겹치게 설계해 **신드롬 자체가 오류 위치의 이진 표현**이 되게 함
- 7비트 중 3비트가 중복 → 부호율 4/7 ≈ 0.57

## 부호의 종류

- **블록 부호**: 고정 크기 블록 단위 (Hamming, Reed-Solomon, BCH)
  - **Reed-Solomon**: 심볼(바이트) 단위, **버스트 오류**에 강함 → QR코드, CD/DVD, RAID
- **합성곱 부호(convolutional)**: 스트림에 상태 기계로 중복 (Viterbi 복호)
- **현대 부호 (용량 근접)**:
  - **LDPC**: 희소 패리티 검사 행렬, 반복 복호 → WiFi, 5G, SSD
  - **터보 부호**: 반복 복호 → 3G/4G
  - **Polar 부호**: 섀넌 용량 도달 증명 → 5G 제어채널
  - 이들이 [[channel-capacity]]의 섀넌 한계에 **실제로 근접** (수십 년의 성과)

## 채널 코딩 정리와의 관계

```
섀넌([[channel-capacity]]): R < C면 오류 0 가능 (존재 증명, 비구성적)
오류정정 부호: 그걸 실제로 구성하는 방법
```

- 섀넌은 "좋은 부호가 존재한다"만 증명 (랜덤 부호화)
- 실용 부호는 **구조**를 넣어 인코딩·복호를 효율적으로 (랜덤 부호는 복호 불가능하게 비쌈)
- LDPC/polar가 "존재하는 좋은 부호"를 구성적으로 실현한 것

## 왜 중요한가

- **어디에나**: SSD/RAM(ECC), QR코드, CD, 위성, 5G, 딥스페이스(보이저), RAID(database/의 내구성)
- **신뢰성의 물리적 기반**: 물리 매체는 항상 잡음 → 부호가 디지털 완전성을 만듦
- security/[[hashing]]의 무결성 검출과 구별: 부호는 **정정**, 해시는 **검출·인증**

## 셀프 체크

> [!question]- 최소 거리 d_min과 검출·정정 능력의 관계는?
> 검출은 `d_min-1`개, 정정은 `⌊(d_min-1)/2⌋`개 오류까지 가능. 부호어들을 공간에서 멀리 떨어뜨릴수록 오류가 나도 가장 가까운 부호어가 여전히 원본이라 복원된다. d_min=3이면 1개 정정, d_min=5면 2개 정정.

> [!question]- Hamming(7,4)에서 신드롬이 어떻게 오류 위치를 바로 짚어내나?
> 3개 패리티 비트가 서로 다른 데이터 비트 조합을 겹치게 커버하도록 배치했기 때문. 수신 후 신드롬 비트 s1,s2,s3를 계산하면 `pos=s1+2·s2+4·s3`가 정확히 뒤집힌 비트 위치(이진수)가 된다. 그 자리만 되뒤집으면 복원.

> [!question]- 섀넌의 채널 코딩 정리와 실제 오류정정 부호는 어떤 관계인가?
> 섀넌은 랜덤 부호화로 "R<C면 좋은 부호가 존재한다"만 비구성적으로 증명했다. 랜덤 부호는 복호 비용이 비현실적이라 실용 부호는 구조를 넣어 인코딩·복호를 효율화한다. LDPC·터보·polar가 그 "존재하는 좋은 부호"를 구성적으로 실현해 섀넌 한계에 근접한다.

## 연습문제

> [!example]- 문제: 부호어 집합 `{000000, 111000, 000111, 111111}`의 최소 거리 d_min과 정정 가능한 오류 수를 구하라.
> **풀이**
> 쌍별 해밍 거리: 000000↔111000=3, 000000↔000111=3, 111000↔000111=6, 000000↔111111=6, 111000↔111111=3, 000111↔111111=3.
> `d_min = 3`. 정정 능력 `⌊(3-1)/2⌋ = 1`개, 검출 능력 `3-1 = 2`개. 부호율은 정보 2비트 / 전체 6비트 = 1/3.

> [!example]- 문제: Hamming(7,4)에서 위치 1~7 중 위치 5의 비트가 뒤집혔다면 신드롬 비트 (s1,s2,s3)는 무엇인가?
> **풀이**
> 신드롬은 오류 위치를 이진수로 인코딩한다: `pos = s1 + 2·s2 + 4·s3`.
> `5 = 1 + 0·2 + 1·4` → `s1=1, s2=0, s3=1`. 복호기는 (1,0,1)→5를 읽어 5번 비트를 되뒤집는다.

> **점검 포인트**: (1) 부호어 사이 거리를 벌리면 최근접 복호로 정정된다는 기하적 직관, (2) d_min → 정정 `⌊(d-1)/2⌋`의 공식, (3) 섀넌의 존재 증명 vs 실용 부호(구조·효율 복호)의 차이.
