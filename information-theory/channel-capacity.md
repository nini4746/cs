# 채널 용량 (Channel Capacity)

## 한 줄 요약

잡음 있는 채널로 신뢰성 있게 보낼 수 있는 최대 전송률. 용량은 입력 분포를 조절해 얻는 최대 상호정보량 `C=max I(X;Y)`. 섀넌의 잡음 채널 코딩 정리 - **전송률이 C보다 낮으면 오류율을 0에 임의로 가깝게** 만들 수 있고, C를 넘으면 불가능하다. 이 놀라운 사실이 모든 통신·저장의 기반.

## 왜 필요한가

- 잡음이 있는데 어떻게 오류 없이 전송하나
- 용량이 왜 max 상호정보량인가
- 소스 코딩([[source-coding-theorem]])과 뭐가 다른가

## 채널 모델

```
송신 X → [잡음 채널 p(y|x)] → 수신 Y
채널 = 조건부 확률 p(y|x) (입력이 출력으로 어떻게 왜곡되나)
```

- **이진 대칭 채널(BSC)**: 각 비트가 확률 p로 뒤집힘 (대표 모델)
- **이진 소거 채널(BEC)**: 확률로 비트가 지워짐(?)
- 수신 Y만 보고 송신 X를 최대한 복원하는 게 목표

## 용량 = max 상호정보량

```
C = max_{p(x)} I(X;Y)      (입력 분포를 최적화한 상호정보 [[mutual-information]])
```

- I(X;Y)는 "Y를 보고 X에 대해 얻는 정보" ([[mutual-information]])
- 입력 분포를 조절해 이 정보를 최대화한 값이 채널의 한계
- 단위: 채널 사용당 비트 (bits/use)

## 이진 대칭 채널 용량

```
C = 1 - H_b(p)      (H_b = 이진 엔트로피, p = 비트 반전 확률)
```

### 코드로 확인

```
오류확률 p=0.0  : H_b=0.0000  C=1.0000 bits/사용
오류확률 p=0.01 : H_b=0.0808  C=0.9192 bits/사용
오류확률 p=0.1  : H_b=0.4690  C=0.5310 bits/사용
오류확률 p=0.25 : H_b=0.8113  C=0.1887 bits/사용
오류확률 p=0.5  : H_b=1.0000  C=0.0000 bits/사용
```

- **p=0 (무잡음)**: C=1 (비트당 1비트 완벽 전송)
- **p=0.5 (완전 잡음)**: C=0 - 수신이 동전 던지기라 **정보 0** (송신과 무관)
- **p=0.1**: C=0.531 - 잡음 때문에 비트당 0.531비트만 신뢰 전송 가능
- 직관: 잡음 H_b(p)만큼 용량이 깎임 (수신의 불확실성 중 잡음 몫)
- 흥미: p=1(항상 반전)도 C=1 (반전을 알면 되돌림) - p=0.5가 최악

## 잡음 채널 코딩 정리 (핵심, 놀라운 결과)

```
전송률 R < C  →  오류율을 임의로 0에 가깝게 만드는 부호 존재
전송률 R > C  →  오류율이 0으로 못 감 (신뢰 전송 불가)
```

- **직관에 반함**: 잡음이 있어도 **오류 0**이 (거의) 가능하다! 단 전송률을 C 아래로
- 방법: 여러 비트를 묶어 **중복(부호)**으로 보냄 → 잡음을 부호로 흡수 ([[error-correcting-codes]])
- 증명은 **랜덤 부호화**(random coding): 무작위 부호가 평균적으로 좋다는 존재 증명 (구성적 아님)
- 대가: C에 가까울수록 블록 길이가 길어짐 (지연↑)

## 소스 코딩 vs 채널 코딩

정보이론의 두 축 - 반대 방향:

```
소스 코딩([[source-coding-theorem]]): 중복 제거 (압축) → H로 줄임
채널 코딩(여기):               중복 추가 (부호) → 잡음 견딤
```

- **소스-채널 분리 정리**: 압축(소스)과 오류정정(채널)을 **따로** 최적화해도 전체 최적 (놀랍게도)
- 그래서 실무는 압축 후 → 오류정정 부호를 얹음 (독립 설계 가능)

## 연속 채널: 섀넌-하틀리

대역폭 있는 아날로그(가우시안) 채널:

```
C = B · log₂(1 + S/N)    (B=대역폭, S/N=신호대잡음비)
```

- 대역폭·전력을 늘리면 용량↑ (WiFi·셀룰러 설계의 기초, network/[[wifi]])
- 로그라 SNR을 2배 해도 용량은 조금만 (수확 체감)

## 왜 중요한가

- **모든 통신·저장의 한계**: 모뎀, WiFi, 5G, SSD, QR코드가 다 이 이론 위
- **현대 부호가 용량에 근접**: LDPC·터보·polar 부호가 섀넌 한계에 거의 도달 ([[error-correcting-codes]])
- 잡음을 두려워 말고 **부호로 이긴다**는 패러다임

## 셀프 체크

> [!question]- 채널 용량이 왜 `max_{p(x)} I(X;Y)`로 정의되나?
> 채널은 p(y|x)로 고정되지만 입력 분포 p(x)는 우리가 고를 수 있다. I(X;Y)는 수신 Y가 송신 X에 대해 주는 정보량이고, 이를 입력 분포로 최대화한 값이 그 채널로 신뢰성 있게 보낼 수 있는 정보의 물리적 한계다.

> [!question]- BSC에서 p=0.5일 때 C=0이고 p=1일 때 C=1인 이유는?
> p=0.5면 출력이 입력과 무관한 동전던지기라 I=0, 용량 0(최악). p=1이면 항상 반전되지만 수신 측이 반전을 알고 되돌리면 완벽 복원되어 C=1. 즉 확정적 반전은 잡음이 아니다.

> [!question]- 잡음 채널 코딩 정리가 "직관에 반한다"는 것은 무엇을 말하나?
> 잡음이 있어도 전송률 R을 용량 C 아래로 두면 오류율을 임의로 0에 가깝게 만드는 부호가 존재한다는 점. 여러 비트를 묶어 중복(부호)으로 보내면 잡음을 흡수할 수 있다. R>C면 신뢰 전송이 불가능하다.

## 연습문제

> [!example]- 문제: 비트 반전 확률 p=0.11인 BSC의 용량을 계산하라 (log₂0.11≈-3.184, log₂0.89≈-0.168).
> **풀이**
> `C = 1 - H_b(p)`, `H_b(p) = -p log₂ p - (1-p) log₂(1-p)`.
> `H_b(0.11) = -0.11·(-3.184) - 0.89·(-0.168) = 0.350 + 0.150 = 0.500`.
> `C = 1 - 0.500 = 0.500 bits/use`. 즉 비트당 약 0.5비트만 신뢰 전송 가능.

> [!example]- 문제: 대역폭 B=1MHz, SNR=30dB인 가우시안 채널의 섀넌-하틀리 용량을 구하라.
> **풀이**
> 먼저 SNR을 배율로: `30dB = 10^(30/10) = 1000`.
> `C = B·log₂(1+S/N) = 10⁶·log₂(1001)`.
> `log₂(1001) ≈ log₂(1024) ≈ 9.97`.
> `C ≈ 10⁶·9.97 ≈ 9.97 Mbps`. dB를 배율로 바꾼 뒤 대입하는 것이 핵심.

## 파인만

> [!note]- 백지에 "잡음이 있는데 오류 0이 가능하다"를 남에게 납득시켜보라.
> **점검 포인트**: (1) C=max I이고 입력 분포를 왜 최적화하는지, (2) BSC 용량 1-H_b(p)에서 p=0.5가 최악인 이유, (3) R<C면 오류→0, R>C면 불가라는 정리의 두 방향과 랜덤 부호화 아이디어.
