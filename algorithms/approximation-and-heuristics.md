# 근사와 휴리스틱 (Approximation and Heuristics)

## 한 줄 요약

NP-hard 문제를 실전에서 다루는 법. 근사 알고리즘은 "최적의 몇 배 이내"를 보장하고, 휴리스틱은 보장은 없지만 실전에서 잘 동작한다. 최적해를 포기하는 대신 실용성을 얻는다.

## 왜 필요한가

- [[p-vs-np]]의 NP-hard 문제를 실제로 어떻게 푸나
- 보장 있는 근사 vs 보장 없는 휴리스틱
- 실무 최적화 문제 접근법

## 근사 알고리즘 (approximation)

**최적해의 일정 비율 이내를 보장**하며 다항 시간에:

- **근사 비율 ρ**: 알고리즘 해 ≤ ρ × 최적 (최소화 문제). ρ=2면 "최적의 2배 이내 보장"
- 최적을 못 구해도 "얼마나 나쁠 수 있나"에 상한

### 예: 정점 커버 2-근사

모든 간선을 덮는 최소 정점 집합 (NP-hard):

```
간선 (u,v)를 아무거나 골라 u와 v 둘 다 커버에 넣음
그 정점들에 붙은 간선 제거, 반복
```

- 이 그리디가 **최적의 2배 이내** 보장 (증명: 고른 간선들은 서로 안 겹쳐 최적도 각각 하나는 포함해야)
- 단순한 그리디인데 보장이 붙음

### 예: TSP 2-근사 (삼각 부등식)

MST를 만들고([[mst]]) DFS 순회 → 최적 투어의 2배 이내. Christofides는 1.5배까지.

### 근사 가능성의 스펙트럼

- **PTAS**: 원하는 만큼(1+ε) 근사 가능 (ε 작을수록 느림)
- **상수 근사**: 정점 커버(2), TSP(1.5)
- **로그 근사**: 집합 커버 O(log n)
- **근사 불가**: 일부 문제는 상수 근사도 NP-hard (PCP 정리)

## 휴리스틱 (heuristics)

**보장은 없지만 실전에서 좋은** 방법. 이론적 최악을 포기하는 대신 실용:

### 지역 탐색 (local search)

현재 해에서 조금씩 개선:
- **hill climbing**: 이웃 해 중 더 나은 것으로 이동. 지역 최적에 갇힘
- **담금질(simulated annealing)**: 가끔 나쁜 쪽도 허용(온도) → 지역 최적 탈출
- **타부 서치**: 최근 방문 금지

### 메타휴리스틱

- **유전 알고리즘**: 해를 "진화" (교배, 변이, 선택)
- **개미 군집**: 페로몬 기반
- 문제 구조 몰라도 적용 가능, 보장 없음

### 문제 특화 휴리스틱

- TSP: nearest neighbor, 2-opt (교차 제거)
- SAT: DPLL/CDCL (실전 SAT 솔버) → [[p-vs-np]]
- 스케줄링: 우선순위 규칙

## 근사 vs 휴리스틱

| | 근사 | 휴리스틱 |
|---|---|---|
| 보장 | ρ배 이내 | 없음 |
| 분석 | 증명 가능 | 경험적 |
| 최악 | 상한 있음 | 나쁠 수 있음 |
| 실전 성능 | 보통 보장보다 좋음 | 종종 매우 좋음 |

**실무 접근**: 보장이 필요하면 근사, 실전 성능만 중요하면 휴리스틱. 둘을 섞기도 (휴리스틱으로 시작해 근사로 검증).

## 정확한 해가 필요할 때

근사도 싫고 정확해야 하면 (작은 입력):

- **분기 한정(branch and bound)**: 탐색 트리를 가지치기
- **정수 계획법(ILP)**: 솔버(Gurobi, CPLEX)에 맡김
- **DP**: 비트마스크 등 → [[dp-optimization]]
- **SAT/SMT 솔버**: 제약을 인코딩

작은 n이나 구조가 좋으면 정확한 해도 가능. n이 크면 근사/휴리스틱.

## 실전 결정 흐름

```
문제가 NP-hard?
  아니오 → 정확한 다항 알고리즘
  예 →
    입력 작음/구조 좋음? → 정확한 해 (분기한정, DP, ILP)
    보장 필요? → 근사 알고리즘
    실전 성능만? → 휴리스틱/메타휴리스틱
```

## 셀프 체크

> [!question]- 근사 비율 ρ=2가 최소화 문제에서 정확히 무엇을 보장하나?
> 알고리즘이 낸 해의 비용이 최적해의 2배를 넘지 않는다는 뜻이다. 최적을 못 구해도 "얼마나 나쁠 수 있나"에 상한을 준다. 정점 커버 그리디, 삼각 부등식 TSP가 대표적인 2-근사다.

> [!question]- 근사 알고리즘과 휴리스틱의 근본적 차이는 무엇인가?
> 근사는 최적 대비 ρ배 이내라는 증명 가능한 보장이 있고, 휴리스틱은 보장이 없다. 근사는 최악에 상한이 있지만 휴리스틱은 나쁠 수 있다. 다만 휴리스틱의 실전 성능이 종종 매우 좋다.

> [!question]- hill climbing이 지역 최적에 갇히는 문제를 담금질(simulated annealing)은 어떻게 푸나?
> hill climbing은 더 나은 이웃으로만 이동해 지역 최적에서 멈춘다. 담금질은 온도 파라미터에 따라 가끔 더 나쁜 쪽으로의 이동도 허용해 지역 최적을 탈출한다.

> [!question]- NP-hard 문제인데 정확한 해가 꼭 필요할 때 어떤 조건이면 가능한가?
> 입력이 작거나 문제 구조가 좋을 때다. 이 경우 분기 한정, 비트마스크 DP, ILP 솔버, SAT/SMT 솔버로 정확한 해를 구할 수 있다. n이 크면 근사/휴리스틱으로 넘어간다.

## 연습문제

> [!example]- 문제: 정점 커버 그리디가 최적의 2배 이내임을 증명하라.
> **풀이**
> 알고리즘이 고른 간선들의 집합을 M이라 하자. M의 간선들은 서로 정점을 공유하지 않는다(고를 때마다 붙은 간선을 전부 제거하므로).
> 임의의 정점 커버는 M의 각 간선을 덮어야 하고, M의 간선끼리 정점이 겹치지 않으므로 서로 다른 정점을 최소 하나씩 써야 한다. 따라서 최적 |OPT| ≥ |M|.
> 알고리즘은 간선마다 정점 2개를 넣으므로 해의 크기 = 2|M| ≤ 2|OPT|. 즉 2-근사다.

> [!example]- 문제: 어떤 최적화 문제를 "이진 탐색으로 푸는 판정 문제"로 바꾸는 상황과, NP-hard일 때 이 접근의 한계를 설명하라.
> **풀이**
> 파라메트릭 서치처럼 "값 k로 조건 만족 가능?"이라는 판정이 단조면 최적값을 이분 탐색할 수 있다.
> 그러나 판정 자체가 NP-hard면(예: TSP에서 "길이 k 이하 투어 존재?") 각 판정이 다항 시간이 아니므로 이 방법으로 다항 시간에 못 푼다. 이때는 근사나 휴리스틱으로 넘어가야 한다.

> [!example]- 문제: 실전 결정 흐름을 의사코드로 재구성하라 - NP-hard 문제를 마주쳤을 때 어떤 순서로 판단하는가.
> **풀이**
> ```
> if 문제가 NP-hard 아님: 정확한 다항 알고리즘 사용
> else:
>   if 입력 작음 or 구조 좋음: 정확한 해 (분기한정 / DP / ILP)
>   elif 보장 필요: 근사 알고리즘 (ρ배 보장)
>   else: 휴리스틱 / 메타휴리스틱
> ```
> 핵심은 "보장이 필요한가 vs 실전 성능만 중요한가"가 근사와 휴리스틱을 가른다는 점이다.
