# 점근 분석 (Asymptotic Analysis)

## 한 줄 요약

입력이 커질 때 알고리즘의 성장률을 표기하는 언어. Big-O는 상한, Θ는 정확한 차수. 상수를 무시해 본질을 보지만, 상수가 실제로 중요해지는 경우도 안다.

## 왜 필요한가

- 알고리즘 성능을 하드웨어 독립적으로 비교
- "O(n log n)이 O(n²)보다 낫다"의 정확한 의미
- 언제 Big-O가 거짓말을 하나 (상수, 작은 n)

## 왜 상수를 무시하나

`3n² + 5n + 100`에서 n이 커지면 `n²`이 지배. 상수(3)와 저차항(5n+100)은 큰 n에서 무의미:

- n=1000: n² = 100만, 5n = 5000 → n²이 200배
- 하드웨어·언어마다 상수는 다르지만 **성장률**은 알고리즘 고유
- → 성장률만 보면 본질 비교 가능

## 세 표기

| 표기 | 의미 | 비유 |
|---|---|---|
| **O(f)** | 상한 (이보다 빠르거나 같음) | ≤ |
| **Ω(f)** | 하한 (이보다 느리거나 같음) | ≥ |
| **Θ(f)** | 정확한 차수 (상한=하한) | = |

- **O**: "최악이 이보다 나쁘지 않다" - 가장 많이 씀
- **Θ**: "정확히 이 차수" - 더 강한 주장
- 흔한 오용: "평균 O(n)"처럼 O를 "정확히"로 씀. 엄밀히는 Θ지만 관례상 O로 씀

### 형식 정의

`f(n) = O(g(n))` ⟺ 어떤 상수 c, n₀가 존재해 모든 n ≥ n₀에서 `f(n) ≤ c·g(n)`.

즉 충분히 큰 n에서 g가 f의 상한 (상수배 허용).

## 성장률 계층

느린 것부터 (좋은 것부터):

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

| 복잡도 | 예 | n=10⁶에서 |
|---|---|---|
| O(1) | 배열 인덱싱 | 1 |
| O(log n) | 이진 탐색 | ~20 |
| O(n) | 선형 순회 | 10⁶ |
| O(n log n) | 병합 정렬 | ~2×10⁷ |
| O(n²) | 중첩 루프 | 10¹² (하루+) |
| O(2ⁿ) | 부분집합 완전탐색 | 우주 나이 초과 |

**O(n²)와 O(n log n)의 차이가 실전을 가름**: n=100만에서 5만 배. 이게 알고리즘 선택이 중요한 이유.

## 최악/평균/최선

같은 알고리즘도 입력에 따라 다름:

- **최악(worst)**: 보장. 가장 많이 분석 (O로)
- **평균(average)**: 전형적 입력. 확률 필요 → math/[[probability-basics]]
- **최선(best)**: 보통 무의미

예: 퀵정렬 최악 O(n²), 평균 O(n log n). 해시 테이블 평균 O(1), 최악 O(n) ([[hash-tables]]).

## Big-O가 거짓말할 때

점근은 "충분히 큰 n"의 이야기. 실전 함정:

1. **상수가 큼**: O(n) 알고리즘의 상수가 O(n log n)보다 크면 작은 n에서 역전. 병합 정렬이 작은 배열에서 삽입 정렬로 바꾸는 이유 ([[comparison-sorts]])
2. **작은 n**: n=10이면 O(n²)이 O(n log n)보다 빠를 수 있음
3. **캐시 무시**: Big-O는 모든 연산을 같은 비용으로 봄. 실제론 캐시 미스가 100배 ([[memory-hierarchy]]). 배열이 링크드 리스트보다 빠른 이유를 Big-O는 못 봄 ([[linked-lists]] 217배 실측)
4. **피보나치 힙**: 이론 최적인데 상수 커서 실무 안 씀 ([[heaps]])

**교훈**: Big-O는 필수 첫 필터지만 최종 답이 아님. 실측이 최종.

## 셀프 체크

> [!question]- 큰 n에서 상수와 저차항을 무시해도 되는 이유는 무엇인가?
> `3n² + 5n + 100`에서 n이 커지면 n² 항이 지배하고 나머지는 무의미해진다(n=1000이면 n²이 5n의 200배). 상수는 하드웨어/언어마다 다르지만 성장률은 알고리즘 고유이므로, 성장률만 봐야 본질을 하드웨어 독립적으로 비교할 수 있다.

> [!question]- O, Ω, Θ의 차이는 무엇이고 Θ가 더 강한 주장인 이유는?
> O는 상한(≤), Ω는 하한(≥), Θ는 상한=하한(=)이다. Θ는 상한과 하한을 동시에 주장하므로 "정확히 이 차수"라는 더 강한 진술이다. 흔히 관례상 Θ여야 할 곳도 O로 쓴다.

> [!question]- `f(n) = O(g(n))`의 형식 정의를 말하고 그 의미를 설명하라.
> 어떤 상수 c, n₀가 존재해 모든 n ≥ n₀에서 f(n) ≤ c·g(n)이 성립한다는 뜻이다. 즉 충분히 큰 n에서 g가 상수배 허용하에 f의 상한이 된다.

> [!question]- Big-O가 실전에서 "거짓말"하는 대표적 경우 몇 가지를 들어라.
> (1) 상수가 크면 작은 n에서 상위 차수 알고리즘에 역전당한다(병합→삽입 전환), (2) n이 작으면 O(n²)이 O(n log n)보다 빠를 수 있다, (3) 캐시 미스를 무시해 배열이 링크드 리스트보다 빠른 것을 못 본다, (4) 피보나치 힙처럼 이론 최적이나 상수가 커서 실무에서 안 쓰는 경우.

> [!question]- 최악/평균/최선 중 왜 최악을 가장 많이 분석하나?
> 최악은 어떤 입력에서도 성능을 보장하기 때문이다. 평균은 확률 분포가 필요하고 최선은 보통 무의미하다. 예로 퀵정렬은 최악 O(n²)이지만 평균 O(n log n)이다.

## 연습문제

> [!example]- 문제: `f(n) = 3n² + 5n + 100`이 O(n²)임을 형식 정의로 증명하라.
> **풀이**
> c와 n₀를 찾으면 된다. n ≥ 1에서 5n ≤ 5n², 100 ≤ 100n²이므로
> 3n² + 5n + 100 ≤ 3n² + 5n² + 100n² = 108n².
> 따라서 c = 108, n₀ = 1을 잡으면 모든 n ≥ 1에서 f(n) ≤ 108n². 즉 f(n) = O(n²). ∎

> [!example]- 문제: O(n²)와 O(n log n)이 n=100만에서 실전을 얼마나 가르는지 구체적으로 계산하라.
> **풀이**
> n = 10⁶에서 n² = 10¹², n log n ≈ 10⁶ × 20 = 2×10⁷.
> 비율 = 10¹² / (2×10⁷) = 5×10⁴, 즉 약 5만 배 차이다.
> n log n 알고리즘이 순간에 끝날 일을 n² 알고리즘은 하루 넘게 돌린다. 이것이 알고리즘 선택이 중요한 이유다.

> [!example]- 문제: O(n) 알고리즘이 O(n log n) 알고리즘보다 실제로 느려지는 상황을 하나 설계하라.
> **풀이**
> O(n) 알고리즘의 숨은 상수가 매우 크다고 하자: 비용 ≈ 1000·n. O(n log n)의 상수는 1이라 하면 비용 ≈ n·log₂n.
> 두 비용이 같아지는 지점: 1000n = n·log₂n → log₂n = 1000 → n = 2¹⁰⁰⁰.
> 즉 현실적 크기 전 범위에서 O(n)이 느리다. 병합 정렬이 작은 배열에서 삽입 정렬로 바꾸는 것이 같은 원리다.
