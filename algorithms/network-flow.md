# 네트워크 유량 (Network Flow)

## 한 줄 요약

소스에서 싱크로 보낼 수 있는 최대 유량을 구하는 문제. max-flow = min-cut이라는 아름다운 쌍대성이 있고, 이분 매칭·할당 등 겉보기 다른 문제들이 이걸로 환원된다.

## 왜 필요한가

- 매칭·할당·분리 문제를 통합하는 강력한 틀
- max-flow min-cut 정리 (조합 최적화의 보석)
- 겉보기 무관한 문제를 유량으로 모델링

## 문제 정의

방향 그래프, 각 간선에 **용량(capacity)**. 소스 s에서 싱크 t로:

- 각 간선의 유량 ≤ 용량
- 각 정점(s,t 제외)에서 들어온 유량 = 나간 유량 (보존)
- **s에서 t로 보낼 수 있는 최대 총 유량**?

비유: 파이프망에서 물을 최대한 보내기. 각 파이프 굵기(용량) 제한.

## Ford-Fulkerson / Edmonds-Karp

**증가 경로(augmenting path)를 반복해서 찾아** 유량을 늘림:

```
1. 잔여 그래프(residual)에서 s→t 경로 찾기
2. 그 경로의 최소 잔여 용량만큼 유량 증가
3. 잔여 그래프 갱신 (역방향 간선 추가 - 되돌리기 허용)
4. 증가 경로 없을 때까지 → 최대 유량
```

- **잔여 그래프**: 남은 용량 + 역방향(취소 가능). 잘못된 초기 선택을 되돌릴 수 있게
- **Edmonds-Karp**: 증가 경로를 BFS로 찾음 → O(VE²)
- Dinic: 더 빠름 O(V²E)

## max-flow min-cut 정리

**핵심 정리**: 최대 유량 = 최소 컷.

- **컷(cut)**: s쪽과 t쪽으로 정점을 나눔. 컷의 용량 = s쪽→t쪽 간선 용량 합
- **최소 컷**: 가장 작은 용량의 컷 = "병목"
- 정리: **max-flow = min-cut**. 보낼 수 있는 최대 = 끊어야 하는 최소

직관: 병목(최소 컷)이 흐름을 제한. 물을 막으려면 최소 컷의 파이프를 끊으면 됨. 이 쌍대성이 유량 이론의 핵심 아름다움.

## 겉보기 다른 문제들의 환원

유량의 힘 = **많은 문제가 유량으로 변환**됨:

### 이분 매칭 (bipartite matching)

두 그룹(예: 지원자-일자리) 사이 최대 매칭:

```
소스 → 왼쪽 정점들 (용량 1)
왼쪽 → 오른쪽 (가능한 매칭, 용량 1)
오른쪽 → 싱크 (용량 1)
최대 유량 = 최대 매칭
```

- 용량 1이 "각자 하나만" 강제 → 최대 유량이 최대 매칭 수
- 작업 할당, 결혼 문제, 스케줄링

### 그 밖

- **최소 컷 = 이미지 분할** (전경/배경 분리)
- **프로젝트 선택** (이익 최대화)
- **간선/정점 분리도** (연결성)
- **최소 경로 커버**

"이 문제를 유량으로 모델링할 수 있나?"가 강력한 문제 해결 도구.

## König 정리 (곁가지)

이분 그래프에서 **최대 매칭 = 최소 정점 커버**. max-flow min-cut의 이분 그래프 버전. 조합론의 여러 min-max 정리가 유량 쌍대성의 그림자.

## 복잡도

| 알고리즘 | 복잡도 |
|---|---|
| Edmonds-Karp | O(VE²) |
| Dinic | O(V²E) |
| 이분 매칭 (Hopcroft-Karp) | O(E√V) |

실전 그래프에선 대개 이론 최악보다 훨씬 빠름.
