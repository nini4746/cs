# 그리디 (Greedy)

## 한 줄 요약

매 단계 지역 최적을 선택해 전역 최적에 도달하는 방법. 빠르고 단순하지만, 그리디 선택이 최적임을 증명해야 한다. 되는 문제(활동 선택, 허프만)와 안 되는 문제(0/1 배낭)를 구분하는 게 핵심.

## 왜 필요한가

- DP보다 빠른 대안 (되는 경우)
- 왜 어떤 그리디는 맞고 어떤 건 틀리나
- 증명 없이 그리디를 믿으면 위험

## 그리디의 아이디어

매 단계 **당장 가장 좋아 보이는 선택**을 하고 되돌아보지 않음:

- DP처럼 모든 선택을 고려하지 않음 → 빠름 (보통 O(n log n))
- 대신 그 지역 선택이 전역 최적으로 이어진다는 **보장이 필요**

## 성립 조건 두 가지

1. **그리디 선택 속성(greedy choice property)**: 지역 최적 선택이 전역 최적해의 일부. 즉 첫 그리디 선택을 포함하는 최적해가 존재
2. **최적 부분 구조**: 선택 후 남은 부분 문제의 최적해 + 그리디 선택 = 전체 최적

이 둘을 **증명**해야 그리디가 맞음. 증명 없으면 그럴듯해 보여도 틀릴 수 있음.

## 증명: 교환 논증

그리디가 최적임을 보이는 표준 ([[correctness-proofs]]):

- 최적해 O가 그리디 해 G와 다르다고 가정
- O의 한 선택을 그리디 선택으로 **교환**해도 여전히 최적(또는 더 나음)임을 보임
- 반복하면 O를 G로 바꿀 수 있음 → G도 최적

## 되는 예시

### 활동 선택 (activity selection)

겹치지 않게 최대 개수의 활동 선택. **끝나는 시간이 빠른 것부터** 그리디:

- 가장 빨리 끝나는 활동을 선택하면 남은 시간이 최대 → 이후 선택지 최대
- 교환 논증으로 최적 증명됨
- O(n log n) (정렬)

### 허프만 코딩 (Huffman)

빈도 기반 최적 접두사 코드. **빈도 최소인 둘을 반복 병합** ([[heaps]] 우선순위 큐):

- 빈도 낮은 문자에 긴 코드 → 전체 길이 최소
- 그리디 선택(최소 둘 병합)이 최적 증명됨
- 데이터 압축(gzip 등)의 기반

### 그 밖

- 다익스트라(가장 가까운 정점 먼저) → [[shortest-paths]]
- 크루스칼/프림 MST → [[mst]]
- 동전 교환 (**표준 동전계에서만** - 아래 주의)

## 안 되는 예시 (중요)

### 0/1 배낭

"가치/무게 비율 높은 것부터"는 **최적 아님**:

```
용량 10, 물건: (무게6,가치6), (무게5,가치5), (무게5,가치5)
그리디(비율): 첫 물건(6,6) → 남은 4 → 못 넣음 → 가치 6
최적: 뒤 두 물건(5,5)+(5,5) → 가치 10
```

지역 최적(비율)이 전역을 망침 → **DP 필요** ([[dp-patterns]]).

### 동전 교환

동전 {1,3,4}로 6 만들기:
- 그리디(큰 것부터): 4+1+1 = 3개
- 최적: 3+3 = 2개

표준 동전계(1,5,10,25)에선 그리디가 맞지만 임의 동전계에선 틀림 → DP.

**교훈**: 그리디가 그럴듯해도 반례가 있으면 틀림. 증명하거나 반례를 찾아라.

## 그리디 vs DP

| | 그리디 | DP |
|---|---|---|
| 선택 | 하나 (되돌림 없음) | 모든 선택 고려 |
| 속도 | 빠름 O(n log n) | 느림 (상태×전이) |
| 최적성 | 증명 필요 (조건부) | 항상 (조건 맞으면) |
| 실패 시 | 틀린 답 | - |

**전략**: 그리디가 될 것 같으면 증명 시도 → 되면 그리디(빠름), 반례 나오면 DP. 확신 없으면 DP가 안전.
