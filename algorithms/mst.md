# 최소 신장 트리 (Minimum Spanning Tree)

## 한 줄 요약

가중 무방향 그래프의 모든 정점을 잇는 최소 비용 트리. 크루스칼(간선 정렬 + union-find)과 프림(정점 확장 + 힙) 둘 다 그리디이며 cut property로 정당성이 증명된다.

## 왜 필요한가

- 최소 비용으로 전체 연결 (네트워크 설계, 회로, 도로)
- 두 그리디 알고리즘이 왜 최적인가
- union-find([[union-find]])와 힙([[heaps]])의 응용

## 문제

가중 무방향 그래프에서 **모든 정점을 연결**하면서 **간선 가중치 합 최소**인 트리(신장 트리):

- 트리 = 사이클 없이 V개 정점을 V-1개 간선으로 연결
- 여러 신장 트리 중 최소 비용
- 용도: 최소 비용 네트워크(케이블, 파이프), 클러스터링, 근사 알고리즘 부품

## 크루스칼 (Kruskal)

**간선을 가중치 순으로** 보며 사이클 안 생기면 추가:

```
1. 모든 간선을 가중치 오름차순 정렬
2. 각 간선을 순서대로: 두 끝이 다른 컴포넌트면 추가 (union), 같으면 스킵
3. V-1개 간선 모을 때까지
```

- **사이클 판정 = union-find** ([[union-find]]): 두 끝이 같은 그룹이면 사이클
- 복잡도: **O(E log E)** (정렬이 지배)
- 간선 중심. 희소 그래프에 적합

## 프림 (Prim)

**한 정점에서 시작해 트리를 키움**. 트리에 붙는 최소 간선을 반복 선택:

```
1. 임의 정점에서 시작
2. 트리와 연결된 간선 중 최소 가중치를 우선순위 큐에서 꺼냄
3. 그 간선이 새 정점을 연결하면 트리에 추가
4. 모든 정점 포함까지
```

- **우선순위 큐(힙)** 사용 → **O(E log V)** ([[heaps]])
- 정점 중심. 다익스트라([[shortest-paths]])와 구조 유사 (거리 대신 간선 가중치)
- 밀집 그래프에 유리

## cut property - 정당성

두 그리디가 최적인 이유:

**cut property**: 그래프를 두 집합으로 나누는 임의의 컷에서, **컷을 가로지르는 최소 가중치 간선은 어떤 MST에 반드시 포함**된다.

- 크루스칼: 각 간선 추가 = 두 컴포넌트를 잇는 최소 간선 (컷의 최소) → cut property로 안전
- 프림: 트리와 나머지의 컷에서 최소 간선 선택 → cut property로 안전
- 교환 논증([[correctness-proofs]])으로 증명. 그리디가 MST에 통하는 근거

## 크루스칼 vs 프림

| | 크루스칼 | 프림 |
|---|---|---|
| 중심 | 간선 | 정점 |
| 자료구조 | union-find | 우선순위 큐 |
| 복잡도 | O(E log E) | O(E log V) |
| 적합 | 희소 그래프 | 밀집 그래프 |
| 정렬 | 전체 간선 | 없음 (힙) |

## 어디에 쓰나

- **네트워크 설계**: 최소 케이블/파이프로 전체 연결
- **클러스터링**: MST에서 가장 비싼 간선 제거 → 클러스터 분할 (single-linkage)
- **근사 알고리즘**: TSP 근사(MST의 2배 이내), 슈타이너 트리
- **이미지 분할**: 픽셀 그래프 MST

## 셀프 체크

> [!question]- cut property는 무엇이고 왜 두 그리디의 정당성을 보장하나?
> cut property는 그래프를 두 집합으로 나누는 임의의 컷에서 그 컷을 가로지르는 최소 가중치 간선이 어떤 MST에 반드시 포함된다는 성질이다. 크루스칼의 각 간선 추가와 프림의 최소 간선 선택이 모두 어떤 컷의 최소 간선을 고르는 것이므로, cut property로 각 선택이 안전함이 보장된다.

> [!question]- 크루스칼과 프림의 복잡도가 각각 O(E log E), O(E log V)인 근거는?
> 크루스칼은 모든 간선을 가중치로 정렬하는 비용이 지배하여 O(E log E)다. 프림은 우선순위 큐(힙)로 트리에 붙는 최소 간선을 꺼내며, 힙 연산이 O(log V)이고 간선마다 갱신하므로 O(E log V)다.

> [!question]- 크루스칼에서 사이클 판정을 union-find로 하는 이유는?
> 간선을 추가할 때 두 끝점이 이미 같은 컴포넌트에 있으면 사이클이 생긴다. union-find는 두 원소가 같은 그룹인지(find) 빠르게 판정하고 병합(union)하므로, 각 간선마다 사이클 여부를 거의 상수 시간에 확인할 수 있다.

> [!question]- 프림이 다익스트라와 구조가 유사한 이유와 결정적 차이는?
> 둘 다 한 정점에서 시작해 우선순위 큐로 최소 키 정점을 확장하는 구조다. 차이는 키의 정의로, 다익스트라는 소스로부터의 누적 거리를 쓰고 프림은 트리에 붙는 간선의 가중치 자체를 쓴다.

## 연습문제

> [!example]- 문제: cut property를 교환 논증으로 증명하라.
> **풀이**
> 어떤 컷을 가로지르는 최소 가중치 간선을 e라 하자. e를 포함하지 않는 MST T가 있다고 가정한다.
> T에 e를 추가하면 사이클이 생기고, 그 사이클에는 컷을 가로지르는 또 다른 간선 e'가 존재한다.
> e는 컷의 최소 간선이므로 weight(e) ≤ weight(e').
> T에서 e'를 빼고 e를 넣으면 여전히 신장 트리이며 총 가중치가 늘지 않는다. 따라서 e를 포함하는 MST가 존재한다.

> [!example]- 문제: 크루스칼의 의사코드를 union-find를 써서 작성하고 복잡도를 분석하라.
> **풀이**
> ```
> 간선을 가중치 오름차순 정렬          # O(E log E)
> for v in V: makeset(v)              # O(V)
> T = {}
> for (u,v,w) in 정렬된 간선:
>   if find(u) != find(v):           # 다른 컴포넌트
>     T에 (u,v) 추가
>     union(u,v)
>   if |T| == V-1: break
> return T
> ```
> 정렬 O(E log E)가 지배한다. union-find 연산은 경로 압축+랭크로 거의 상수(역아커만)이므로 전체 반복은 O(E α(V))다. 따라서 O(E log E).

> [!example]- 문제: 간선 가중치가 모두 서로 다르면 MST가 유일함을 논증하라.
> **풀이**
> MST가 두 개(T1, T2) 존재한다고 가정한다. T1에는 있고 T2에는 없는 간선 중 가중치 최소인 것을 e라 하자.
> T2에 e를 추가하면 사이클이 생기고, 그 사이클에는 T1에 없는 간선 e'가 있다. 가중치가 모두 다르므로 weight(e) ≠ weight(e').
> e가 그런 간선 중 최소였으므로 weight(e) < weight(e'). T2에서 e'를 e로 교환하면 더 가벼운 신장 트리가 되어 T2가 MST라는 가정에 모순이다. 따라서 MST는 유일하다.
