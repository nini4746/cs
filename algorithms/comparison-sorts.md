# 비교 정렬 (Comparison Sorts)

## 한 줄 요약

원소를 비교해서 정렬하는 알고리즘. merge/quick/heap이 O(n log n)이고, 비교만으로는 Ω(n log n)보다 빠를 수 없다는 하한이 증명된다. 실무 정렬은 이들을 조합한다.

## 왜 필요한가

- 가장 많이 쓰는 알고리즘. 다른 것의 부품
- O(n log n)과 O(n²)의 실전 차이
- 왜 비교 정렬은 O(n log n)이 한계인가

## O(n log n) 삼총사

| 정렬 | 방식 | 최악 | 평균 | 공간 | 안정성 |
|---|---|---|---|---|---|
| **병합(merge)** | 분할 후 병합 | O(n log n) | O(n log n) | O(n) | 안정 |
| **퀵(quick)** | 피벗 분할 | **O(n²)** | O(n log n) | O(log n) | 불안정 |
| **힙(heap)** | 힙 추출 | O(n log n) | O(n log n) | O(1) | 불안정 |

### 병합 정렬

절반씩 나눠 각각 정렬 후 병합. T(n)=2T(n/2)+O(n) → O(n log n) ([[recurrences]]):
- **안정적**, 최악도 O(n log n) 보장
- O(n) 추가 공간 필요 (병합 버퍼)
- 외부 정렬(디스크), 연결 리스트 정렬에 적합

### 퀵 정렬

피벗을 골라 작은 것/큰 것으로 분할, 재귀:
- **평균 O(n log n)에 상수가 작아 실전 최고속** (제자리, 캐시 친화 → [[memory-hierarchy]])
- **최악 O(n²)**: 피벗이 계속 최악(정렬된 입력 + 끝 피벗). 랜덤 피벗/median-of-3로 회피
- 불안정

### 힙 정렬

힙 만들고 최댓값 반복 추출 → [[heaps]]:
- 최악도 O(n log n) 보장, 제자리 O(1) 공간
- 캐시 지역성 나빠 퀵보다 느린 편

## 하한: Ω(n log n) 증명

**비교만으로는 O(n log n)보다 빠를 수 없다.** 결정 트리(decision tree) 논증:

- n개 원소의 가능한 순서 = n! 가지
- 각 비교는 이진 결정 (≤ 또는 >) → 결정 트리의 한 분기
- n!개 잎을 구분하려면 트리 높이 ≥ log₂(n!)
- log₂(n!) = Θ(n log n) (스털링 근사)
- 트리 높이 = 최악 비교 횟수 → **최소 Ω(n log n)**

그래서 어떤 비교 정렬도 이 벽을 못 넘음. 넘으려면 비교를 안 해야 함 → [[linear-sorts]].

## 실측: O(n log n) vs O(n²)

```
qsort   2000000 elems: 0.155s
insort   40000 elems: 0.242s  (n²)
insort을 2000000개로 외삽: ~604초
```

퀵정렬은 200만 개를 0.155초. 삽입 정렬(O(n²))은 4만 개에 0.242초 - 200만 개로 외삽하면 **~600초(10분)**. 4000배 차이. [[asymptotic-analysis]]의 성장률 격차가 실전에서 이렇게 나타남.

## 그런데 작은 n에선 삽입 정렬이 빠르다

O(n²)인데도 삽입 정렬은 **작은 배열(~수십 개)에서 O(n log n)보다 빠름**:
- 상수가 작음 (비교/이동만, 재귀 오버헤드 없음)
- 캐시 친화, 거의 정렬된 데이터에 O(n)

그래서 실무 정렬은 **하이브리드**:
- **introsort** (C++ std::sort): 퀵 → 깊이 깊어지면 힙(최악 O(n²) 회피) → 작은 조각은 삽입
- **Timsort** (Python, Java): 병합 + 삽입, 이미 정렬된 런(run) 활용 → 실데이터에 빠름, 안정적

[[asymptotic-analysis]]의 "상수가 중요해지는 경우"의 대표.

## 안정성 (stability)

같은 키의 원소가 **원래 순서를 유지**하나:
- 안정: 병합, 삽입, Timsort
- 불안정: 퀵, 힙
- 다중 키 정렬(이름순 후 나이순)에 중요. DB `ORDER BY`도

- 복잡도 계층 → [[asymptotic-analysis]]

- 비교 없는 정렬 → [[linear-sorts]]
