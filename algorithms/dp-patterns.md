# DP 패턴 (DP Patterns)

## 한 줄 요약

자주 나오는 DP 문제 유형들 - LCS, LIS, 배낭, 편집 거리, 구간 DP. 각각의 상태 정의와 점화식을 익히면 새 문제도 이 틀로 환원된다.

## 왜 필요한가

- DP 실전은 패턴 인식 - 유형을 알면 상태/전이가 보임
- 면접·경쟁 프로그래밍 단골
- [[dp-fundamentals]]의 이론을 구체적 문제로

## 1. LCS (최장 공통 부분 수열)

두 문자열의 공통 부분 수열 중 가장 긴 것 (순서 유지, 연속 아님):

```
상태: dp[i][j] = A[0..i], B[0..j]의 LCS 길이
전이: A[i]==B[j] → dp[i][j] = dp[i-1][j-1] + 1
      else       → dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

- O(mn) 시간·공간
- 응용: diff(파일 비교), DNA 정렬, git merge

## 2. LIS (최장 증가 부분 수열)

수열에서 증가하는 가장 긴 부분 수열:

```
상태: dp[i] = i에서 끝나는 LIS 길이
전이: dp[i] = max(dp[j]+1) for j<i where a[j]<a[i]
```

- 기본 O(n²)
- **최적화 O(n log n)**: "각 길이의 LIS 끝 최솟값"을 유지하며 이진 탐색([[binary-search]])으로 갱신. DP + 이분 탐색 조합

## 3. 배낭 (Knapsack)

무게 한도 W, 각 물건 (무게, 가치). 가치 최대화:

### 0/1 배낭 (각 물건 0 or 1개)

```
상태: dp[i][w] = i번째까지, 무게 w 이하 최대 가치
전이: dp[i][w] = max(dp[i-1][w],                    // 안 넣음
                     dp[i-1][w-wt[i]] + val[i])     // 넣음
```

- O(nW). **의사 다항(pseudo-polynomial)** - W가 입력 크기가 아니라 값이라 진짜 다항 아님
- **그리디가 안 되는 대표**: "가치/무게 비율 순"이 최적 아님 → DP 필요 ([[greedy]] 대조)

### 무한 배낭 (물건 무한)

전이에서 `dp[i-1]` 대신 `dp[i]` (같은 물건 재사용). 동전 교환도 이 형태.

## 4. 편집 거리 (Edit Distance)

문자열 A를 B로 바꾸는 최소 연산(삽입/삭제/교체):

```
상태: dp[i][j] = A[0..i]를 B[0..j]로 바꾸는 최소 연산
전이: A[i]==B[j] → dp[i-1][j-1]
      else → 1 + min(dp[i-1][j],    // 삭제
                     dp[i][j-1],    // 삽입
                     dp[i-1][j-1])  // 교체
```

- O(mn). 맞춤법 검사, DNA 정렬, 유사도, diff
- LCS의 사촌 (둘 다 문자열 정렬 DP)

## 5. 구간 DP (Interval DP)

구간 [i,j]의 답을 더 작은 구간으로:

```
상태: dp[i][j] = 구간 [i,j]의 답
전이: dp[i][j] = min/max over k (dp[i][k] + dp[k+1][j] + 결합비용)
```

- 예: 행렬 곱 순서(matrix chain), 돌 합치기, 회문 분할
- O(n³) (구간 O(n²) × 분할점 O(n))

## 패턴 인식 체크리스트

새 DP 문제를 만나면:

1. **선택의 연속인가?** (각 단계 결정) → 상태에 "몇 번째까지"
2. **두 시퀀스 비교?** → dp[i][j] (LCS/편집거리 계열)
3. **용량/예산 제약?** → 배낭 계열, 상태에 남은 자원
4. **구간을 쪼갬?** → 구간 DP dp[i][j]
5. **끝나는 지점 기준?** → LIS 계열 dp[i]

상태를 "무엇을 알면 나머지를 풀 수 있나"로 정의하는 게 핵심.

## 셀프 체크

> [!question]- LCS와 편집 거리가 "사촌"인 이유는 무엇인가?
> 둘 다 dp[i][j]로 두 문자열의 접두사를 비교하는 문자열 정렬 DP다. 문자가 같으면 대각선 dp[i-1][j-1]을 이어받고, 다르면 이웃 상태들에서 최적을 고른다. LCS는 공통 길이를 최대화, 편집 거리는 변환 연산을 최소화한다는 목적만 다르다.

> [!question]- 0/1 배낭이 "의사 다항"이라 불리는 이유는?
> 복잡도가 O(nW)인데 W는 입력의 개수가 아니라 용량 값이기 때문이다. W를 이진수로 표현하면 입력 크기는 log W라, W에 비례하는 시간은 입력 크기에 대해 지수적이다. 그래서 진짜 다항이 아니라 의사 다항이라 한다.

> [!question]- 배낭이 그리디로 풀리지 않는 대표 예인 이유는?
> "가치/무게 비율이 높은 것부터"라는 그리디 선택이 0/1 배낭에서 최적을 보장하지 못하기 때문이다. 한 물건을 통째로 넣거나 빼야 하는 제약 때문에 지역 최적 선택이 전역 최적을 놓칠 수 있어, 모든 선택을 고려하는 DP가 필요하다.

> [!question]- LIS를 O(n²)에서 O(n log n)으로 줄이는 아이디어는?
> "각 길이의 LIS가 끝나는 값 중 최솟값"을 배열로 유지하고, 새 원소마다 이진 탐색으로 들어갈 자리를 찾아 갱신한다. dp[i]를 위해 모든 j를 훑는 대신 이분 탐색으로 갱신 위치를 찾으므로 원소당 O(log n), 전체 O(n log n)이 된다.

> [!question]- 구간 DP가 O(n³)인 이유를 상태와 전이로 설명하라.
> 상태 dp[i][j]는 구간 [i,j]로 O(n²)개다. 각 상태의 전이가 분할점 k를 [i,j) 범위에서 훑으며 dp[i][k]+dp[k+1][j]+결합비용의 최적을 고르므로 전이 1회가 O(n)이다. 상태 수 O(n²) × 전이 O(n) = O(n³).

## 연습문제

> [!example]- 문제: LCS의 상태와 점화식을 직접 유도하고 복잡도를 밝혀라
> **풀이**
> 상태: dp[i][j] = A[0..i], B[0..j]의 LCS 길이.
> 마지막 문자를 본다. A[i]==B[j]면 그 문자를 공통에 붙일 수 있으므로 dp[i][j]=dp[i-1][j-1]+1.
> 다르면 둘 중 하나를 버려야 하므로 dp[i][j]=max(dp[i-1][j], dp[i][j-1]).
> 기저는 한쪽이 빈 문자열이면 0.
> 상태 O(mn)개 × 전이 O(1) = O(mn) 시간·공간.

> [!example]- 문제: 0/1 배낭의 점화식을 세우고 "넣음/안 넣음" 두 갈래가 왜 필요한지 설명하라
> **풀이**
> 상태: dp[i][w] = i번째 물건까지 고려하고 무게 한도 w일 때 최대 가치.
> 물건 i에 대한 결정은 두 가지뿐이다.
> - 안 넣음: dp[i-1][w] (한도 그대로, 물건 i 제외)
> - 넣음: dp[i-1][w-wt[i]] + val[i] (단 w≥wt[i]일 때, 무게만큼 한도 소비)
> dp[i][w] = max(안 넣음, 넣음). 두 선택을 모두 고려해 최적을 취해야 그리디가 놓치는 경우를 잡는다.
> O(nW).

> [!example]- 문제: 새 DP 문제를 만났을 때 유형을 식별하는 체크리스트를 신호별로 정리하라
> **풀이**
> 1. 각 단계마다 결정을 내리는 선택의 연속인가 → 상태에 "몇 번째까지" 포함.
> 2. 두 시퀀스를 비교하는가 → dp[i][j] (LCS/편집 거리 계열).
> 3. 용량·예산 제약이 있는가 → 배낭 계열, 상태에 남은 자원.
> 4. 구간을 쪼개 결합하는가 → 구간 DP dp[i][j].
> 5. 끝나는 지점 기준으로 정의되는가 → LIS 계열 dp[i].
> 공통 원칙: 상태를 "무엇을 알면 나머지를 풀 수 있나"로 정의한다.
