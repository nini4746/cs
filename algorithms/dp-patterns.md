# DP 패턴 (DP Patterns)

## 한 줄 요약

자주 나오는 DP 문제 유형들 - LCS, LIS, 배낭, 편집 거리, 구간 DP. 각각의 상태 정의와 점화식을 익히면 새 문제도 이 틀로 환원된다.

## 왜 필요한가

- DP 실전은 패턴 인식 - 유형을 알면 상태/전이가 보임
- 면접·경쟁 프로그래밍 단골
- [[dp-fundamentals]]의 이론을 구체적 문제로

## 1. LCS (최장 공통 부분 수열)

두 문자열의 공통 부분 수열 중 가장 긴 것 (순서 유지, 연속 아님):

```
상태: dp[i][j] = A[0..i], B[0..j]의 LCS 길이
전이: A[i]==B[j] → dp[i][j] = dp[i-1][j-1] + 1
      else       → dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

- O(mn) 시간·공간
- 응용: diff(파일 비교), DNA 정렬, git merge

## 2. LIS (최장 증가 부분 수열)

수열에서 증가하는 가장 긴 부분 수열:

```
상태: dp[i] = i에서 끝나는 LIS 길이
전이: dp[i] = max(dp[j]+1) for j<i where a[j]<a[i]
```

- 기본 O(n²)
- **최적화 O(n log n)**: "각 길이의 LIS 끝 최솟값"을 유지하며 이진 탐색([[binary-search]])으로 갱신. DP + 이분 탐색 조합

## 3. 배낭 (Knapsack)

무게 한도 W, 각 물건 (무게, 가치). 가치 최대화:

### 0/1 배낭 (각 물건 0 or 1개)

```
상태: dp[i][w] = i번째까지, 무게 w 이하 최대 가치
전이: dp[i][w] = max(dp[i-1][w],                    // 안 넣음
                     dp[i-1][w-wt[i]] + val[i])     // 넣음
```

- O(nW). **의사 다항(pseudo-polynomial)** - W가 입력 크기가 아니라 값이라 진짜 다항 아님
- **그리디가 안 되는 대표**: "가치/무게 비율 순"이 최적 아님 → DP 필요 ([[greedy]] 대조)

### 무한 배낭 (물건 무한)

전이에서 `dp[i-1]` 대신 `dp[i]` (같은 물건 재사용). 동전 교환도 이 형태.

## 4. 편집 거리 (Edit Distance)

문자열 A를 B로 바꾸는 최소 연산(삽입/삭제/교체):

```
상태: dp[i][j] = A[0..i]를 B[0..j]로 바꾸는 최소 연산
전이: A[i]==B[j] → dp[i-1][j-1]
      else → 1 + min(dp[i-1][j],    // 삭제
                     dp[i][j-1],    // 삽입
                     dp[i-1][j-1])  // 교체
```

- O(mn). 맞춤법 검사, DNA 정렬, 유사도, diff
- LCS의 사촌 (둘 다 문자열 정렬 DP)

## 5. 구간 DP (Interval DP)

구간 [i,j]의 답을 더 작은 구간으로:

```
상태: dp[i][j] = 구간 [i,j]의 답
전이: dp[i][j] = min/max over k (dp[i][k] + dp[k+1][j] + 결합비용)
```

- 예: 행렬 곱 순서(matrix chain), 돌 합치기, 회문 분할
- O(n³) (구간 O(n²) × 분할점 O(n))

## 패턴 인식 체크리스트

새 DP 문제를 만나면:

1. **선택의 연속인가?** (각 단계 결정) → 상태에 "몇 번째까지"
2. **두 시퀀스 비교?** → dp[i][j] (LCS/편집거리 계열)
3. **용량/예산 제약?** → 배낭 계열, 상태에 남은 자원
4. **구간을 쪼갬?** → 구간 DP dp[i][j]
5. **끝나는 지점 기준?** → LIS 계열 dp[i]

상태를 "무엇을 알면 나머지를 풀 수 있나"로 정의하는 게 핵심.
