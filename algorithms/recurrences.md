# 점화식 (Recurrences)

## 한 줄 요약

재귀 알고리즘의 실행 시간은 점화식으로 표현된다. 마스터 정리는 분할 정복형 점화식 T(n)=aT(n/b)+f(n)을 세 경우로 즉시 푼다. 재귀 트리는 직관을 준다.

## 왜 필요한가

- 재귀/분할 정복 알고리즘의 복잡도를 어떻게 구하나
- 병합 정렬이 왜 O(n log n)인가 (증명)
- 마스터 정리를 언제 어떻게 쓰나

## 점화식이란

재귀 함수의 실행 시간을 자기 참조로 표현:

```
병합 정렬: T(n) = 2T(n/2) + O(n)
  = 절반 크기 2번 재귀 + 병합 O(n)
기저: T(1) = O(1)
```

이걸 닫힌 형(O 표기)으로 풀어야 복잡도를 앎.

## 방법 1: 재귀 트리 (직관)

재귀 호출을 트리로 그려 각 레벨의 작업량을 합침:

```
병합 정렬 T(n) = 2T(n/2) + cn:

레벨0:        cn                    = cn
레벨1:    cn/2   cn/2               = cn
레벨2:  cn/4 cn/4 cn/4 cn/4         = cn
  ...
레벨 log n:  각 c씩 n개             = cn
```

- 각 레벨의 총 작업 = cn (같음)
- 레벨 수 = log n (n을 반씩 나눠 1까지)
- 총합 = cn × log n = **O(n log n)** ✓

재귀 트리는 "왜"를 보여줌. 마스터 정리는 "빠르게".

## 방법 2: 마스터 정리 (기계적)

**T(n) = a·T(n/b) + f(n)** 꼴 (a≥1, b>1)을 세 경우로 즉시:

- a = 재귀 호출 수
- n/b = 부분 문제 크기
- f(n) = 나누고 합치는 비용

**임계 지수** c* = log_b(a)와 f(n)의 차수를 비교:

| 경우 | 조건 | 결과 |
|---|---|---|
| 1 | f(n) < n^c* (재귀가 지배) | T(n) = Θ(n^c*) |
| 2 | f(n) = Θ(n^c*) (균형) | T(n) = Θ(n^c* log n) |
| 3 | f(n) > n^c* (합치기가 지배) | T(n) = Θ(f(n)) |

### 예시

- **병합 정렬** T=2T(n/2)+n: a=2,b=2 → c*=log₂2=1. f=n=n¹ → 경우2 → **Θ(n log n)** ✓
- **이진 탐색** T=T(n/2)+1: a=1,b=2 → c*=0. f=1=n⁰ → 경우2 → **Θ(log n)** ✓
- **순진한 행렬곱** T=8T(n/2)+n²: a=8,b=2 → c*=3. f=n²<n³ → 경우1 → **Θ(n³)**
- **Strassen** T=7T(n/2)+n²: c*=log₂7≈2.81 → 경우1 → **Θ(n^2.81)** (순진한 것보다 빠름)

## 직관: 어디에 무게가 있나

세 경우의 본질:

- **경우 1 (잎이 지배)**: 재귀 호출이 폭발적 → 작업이 트리 바닥에 몰림 → 잎 개수가 결정
- **경우 2 (균형)**: 모든 레벨이 같은 작업 → 레벨 수(log)만큼 곱함
- **경우 3 (루트가 지배)**: 합치기가 비싸 → 최상위 f(n)이 결정

## 마스터 정리가 안 되는 경우

- 부분 문제 크기가 불균등: T(n) = T(n/3) + T(2n/3) + n → 재귀 트리나 Akra-Bazzi
- f(n)이 다항식이 아님 (log 등 낀 경우 - 확장판 필요)
- a나 b가 상수 아님
- 이럴 땐 재귀 트리, 치환법(substitution, 추측 후 귀납 증명 → [[induction]])

## 방법 3: 치환법

답을 추측하고 **귀납법**으로 증명:

1. T(n) = O(g(n))으로 추측
2. T(n) ≤ c·g(n)을 작은 값에서 성립 가정
3. 점화식에 대입해 n에서도 성립 증명 → [[induction]]

## 셀프 체크

> [!question]- 재귀 트리와 마스터 정리는 각각 무엇에 강한가?
> 재귀 트리는 각 레벨의 작업량과 레벨 수를 그려 "왜" 그 복잡도인지 직관을 준다. 마스터 정리는 같은 문제를 세 경우로 즉시 기계적으로 "빠르게" 푼다.

> [!question]- 마스터 정리의 세 경우는 무엇을 기준으로 갈리나?
> 임계 지수 c* = log_b(a)와 f(n)의 차수를 비교한다. f(n) < n^c*면 경우1(잎이 지배), 같으면 경우2(균형), 크면 경우3(루트가 지배)이다.

> [!question]- 병합 정렬이 왜 Θ(n log n)인가?
> T=2T(n/2)+cn에서 각 레벨 작업이 cn으로 모두 같고 레벨 수가 log n이라 cn·log n이 된다. 마스터 정리로는 c*=1이고 f=n^1이라 경우2에 해당한다.

> [!question]- 마스터 정리를 못 쓰는 경우는 언제인가?
> 부분 문제 크기가 불균등(T(n/3)+T(2n/3)+n)하거나, f(n)이 다항식이 아니거나, a·b가 상수가 아닐 때다. 이럴 땐 재귀 트리, Akra-Bazzi, 치환법을 쓴다.

> [!question]- 치환법은 어떻게 진행하나?
> 답을 T(n)=O(g(n))으로 추측하고, 작은 값에서 T(n)≤c·g(n)을 가정한 뒤, 점화식에 대입해 n에서도 성립함을 귀납적으로 증명한다.

## 연습문제

> [!example]- 문제: 순진한 행렬곱 T=8T(n/2)+n²와 Strassen T=7T(n/2)+n²를 마스터 정리로 풀고, Strassen이 더 빠른 이유를 밝혀라.
> **풀이**
> 둘 다 b=2, f(n)=n². 순진: a=8이라 c*=log₂8=3, f=n²<n³이니 경우1 → Θ(n³).
> Strassen: a=7이라 c*=log₂7≈2.81, f=n²<n^2.81이니 경우1 → Θ(n^2.81).
> 재귀 호출 수를 8에서 7로 줄여 c*를 3에서 2.81로 낮췄기 때문에 더 빠르다.

> [!example]- 문제: T=2T(n/2)+cn이 O(n log n)임을 재귀 트리로 유도하라.
> **풀이**
> 각 레벨의 총 작업량: 레벨0은 cn, 레벨1은 두 부분이 각 cn/2라 합 cn, 레벨2도 4개가 각 cn/4라 cn... 모든 레벨이 cn으로 같다.
> 레벨 수는 n을 반씩 나눠 1까지 가므로 log n.
> 총합 = cn × log n = O(n log n).

> [!example]- 문제: 이진 탐색 T=T(n/2)+1을 마스터 정리로 풀어라.
> **풀이**
> a=1, b=2 → c*=log₂1=0. f(n)=1=n^0이라 f=Θ(n^c*) → 경우2 → Θ(n^0 log n)=Θ(log n).

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: 이 개념을 이해했다면 답할 수 있어야 하는 것들 - (1) T(n)=aT(n/b)+f(n)에서 a, b, f 각각이 뜻하는 것과 c*=log_b(a)를 어떻게 얻는지, (2) 세 경우를 어디에 무게가 실리는지(잎/균형/루트)로 설명, (3) 재귀 트리로 병합 정렬을 O(n log n)까지 직접 전개.
