# 이진 탐색 (Binary Search)

## 한 줄 요약

정렬된 배열에서 절반씩 줄여가며 O(log n)에 찾는다. 단순해 보이지만 off-by-one 버그의 온상이다. 반열린 구간 [lo, hi)로 통일하면 변형(lower_bound, 답 이분 탐색)도 안전하게 짤 수 있다.

## 왜 필요한가

- 가장 기본적인 O(log n) 알고리즘
- off-by-one 없이 짜는 법 (JDK도 9년간 버그였음 → [[bits-and-integers]])
- "답을 이분 탐색"이라는 강력한 응용

## 기본 아이디어

정렬된 배열에서 중간을 보고 목표와 비교, 절반을 버림:

```
목표보다 중간이 작으면 → 오른쪽 절반
크면 → 왼쪽 절반
같으면 → 찾음
```

매 단계 후보가 반 → log₂ n 단계 → **O(log n)**. n=10억이 30단계.

## off-by-one 지옥

이진 탐색은 짧지만 경계 처리가 악명 높음:

- `lo <= hi` vs `lo < hi`?
- `hi = mid` vs `hi = mid - 1`?
- `mid = (lo+hi)/2` → **오버플로우** ([[bits-and-integers]]의 JDK 버그). `mid = lo + (hi-lo)/2`로

이 조합을 헷갈리면 무한 루프나 하나 놓침. 해법: **한 가지 규칙으로 통일.**

## 통일 규칙: 반열린 구간 [lo, hi)

탐색 범위를 **[lo, hi)** (lo 포함, hi 제외)로 일관되게:

```c
int lower_bound(int *a, int n, int x) {  // x 이상인 첫 위치
    int lo = 0, hi = n;              // 전체 = [0, n)
    while (lo < hi) {                // 구간이 비면 종료
        int mid = lo + (hi - lo) / 2;
        if (a[mid] < x) lo = mid + 1;  // mid는 답 아님, 오른쪽
        else hi = mid;                 // mid가 답 후보, 포함
    }
    return lo;   // 첫 x 이상 위치 (없으면 n)
}
```

- 불변식([[correctness-proofs]]): 답은 항상 [lo, hi) 안
- `lo < hi`로 종료 (구간 빔), `mid+1`/`mid`로 갱신
- 이 틀 하나로 대부분 변형 커버

## 변형: lower_bound / upper_bound

단순 "찾기"보다 **경계 찾기**가 실전에 유용:

- **lower_bound(x)**: x **이상**인 첫 위치
- **upper_bound(x)**: x **초과**인 첫 위치
- upper − lower = x의 개수 (중복 처리)

실측 (`{1,3,3,3,5,7,9}`):

```
lower_bound(3)=1    ← 첫 3의 위치 (중복 중 첫)
lower_bound(4)=4    ← 4 없음, 4 이상 첫 위치(5)
lower_bound(0)=0    ← 전부보다 작음
lower_bound(10)=7   ← 전부보다 큼 = n (못 찾음 신호)
```

모든 경계(존재/부재/양끝)가 정확. C++ `std::lower_bound`, Python `bisect_left`가 이것.

## 강력한 응용: 답을 이분 탐색

배열이 아니라 **답의 범위를 이분 탐색**. "조건을 만족하는 최소/최대 값"을 찾을 때:

```
"단조성"이 있으면 이분 탐색 가능:
  어떤 값 x에서 조건이 참이면 x보다 큰(작은) 것도 참
→ 참/거짓 경계를 이분 탐색
```

예:
- **최소 용량 찾기**: "이 용량으로 D일 안에 배송 가능?" → 가능/불가능이 단조 → 최소 용량 이분 탐색
- **제곱근**: f(x)=x²이 단조 → sqrt 이분 탐색
- **파라메트릭 서치**: 최적화를 판정 문제로 바꿔 이분

핵심: **판정 함수가 단조**면(어느 지점부터 참/거짓 뒤집힘) 그 경계를 O(log(범위) × 판정비용)에. 배열 없이도 이분 탐색.

## 조건: 단조성

이진 탐색은 **정렬(단조성)**이 전제:
- 배열: 정렬돼야
- 답 이분: 판정 함수가 단조여야
- 단조성 없으면 절반을 못 버림 → 이진 탐색 불가

## 셀프 체크

> [!question]- 이진 탐색이 O(log n)인 이유를 단계 수로 설명하라.
> 매 단계마다 후보 범위가 절반으로 줄기 때문이다. n에서 시작해 반씩 줄이면 log₂n 단계에서 범위가 1이 된다. n=10억이 약 30단계다.

> [!question]- 반열린 구간 [lo, hi)로 통일하면 off-by-one이 왜 줄어드나?
> lo 포함/hi 제외로 경계 규칙을 하나로 고정하기 때문이다. 종료 조건은 항상 `lo < hi`(구간이 비면 끝), 갱신은 답이 아닌 쪽은 `lo = mid+1`, 답 후보인 쪽은 `hi = mid`로 통일된다. 불변식 "답은 항상 [lo, hi) 안"이 유지된다.

> [!question]- `mid = (lo+hi)/2`가 왜 위험하고 어떻게 고치나?
> lo+hi가 int 범위를 넘으면 오버플로우가 난다(JDK가 9년간 이 버그였다). `mid = lo + (hi-lo)/2`로 쓰면 lo+hi를 직접 더하지 않아 안전하다.

> [!question]- lower_bound와 upper_bound의 차이, 그리고 둘로 무엇을 계산하나?
> lower_bound(x)는 x 이상인 첫 위치, upper_bound(x)는 x 초과인 첫 위치다. upper − lower가 배열 안 x의 개수라서 중복 처리에 쓴다.

> [!question]- "답을 이분 탐색"이 성립하려면 무엇이 필요한가?
> 판정 함수의 단조성이다. 어떤 값에서 조건이 참이면 그보다 큰(또는 작은) 값에서도 참이어야, 참/거짓의 경계를 이분 탐색할 수 있다. 단조성이 없으면 절반을 버릴 수 없어 이진 탐색 자체가 불가능하다.

## 연습문제

> [!example]- 문제: lower_bound를 반열린 구간 [lo, hi) 틀로 직접 구현하고 불변식을 밝혀라.
> **풀이**
> ```c
> int lower_bound(int *a, int n, int x) {
>     int lo = 0, hi = n;              // [0, n)
>     while (lo < hi) {
>         int mid = lo + (hi - lo) / 2;
>         if (a[mid] < x) lo = mid + 1;  // mid는 답 아님
>         else hi = mid;                 // mid가 답 후보
>     }
>     return lo;
> }
> ```
> 불변식: 답(x 이상인 첫 위치)은 항상 [lo, hi) 안에 있다. `a[mid] < x`면 mid 이하는 전부 답이 아니므로 lo를 mid+1로, 아니면 mid도 후보라 hi를 mid로 좁힌다. 구간이 비면 lo가 답이다.

> [!example]- 문제: "D일 안에 배송 가능한 최소 선박 용량"을 답 이분 탐색으로 설계하라.
> **풀이**
> 판정 함수 feasible(cap): 용량 cap으로 짐을 순서대로 실어 며칠 걸리는지 세고 D일 이하면 참.
> 이 함수는 단조다 - cap이 크면 참일수록 유지된다. 따라서 참이 되는 최소 cap을 이분 탐색한다.
> 범위 lo = max(개별 짐 무게)(한 개는 실어야 함), hi = sum(모든 무게)(하루에 다 실음). [lo, hi]에서 feasible이 참인 첫 경계를 찾으면 답. 복잡도 O(n log(sum)).

> [!example]- 문제: `hi = mid` 대신 `hi = mid - 1`을 [lo, hi) 틀에 잘못 쓰면 무슨 일이 생기는지 반례로 보여라.
> **풀이**
> [lo, hi)에서 hi는 제외되는 경계이자 "아직 확정 안 된 답 후보의 상한"이다. `a[mid] >= x`일 때 mid는 답 후보이므로 버리면 안 된다.
> 예: a = {2, 4}, x = 4. lo=0, hi=2, mid=1, a[1]=4 ≥ 4. 올바르면 hi=mid=1이 되어 결국 lo=1을 반환(정답).
> `hi = mid - 1 = 0`으로 하면 lo=0, hi=0으로 구간이 비어 lo=0을 반환 - 첫 4의 위치 1을 놓친다. 답 후보를 버렸기 때문이다.
