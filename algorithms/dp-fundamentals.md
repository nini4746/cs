# 동적 계획법 기초 (DP Fundamentals)

## 한 줄 요약

부분 문제가 겹칠 때, 한 번 푼 결과를 저장해 재사용한다. 지수 시간을 다항 시간으로 바꾸는 마법. 최적 부분 구조와 중복 부분 문제가 있으면 DP가 적용된다.

## 왜 필요한가

- 왜 어떤 재귀는 지수 시간이고 DP는 다항인가
- DP를 언제 쓸 수 있는지 판별
- top-down vs bottom-up 선택

## 두 조건

DP가 적용되려면:

1. **최적 부분 구조(optimal substructure)**: 전체 최적해가 부분 문제 최적해로 구성됨
2. **중복 부분 문제(overlapping subproblems)**: 같은 부분 문제가 여러 번 등장

분할 정복([[divide-and-conquer]])은 부분 문제가 **독립**(안 겹침)이라 DP 불필요. 겹치면 DP.

## 실증: 피보나치

`fib(n) = fib(n-1) + fib(n-2)`. 순진한 재귀는 같은 값을 반복 계산:

```
fib(5) → fib(4) + fib(3)
         fib(4) → fib(3) + fib(2)   ← fib(3) 또 계산!
```

같은 부분 문제(fib(3) 등)가 지수적으로 중복 → **O(2ⁿ)**.

실측 fib(40):

```
naive: 331,160,281 calls, 0.632s     ← 3.3억 번 호출
dp   : 40 iterations, 0.000000s       ← 40번
naive는 dp보다 ~631,716배 느림
```

63만 배. 순진한 재귀가 fib(3)을 수백만 번 다시 계산. DP는 한 번씩만.

## 두 방식

### top-down (메모이제이션)

재귀 + 캐시. 계산한 값을 저장하고 재사용:

```c
long memo[100]; // 초기 -1
long fib(int n){
    if(n<2) return n;
    if(memo[n]!=-1) return memo[n];    // 이미 있으면 재사용
    return memo[n] = fib(n-1)+fib(n-2);
}
```

- 자연스러운 재귀 형태 유지
- 필요한 부분 문제만 계산 (희소하면 유리)
- 재귀 오버헤드, 스택 깊이 주의

### bottom-up (타뷸레이션)

작은 것부터 표를 채움:

```c
long fib(int n){
    long a=0, b=1;
    for(int i=0;i<n;i++){ long t=a+b; a=b; b=t; }
    return a;
}
```

- 재귀 없음, 반복만 → 빠르고 스택 안전
- **공간 최적화 가능**: 피보나치는 직전 2개만 필요 → O(1) 공간 (전체 배열 불필요)
- 계산 순서를 직접 관리 (의존성 순서로)

보통 bottom-up이 빠르지만, 부분 문제가 희소하거나 순서가 복잡하면 top-down.

## DP 설계 단계

1. **상태 정의**: 부분 문제를 뭐로 표현? (dp[i] = 무엇의 답)
2. **점화식**: dp[i]를 더 작은 dp로 (전이)
3. **기저**: 가장 작은 경우
4. **순서**: 의존하는 것을 먼저 계산
5. **답**: 어느 dp 값이 최종 답

예 (피보나치): 상태 dp[i]=i번째 피보나치, 전이 dp[i]=dp[i-1]+dp[i-2], 기저 dp[0]=0,dp[1]=1.

## 복잡도

**상태 수 × 전이 비용.** 피보나치: n개 상태 × O(1) 전이 = O(n). 대부분 DP 복잡도가 이 곱.

## DP vs 다른 패러다임

| | DP | 분할 정복 | 그리디 |
|---|---|---|---|
| 부분 문제 | 겹침 (저장) | 독립 | 하나만 |
| 모든 선택 | 다 고려 후 최적 | - | 지역 최적 즉시 |
| 안전성 | 항상 최적 | - | 증명 필요 |

DP는 **모든 선택을 고려**해 최적을 보장. 그리디([[greedy]])는 하나만 골라 빠르지만 최적 보장엔 증명 필요. 그리디가 틀리는 문제(0/1 배낭)를 DP는 맞게 품.

## 셀프 체크

> [!question]- DP가 적용되기 위한 두 조건은 무엇이며 각각 왜 필요한가?
> 최적 부분 구조와 중복 부분 문제다. 최적 부분 구조가 있어야 전체 최적해를 부분 문제 최적해로 조립할 수 있고, 중복 부분 문제가 있어야 저장·재사용으로 이득을 본다. 겹치지 않으면 저장할 이유가 없어 분할 정복이면 충분하다.

> [!question]- 순진한 피보나치 재귀가 O(2ⁿ)인 이유를 중복 관점에서 설명하라.
> fib(5)가 fib(4)와 fib(3)을 부르고, fib(4)가 다시 fib(3)을 부르는 식으로 같은 부분 문제가 지수적으로 여러 번 재계산되기 때문이다. fib(40) 실측에서 순진한 재귀는 3.3억 번 호출, DP는 40번으로 약 63만 배 차이가 났다.

> [!question]- top-down과 bottom-up은 언제 각각 유리한가?
> bottom-up은 재귀 오버헤드가 없고 스택이 안전하며 공간 압축이 쉬워 보통 더 빠르다. 반면 부분 문제가 희소해 일부만 필요하거나 계산 순서가 복잡할 때는 필요한 것만 계산하고 자연스러운 재귀 형태를 유지하는 top-down이 낫다.

> [!question]- DP의 복잡도를 어떻게 어림하는가?
> 상태 수 × 전이 1회 비용이다. 피보나치는 상태가 n개이고 전이가 O(1)이라 O(n)이 된다. 대부분의 DP 복잡도가 이 곱으로 설명된다.

> [!question]- 같은 문제에서 그리디 대신 DP를 써야 하는 경우는?
> 그리디는 매 단계 지역 최적 하나만 고르므로 그 선택이 전역 최적임을 증명해야 안전하다. 0/1 배낭처럼 지역 최적이 전역 최적을 보장하지 못하는 문제에서는, 모든 선택을 고려한 뒤 최적을 취하는 DP가 옳은 답을 준다.

## 연습문제

> [!example]- 문제: 피보나치를 top-down 메모이제이션으로 직접 구현하고 왜 O(n)인지 밝혀라
> **풀이**
> ```c
> long memo[100]; // 초기 -1
> long fib(int n){
>     if(n<2) return n;
>     if(memo[n]!=-1) return memo[n];
>     return memo[n] = fib(n-1)+fib(n-2);
> }
> ```
> 각 memo[n]은 처음 한 번만 실제 계산되고 이후엔 캐시에서 즉시 반환된다.
> 서로 다른 상태가 n개, 각 상태의 전이가 O(1)이므로 상태 수 × 전이 = O(n).

> [!example]- 문제: 피보나치 bottom-up을 O(1) 공간으로 설계하고 그게 가능한 근거를 대라
> **풀이**
> dp[i]가 dp[i-1], dp[i-2]에만 의존하므로 전체 배열 대신 직전 두 값만 유지하면 된다.
> ```c
> long fib(int n){
>     long a=0, b=1;
>     for(int i=0;i<n;i++){ long t=a+b; a=b; b=t; }
>     return a;
> }
> ```
> 의존 범위가 상수(직전 2개)이므로 표를 굴리며 O(1) 공간으로 압축 가능하다.

> [!example]- 문제: 임의의 최적화 DP를 세우는 5단계 절차를 순서대로 쓰고 각 단계가 하는 일을 말하라
> **풀이**
> 1. 상태 정의 - 부분 문제를 무엇으로 표현할지(dp[i]가 무엇의 답인지) 정한다.
> 2. 점화식 - dp[i]를 더 작은 dp들로 잇는 전이를 세운다.
> 3. 기저 - 가장 작은 경우의 값을 직접 정한다.
> 4. 순서 - 의존하는 상태를 먼저 계산하도록 순서를 잡는다.
> 5. 답 - 어느 dp 값이 최종 답인지 지정한다.
