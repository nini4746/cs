# P vs NP

## 한 줄 요약

P는 빠르게 풀 수 있는 문제, NP는 답을 빠르게 검증할 수 있는 문제. "푸는 것과 검증하는 것이 같은가(P=NP?)"는 컴퓨터 과학 최대 미해결 문제. NP-완전 문제는 하나만 빨리 풀려도 전부 풀린다.

## 왜 필요한가

- "이 문제는 효율적으로 풀 수 있나"의 이론적 경계
- NP-hard를 만나면 최적해를 포기하고 근사로 가야 하는 이유
- 왜 어떤 문제(TSP, SAT)는 아무도 빠른 알고리즘을 못 찾나

## P: 다항 시간에 푸는 문제

**결정 문제**(예/아니오)를 **다항 시간 O(n^k)**에 푸는 것들:

- 정렬, 최단 경로, 매칭, 최대 유량... 지금까지 배운 대부분
- "효율적으로 풀 수 있다"의 표준 정의 (다항 = 실용적, 지수 = 비실용)

## NP: 다항 시간에 검증하는 문제

답(증거, certificate)이 주어지면 **다항 시간에 맞는지 확인**할 수 있는 문제:

- 예: "이 그래프에 해밀턴 경로가 있나?" - 경로를 주면 확인은 쉬움(다항), 찾는 건 어려움
- NP = Nondeterministic Polynomial (비결정적 튜링 머신이 다항, → automata/[[turing-machines]])
- **P ⊆ NP**: 빨리 풀면 빨리 검증도 됨 (풀어서 비교)

핵심 직관: **찾기 vs 확인**. 스도쿠를 푸는 건 어렵지만 푼 답이 맞는지 확인은 쉬움. NP = 확인이 쉬운 문제.

## P = NP?

**미해결**: 검증이 쉬운 모든 문제가 푸는 것도 쉬운가?

- P = NP라면: 검증 가능 = 풀이 가능 → 암호학 붕괴([[crypto-basics]]), 최적화 혁명
- P ≠ NP라면 (대부분의 믿음): 본질적으로 어려운 문제가 존재
- Clay 연구소 100만 달러 현상 문제. 아무도 증명 못 함

## NP-완전 (NP-Complete)

NP에서 **가장 어려운** 문제들. 두 조건:
1. NP에 속함
2. NP의 **모든** 문제가 이것으로 다항 시간 환원됨

**핵심 성질**: NP-완전 문제 **하나라도** 다항 시간에 풀리면 → **NP 전체가 P** (전부 풀림). 반대로 하나라도 안 풀리면 전부 안 풀림. 한 배를 탐.

### Cook-Levin 정리

**SAT(불리언 만족성)가 첫 NP-완전 문제**임을 증명 (1971). 이후 환원으로 수천 개가 NP-완전으로 판명:

- **3-SAT**: 논리식 만족 할당
- **TSP** (결정판): 비용 K 이하 순회 존재?
- **배낭, 그래프 채색, 클리크, 정점 커버, 해밀턴 경로**...
- 이들은 서로 환원 가능 → 하나 풀면 다 풀림

## 환원 (reduction)

문제 A를 B로 **다항 시간에 변환**해서 "B를 풀면 A도 풀림"을 보임:

- A ≤ B: A가 B로 환원 → B가 A만큼은 어려움
- NP-완전 증명법: 알려진 NP-완전 문제를 새 문제로 환원
- 환원이 NP-완전성 이론의 핵심 도구 → automata/[[reductions]]

## NP-hard

NP-완전의 "NP 소속" 조건을 뺀 것. **NP의 모든 문제만큼 어렵지만** NP에 속할 필요 없음(더 어려울 수도):

- 최적화 TSP(최소 비용 찾기)는 NP-hard (결정판이 NP-완전)
- 정지 문제는 NP-hard이지만 NP 아님 (결정 불가 → automata/[[decidability]])

## 실전 함의

NP-hard 문제를 만나면 **최적해를 다항 시간에 포기**하고:

1. **근사 알고리즘**: 최적의 몇 % 이내 보장 → [[approximation-and-heuristics]]
2. **휴리스틱**: 보장 없지만 실전에서 잘 됨 (유전 알고리즘, 담금질)
3. **정확하지만 지수**: 작은 입력엔 분기 한정, DP(비트마스크 → [[dp-optimization]])
4. **특수 케이스**: 입력에 구조가 있으면 다항 (트리, 평면 그래프)
5. **SAT 솔버**: 실전 SAT 인스턴스는 현대 솔버가 빠르게 (최악은 지수지만)

"NP-hard = 절대 못 품"이 아니라 "최악 보장은 못 하니 타협"이 실전 자세.

## 연결

- 환원 → automata/[[reductions]]
- 근사·휴리스틱 → [[approximation-and-heuristics]]
- 암호학이 P≠NP에 기댐 → security/[[crypto-basics]]
- 결정 불가 (더 어려움) → automata/[[decidability]]
