# 분할 정복 (Divide and Conquer)

## 한 줄 요약

문제를 같은 형태의 작은 부분으로 나누고, 각각 재귀로 풀고, 결과를 합친다. 병합 정렬·퀵정렬·FFT가 이 패턴이며, 복잡도는 마스터 정리로 분석한다.

## 왜 필요한가

- 여러 핵심 알고리즘의 공통 설계 패턴
- 왜 어떤 문제가 O(n²)에서 O(n log n)으로 개선되나
- DP·그리디와 어떻게 다른가

## 세 단계

1. **분할(divide)**: 문제를 같은 형태의 부분 문제로 (보통 절반)
2. **정복(conquer)**: 부분 문제를 재귀로 풀기 (충분히 작으면 직접)
3. **결합(combine)**: 부분 해를 합쳐 전체 해

복잡도: T(n) = a·T(n/b) + (분할+결합 비용) → 마스터 정리로 풀기 ([[recurrences]]).

## 대표 예시

### 병합 정렬

- 분할: 배열 절반
- 정복: 각 절반 재귀 정렬
- 결합: 두 정렬된 절반 병합 O(n)
- T(n)=2T(n/2)+O(n) → O(n log n) → [[comparison-sorts]]

### 퀵정렬

- 분할: 피벗 기준 분할 O(n) (여기가 일)
- 정복: 양쪽 재귀
- 결합: 없음 (제자리)
- 분할에 무게 → [[comparison-sorts]]

### 이진 탐색

- 분할: 중간으로 절반
- 정복: 한쪽만 재귀
- T(n)=T(n/2)+O(1) → O(log n) → [[binary-search]]

## 고전 예: 최근접 점 쌍

평면의 n개 점에서 가장 가까운 두 점. 순진하게 모든 쌍 O(n²). 분할 정복으로 **O(n log n)**:

1. 분할: x좌표 중앙선으로 좌/우 분할
2. 정복: 각 절반의 최근접 쌍 재귀 (거리 d_L, d_R)
3. 결합: **경계선 근처**(±min(d_L,d_R) 띠)만 검사. 이 띠 안에서 각 점은 상수 개만 비교하면 됨 (기하학적 논증)
4. T(n)=2T(n/2)+O(n) → O(n log n)

결합 단계의 영리함(띠 안만, 상수 개만)이 O(n²)를 O(n log n)으로.

## 고전 예: 카라츠바 곱셈

큰 수 곱셈을 순진하게 하면 O(n²) (n자리). 카라츠바:

```
x = x₁·10^m + x₀,  y = y₁·10^m + y₀
xy = x₁y₁·10^2m + (x₁y₀+x₀y₁)·10^m + x₀y₀
```

순진하게는 곱셈 4번. 카라츠바는 **3번으로** (중간항을 (x₁+x₀)(y₁+y₀)−x₁y₁−x₀y₀로):

- T(n)=3T(n/2)+O(n) → O(n^log₂3) ≈ **O(n^1.585)**
- 곱셈 하나를 줄여 지수를 낮춤. 큰 수 라이브러리(BigInteger)의 기법

같은 발상: Strassen 행렬곱 (8→7 곱셈) → [[recurrences]].

## 분할 정복 vs DP vs 그리디

| | 분할 정복 | DP | 그리디 |
|---|---|---|---|
| 부분 문제 | 독립 (안 겹침) | **겹침** (메모이제이션) | 하나만 선택 |
| 결합 | 부분 해를 합침 | 부분 해 재사용 | 지역 최적 |
| 예 | 병합 정렬 | 편집 거리 | 활동 선택 |

핵심 차이: 분할 정복은 부분 문제가 **독립**(각각 한 번). 부분 문제가 **겹치면** DP([[dp-fundamentals]])로 메모이제이션. 피보나치를 분할 정복으로 하면 지수 시간, DP로 하면 O(n)인 이유.

## 병렬화 이점

부분 문제가 독립이라 **병렬 실행 자연스러움**:
- 좌/우 절반을 다른 코어에서 ([[multicore-and-numa]])
- fork-join 병렬 (병렬 병합 정렬, 병렬 퀵정렬)
- 맵리듀스도 분할 정복 정신 → distributed-systems/
