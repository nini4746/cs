# 그래프 순회 (Graph Traversal)

## 한 줄 요약

그래프의 모든 정점을 방문하는 두 방식 - BFS(너비 우선, 큐)와 DFS(깊이 우선, 스택/재귀). 최단 거리(무가중), 위상 정렬, 사이클 탐지, 연결 요소 등 수많은 문제의 기반.

## 왜 필요한가

- 거의 모든 그래프 알고리즘의 출발점
- BFS vs DFS 언제 뭘 쓰나
- 최단 경로·위상 정렬이 이 위에 세워짐

## 그래프 표현

- **인접 리스트**: 각 정점이 이웃 목록. O(V+E) 공간. 희소 그래프에 적합 (대부분 실전)
- **인접 행렬**: V×V 불리언. O(V²) 공간. 밀집 그래프, 간선 조회 O(1)

순회 복잡도는 표현에 의존: 인접 리스트면 **O(V+E)** (각 정점·간선 한 번).

## BFS (너비 우선)

시작점에서 **가까운 것부터** 층별로. **큐** 사용 ([[stacks-and-queues]]):

```
1. 시작 정점을 큐에, 방문 표시
2. 큐에서 꺼내 → 미방문 이웃을 큐에 추가, 방문 표시
3. 큐 빌 때까지 반복
```

특성:
- **무가중 그래프 최단 경로**: 층별 확장이라 먼저 도달 = 최단 (간선 수 기준)
- 시작점에서 거리 순 방문
- 메모리: 한 층의 정점 수만큼 큐

용도: 최단 경로(무가중), 레벨 순회, 이분 그래프 판정, 미로 최단 경로

## DFS (깊이 우선)

**갈 수 있는 데까지 깊이** 들어갔다 막히면 되돌아옴. **스택**(또는 재귀):

```
방문(v):
  v 방문 표시
  각 미방문 이웃 u: 방문(u)   # 재귀 = 암묵적 스택
```

특성:
- 재귀로 자연스러움 (호출 스택 = DFS 스택 → [[procedures-and-stack]])
- 경로/구조 탐색에 적합
- 스택 깊이 = 경로 길이 (깊은 그래프면 스택 오버플로 주의 → 반복 DFS)

용도: 위상 정렬, 사이클 탐지, 연결 요소, 백트래킹, 트리 순회

## BFS vs DFS

| | BFS | DFS |
|---|---|---|
| 자료구조 | 큐 | 스택/재귀 |
| 방문 순서 | 가까운 것부터 | 깊은 것부터 |
| 최단경로(무가중) | ✓ | ✗ |
| 메모리 | 층 너비 | 경로 깊이 |
| 용도 | 최단거리, 레벨 | 구조, 위상, 사이클 |

## DFS 응용

### 위상 정렬 (topological sort)

DAG(방향 비순환 그래프)에서 "선행 관계를 지키는 순서". DFS 후 완료 순서의 역순:

- 의존성 해결: 빌드 순서, 강의 선수과목, 작업 스케줄
- 사이클 있으면 불가능 (순환 의존)
- O(V+E)

### 사이클 탐지

DFS 중 **현재 경로에 있는 정점**을 다시 만나면 사이클 (방향 그래프). 회색/검정 색칠로 구분.

### 강연결 요소 (SCC)

방향 그래프에서 서로 도달 가능한 최대 집합. Tarjan/Kosaraju 알고리즘 (DFS 2번 또는 로우링크). O(V+E).

### 연결 요소 (무방향)

DFS/BFS로 한 덩어리씩 방문 → 몇 개의 연결된 덩어리인지. Union-Find로도 ([[union-find]]).

## 방문 표시가 핵심

순회의 정확성·종료는 **방문 표시**에 달림:
- 표시 안 하면 사이클에서 무한 루프
- 각 정점을 한 번만 처리 → O(V+E) 보장

## 셀프 체크

> [!question]- 무가중 그래프의 최단 경로를 BFS가 보장하고 DFS는 못 하는 이유는?
> BFS는 큐로 시작점에서 가까운 층부터 확장하므로, 어떤 정점에 처음 도달하는 순간이 곧 최소 간선 수 경로다. DFS는 갈 수 있는 데까지 깊이 들어갔다 되돌아오므로 먼 길로 먼저 도달할 수 있어 최단을 보장하지 못한다.

> [!question]- 순회 복잡도가 O(V+E)가 되는 근거는 무엇인가?
> 방문 표시 덕분에 각 정점을 한 번만 처리하고, 인접 리스트에서 각 간선을 한 번씩만 훑기 때문이다. 정점 처리 V번 + 간선 순회 E번 = O(V+E). 인접 행렬이면 이웃 확인이 V²라 O(V²)가 된다.

> [!question]- 위상 정렬을 DFS로 구할 때 왜 완료 순서의 역순인가?
> DFS에서 어떤 정점 v의 처리가 끝나는(완료되는) 시점에는 v가 도달할 수 있는 모든 후행 정점이 이미 완료돼 있다. 따라서 완료 순서를 뒤집으면 모든 간선이 앞→뒤 방향을 지키게 되어 선행 관계가 보존된다. 사이클이 있으면 순서 자체가 불가능하다.

> [!question]- 방향 그래프의 사이클 탐지에서 "현재 경로"를 따로 구분하는 이유는?
> 단순히 방문했다는 표시만으로는 이미 끝난 다른 가지의 정점과 지금 재귀 스택에 올라있는 정점을 구별할 수 없기 때문이다. 회색(현재 경로에 있음)/검정(완료)으로 색칠해, 회색 정점을 다시 만났을 때만 사이클로 판정한다.

> [!question]- 방문 표시가 없으면 무엇이 잘못되는가?
> 사이클이 있는 그래프에서 같은 정점을 계속 다시 방문해 무한 루프에 빠진다. 방문 표시가 있어야 각 정점을 한 번만 처리해 종료가 보장되고 O(V+E) 복잡도도 성립한다.

## 연습문제

> [!example]- 문제: BFS 의사코드를 큐로 직접 작성하고 각 정점이 큐에 몇 번 들어가는지 밝혀라
> **풀이**
> ```
> BFS(s):
>   방문[s] = true; 큐.push(s)
>   while 큐 not empty:
>     v = 큐.pop()
>     각 이웃 u of v:
>       if not 방문[u]:
>         방문[u] = true; 큐.push(u)
> ```
> 각 정점은 방문 표시되는 순간 딱 한 번 큐에 들어간다(넣기 전에 방문 표시).
> 그래서 push/pop이 정점당 1회, 이웃 순회는 간선당 1회 → O(V+E).

> [!example]- 문제: 방향 그래프에서 DFS 기반 사이클 탐지를 색칠로 설계하라
> **풀이**
> 각 정점에 흰색(미방문)/회색(현재 재귀 경로)/검정(완료) 상태를 둔다.
> ```
> DFS(v):
>   색[v] = 회색
>   각 이웃 u of v:
>     if 색[u] == 회색:  return 사이클 있음   // 역방향 간선
>     if 색[u] == 흰색 and DFS(u): return 사이클 있음
>   색[v] = 검정
>   return 사이클 없음
> ```
> 회색 정점을 다시 만나면 현재 경로로 되돌아오는 간선이므로 사이클. 검정은 이미 끝난 가지라 사이클이 아니다. O(V+E).

> [!example]- 문제: 같은 그래프에서 BFS와 DFS의 메모리 사용이 어떻게 달라지는지 설명하고 극단 예를 들어라
> **풀이**
> BFS는 큐에 한 층의 정점을 모두 담으므로 메모리가 최대 층 너비에 비례한다.
> DFS는 재귀(스택)에 현재 경로만 담으므로 메모리가 경로 깊이에 비례한다.
> 넓고 얕은 그래프(예: 루트 하나에 자식 백만 개)는 BFS 큐가 한 번에 백만 개로 폭발하고, DFS는 깊이 2라 작다.
> 반대로 길게 이어진 사슬(깊이 백만)은 DFS 스택이 깊어져 스택 오버플로 위험, BFS는 층 너비 1이라 작다.

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) BFS와 DFS를 자료구조(큐/스택)·방문 순서·메모리로 대비해 설명할 수 있는가. (2) 왜 BFS가 무가중 최단 경로를 주는지 층별 확장으로 말할 수 있는가. (3) 위상 정렬·사이클 탐지·연결 요소가 어떻게 순회 위에 세워지는지, 방문 표시가 왜 필수인지 짚을 수 있는가.
