# 정당성 증명 (Correctness Proofs)

## 한 줄 요약

알고리즘이 "빠른가"만큼 "맞는가"도 증명해야 한다. 루프는 루프 불변식으로, 재귀는 귀납법으로 정당성을 보인다. 테스트는 버그의 존재만 보이고, 증명은 부재를 보인다.

## 왜 필요한가

- 테스트는 몇몇 입력만 확인 - 모든 입력을 보장 못 함
- 미묘한 알고리즘(이진 탐색, 그리디)이 정말 맞는지
- 왜 그리디가 어떤 문제엔 되고 어떤 문제엔 안 되나 → [[greedy]]

## 테스트 vs 증명

Dijkstra: "테스트는 버그의 **존재**를 보일 뿐, **부재**를 못 보인다."

- 테스트: 유한한 입력 확인. 통과해도 안 본 입력에서 틀릴 수 있음
- 증명: 모든 입력에 대해 정당성 보장
- 실무는 테스트 위주지만, 핵심 알고리즘은 증명으로 확신

## 루프 불변식 (loop invariant)

루프의 정당성을 보이는 표준 도구. **매 반복 시작 시 참인 성질**을 찾아 세 단계로 증명:

1. **초기화(initialization)**: 첫 반복 전 불변식이 참
2. **유지(maintenance)**: 한 반복이 불변식을 보존 (참이면 다음도 참)
3. **종료(termination)**: 루프 끝날 때 불변식 + 종료 조건 → 원하는 결과

수학적 귀납법([[induction]])의 루프 버전. 초기화=기저, 유지=귀납 단계.

### 예: 삽입 정렬

```c
for (i = 1; i < n; i++) {
    // 불변식: a[0..i-1]이 정렬됨
    key = a[i];
    j = i-1;
    while (j >= 0 && a[j] > key) { a[j+1] = a[j]; j--; }
    a[j+1] = key;
}
```

- **초기화**: i=1일 때 a[0..0]은 원소 하나 → 정렬됨 ✓
- **유지**: a[0..i-1] 정렬됐으면, key를 올바른 위치에 삽입해 a[0..i]도 정렬 ✓
- **종료**: i=n일 때 a[0..n-1] 전체 정렬 ✓

불변식이 "무엇이 항상 참인가"를 명확히 해 정당성이 드러남.

## 재귀의 정당성: 귀납법

재귀 알고리즘은 **강귀납법**으로 ([[induction]]):

1. **기저**: 가장 작은 경우 (n=0,1)가 맞음
2. **귀납**: 더 작은 모든 입력에서 맞다고 가정 → 현재 입력도 맞음 증명

예: `fact(n) = n * fact(n-1)`. 기저 fact(0)=1 ✓. fact(n-1)이 (n-1)!이라 가정하면 fact(n) = n×(n-1)! = n! ✓.

재귀 호출을 "작은 경우는 이미 맞다"고 믿고 현재만 증명 - 재귀 정당성의 핵심 사고.

## 그리디의 정당성: 교환 논증

그리디 알고리즘([[greedy]])이 최적임을 보이는 법:

- **교환 논증(exchange argument)**: 최적해가 그리디와 다르다고 가정 → 그리디 선택으로 바꿔도 최악이 아님을 보임 → 그리디도 최적
- **그리디 선택 속성**: 지역 최적 선택이 전역 최적의 일부
- **최적 부분 구조**: 부분 문제의 최적이 전체 최적의 일부

이걸 증명 못 하면 그리디는 틀림. 그리디가 어떤 문제엔 되고(활동 선택) 어떤 문제엔 안 되는(0/1 배낭) 이유 → [[greedy]], [[dp-fundamentals]].

## 종료 증명

정당성 ≠ 종료. 무한 루프면 정당성 무의미. 종료는 보통:

- **감소하는 양(variant/measure)**: 매 반복 줄어드는 음이 아닌 정수 (예: 남은 원소 수). 0 되면 종료 → 유한 반복 보장
- 유클리드 호제법: 나머지가 매번 줄어 → 종료

## 실무에서

- 모든 코드를 증명하진 않음 (비쌈). 하지만:
- **핵심 알고리즘**(이진 탐색 off-by-one → [[binary-search]], 동시성 → os/[[threads-and-races]])은 불변식 사고가 버그를 예방
- **형식 검증(formal verification)**: TLA+, Coq 등으로 중요 시스템 증명 (분산 프로토콜 → distributed-systems/[[consensus-problem]])
- 불변식을 주석/assert로 코드에 남기면 리뷰·디버깅에 도움
