# 정당성 증명 (Correctness Proofs)

## 한 줄 요약

알고리즘이 "빠른가"만큼 "맞는가"도 증명해야 한다. 루프는 루프 불변식으로, 재귀는 귀납법으로 정당성을 보인다. 테스트는 버그의 존재만 보이고, 증명은 부재를 보인다.

## 왜 필요한가

- 테스트는 몇몇 입력만 확인 - 모든 입력을 보장 못 함
- 미묘한 알고리즘(이진 탐색, 그리디)이 정말 맞는지
- 왜 그리디가 어떤 문제엔 되고 어떤 문제엔 안 되나 → [[greedy]]

## 테스트 vs 증명

Dijkstra: "테스트는 버그의 **존재**를 보일 뿐, **부재**를 못 보인다."

- 테스트: 유한한 입력 확인. 통과해도 안 본 입력에서 틀릴 수 있음
- 증명: 모든 입력에 대해 정당성 보장
- 실무는 테스트 위주지만, 핵심 알고리즘은 증명으로 확신

## 루프 불변식 (loop invariant)

루프의 정당성을 보이는 표준 도구. **매 반복 시작 시 참인 성질**을 찾아 세 단계로 증명:

1. **초기화(initialization)**: 첫 반복 전 불변식이 참
2. **유지(maintenance)**: 한 반복이 불변식을 보존 (참이면 다음도 참)
3. **종료(termination)**: 루프 끝날 때 불변식 + 종료 조건 → 원하는 결과

수학적 귀납법([[induction]])의 루프 버전. 초기화=기저, 유지=귀납 단계.

### 예: 삽입 정렬

```c
for (i = 1; i < n; i++) {
    // 불변식: a[0..i-1]이 정렬됨
    key = a[i];
    j = i-1;
    while (j >= 0 && a[j] > key) { a[j+1] = a[j]; j--; }
    a[j+1] = key;
}
```

- **초기화**: i=1일 때 a[0..0]은 원소 하나 → 정렬됨 ✓
- **유지**: a[0..i-1] 정렬됐으면, key를 올바른 위치에 삽입해 a[0..i]도 정렬 ✓
- **종료**: i=n일 때 a[0..n-1] 전체 정렬 ✓

불변식이 "무엇이 항상 참인가"를 명확히 해 정당성이 드러남.

## 재귀의 정당성: 귀납법

재귀 알고리즘은 **강귀납법**으로 ([[induction]]):

1. **기저**: 가장 작은 경우 (n=0,1)가 맞음
2. **귀납**: 더 작은 모든 입력에서 맞다고 가정 → 현재 입력도 맞음 증명

예: `fact(n) = n * fact(n-1)`. 기저 fact(0)=1 ✓. fact(n-1)이 (n-1)!이라 가정하면 fact(n) = n×(n-1)! = n! ✓.

재귀 호출을 "작은 경우는 이미 맞다"고 믿고 현재만 증명 - 재귀 정당성의 핵심 사고.

## 그리디의 정당성: 교환 논증

그리디 알고리즘([[greedy]])이 최적임을 보이는 법:

- **교환 논증(exchange argument)**: 최적해가 그리디와 다르다고 가정 → 그리디 선택으로 바꿔도 최악이 아님을 보임 → 그리디도 최적
- **그리디 선택 속성**: 지역 최적 선택이 전역 최적의 일부
- **최적 부분 구조**: 부분 문제의 최적이 전체 최적의 일부

이걸 증명 못 하면 그리디는 틀림. 그리디가 어떤 문제엔 되고(활동 선택) 어떤 문제엔 안 되는(0/1 배낭) 이유 → [[greedy]], [[dp-fundamentals]].

## 종료 증명

정당성 ≠ 종료. 무한 루프면 정당성 무의미. 종료는 보통:

- **감소하는 양(variant/measure)**: 매 반복 줄어드는 음이 아닌 정수 (예: 남은 원소 수). 0 되면 종료 → 유한 반복 보장
- 유클리드 호제법: 나머지가 매번 줄어 → 종료

## 실무에서

- 모든 코드를 증명하진 않음 (비쌈). 하지만:
- **핵심 알고리즘**(이진 탐색 off-by-one → [[binary-search]], 동시성 → os/[[threads-and-races]])은 불변식 사고가 버그를 예방
- **형식 검증(formal verification)**: TLA+, Coq 등으로 중요 시스템 증명 (분산 프로토콜 → distributed-systems/[[consensus-problem]])
- 불변식을 주석/assert로 코드에 남기면 리뷰·디버깅에 도움

## 셀프 체크

> [!question]- "테스트는 버그의 존재만 보이고 증명은 부재를 보인다"는 말의 의미는?
> 테스트는 유한한 입력만 확인하므로 통과해도 확인 안 한 입력에서 틀릴 수 있다(존재 확인). 증명은 모든 입력에 대해 정당성을 보장한다(부재 확인). 그래서 실무는 테스트 위주라도 핵심 알고리즘은 증명으로 확신을 얻는다.

> [!question]- 루프 불변식으로 정당성을 보이는 세 단계는 무엇인가?
> 초기화(첫 반복 전 불변식이 참), 유지(한 반복이 불변식을 보존), 종료(루프 끝의 불변식 + 종료 조건이 원하는 결과를 줌)다. 이는 수학적 귀납법의 루프 버전으로 초기화=기저, 유지=귀납 단계에 대응한다.

> [!question]- 재귀 알고리즘을 강귀납법으로 증명할 때 재귀 호출을 어떻게 다루나?
> "더 작은 모든 입력에서는 이미 맞다"고 가정하고 현재 입력만 증명한다. 기저(가장 작은 경우)가 맞음을 보이고, 귀납 가정 아래 현재도 맞음을 보이면 된다. 재귀 호출의 정당성은 믿고 넘어가는 것이 핵심 사고다.

> [!question]- 그리디의 최적성을 보이는 교환 논증(exchange argument)의 골자는?
> 최적해가 그리디 선택과 다르다고 가정한 뒤, 그리디 선택으로 바꿔도 해가 더 나빠지지 않음을 보인다. 그러면 그리디도 최적이다. 이것이 성립하려면 그리디 선택 속성과 최적 부분 구조가 필요하다.

> [!question]- 정당성 증명과 종료 증명은 왜 별개이고 종료는 보통 어떻게 보이나?
> 알고리즘이 맞는 결과를 향해도 무한 루프면 정당성이 무의미하다. 종료는 보통 매 반복 줄어드는 음이 아닌 정수 measure(예: 남은 원소 수)를 제시해, 0이 되면 멈추므로 유한 반복임을 보인다(유클리드 호제법의 나머지 감소).

## 연습문제

> [!example]- 문제: 삽입 정렬의 루프 불변식을 세우고 초기화/유지/종료로 정당성을 증명하라.
> **풀이**
> 불변식: 외부 루프 i의 시작 시 부분 배열 a[0..i-1]은 정렬되어 있다(원래 원소들의 재배열).
> 초기화: i=1일 때 a[0..0]은 원소 하나뿐이라 자명하게 정렬됨.
> 유지: a[0..i-1]이 정렬됐다고 가정하면, 내부 while이 key=a[i]보다 큰 원소들을 오른쪽으로 밀고 key를 올바른 자리에 넣어 a[0..i]가 정렬됨. 다음 반복 시작 시 불변식 성립.
> 종료: i=n일 때 a[0..n-1] 전체가 정렬됨 = 원하는 결과. ∎

> [!example]- 문제: `fact(n) = n * fact(n-1)`, `fact(0) = 1`이 n!임을 귀납법으로 증명하라.
> **풀이**
> 기저: fact(0) = 1 = 0!. ✓
> 귀납 가정: fact(n-1) = (n-1)!이 성립한다고 하자.
> 귀납 단계: fact(n) = n × fact(n-1) = n × (n-1)! = n!. ✓
> 따라서 모든 n ≥ 0에서 fact(n) = n!. ∎

> [!example]- 문제: 매 반복 감소하는 measure를 제시해 유클리드 호제법 gcd(a,b)의 종료를 증명하라.
> **풀이**
> gcd(a, b)는 b=0이면 a를 반환, 아니면 gcd(b, a mod b)를 호출한다.
> measure로 두 번째 인자 b(음이 아닌 정수)를 택한다. 재귀 호출의 두 번째 인자는 a mod b이고, 0 ≤ a mod b < b이므로 measure가 매 호출마다 순감소한다.
> 음이 아닌 정수의 순감소열은 유한하므로 언젠가 b=0에 도달해 종료한다. ∎
