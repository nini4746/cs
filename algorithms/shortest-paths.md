# 최단 경로 (Shortest Paths)

## 한 줄 요약

가중 그래프에서 최소 비용 경로. 다익스트라(음수 간선 없음, 빠름), 벨만-포드(음수 허용, 느림), 플로이드-워셜(모든 쌍). 음수 간선의 존재가 알고리즘 선택을 가른다.

## 왜 필요한가

- GPS 경로, 네트워크 라우팅([[routing]])의 핵심
- 왜 음수 간선이 문제인가
- 어떤 상황에 어떤 알고리즘인가

## 다익스트라 (Dijkstra)

**음수 간선이 없을 때** 단일 출발점 최단 경로. 그리디 ([[greedy]]):

```
1. 시작점 거리 0, 나머지 ∞
2. 미확정 중 거리 최소 정점 선택 (우선순위 큐 → [[heaps]])
3. 그 정점의 이웃 거리 갱신 (relaxation)
4. 모든 정점 확정까지 반복
```

- **우선순위 큐(힙)** 사용 시 **O((V+E) log V)**
- 그리디: "가장 가까운 미확정 정점은 이미 최단" - 음수 없으면 성립
- BFS의 가중 버전 (BFS는 모든 간선 가중치 1인 특수 경우)

### 왜 음수 간선이 안 되나

다익스트라는 "확정한 정점은 더 짧아질 수 없다"고 가정. 음수 간선이 있으면 나중에 음수 경로로 더 짧아질 수 있어 **그리디 가정 붕괴**. 확정을 못 함.

## 벨만-포드 (Bellman-Ford)

**음수 간선 허용**. 모든 간선을 V-1번 완화:

```
V-1번 반복:
  모든 간선 (u,v,w): if dist[u]+w < dist[v]: dist[v] = dist[u]+w
```

- **O(VE)** - 다익스트라보다 느림
- 최단 경로는 최대 V-1개 간선 → V-1번이면 수렴
- **음수 사이클 탐지**: V-1번 후에도 갱신되면 음수 사이클 존재 (최단경로 정의 안 됨 - 무한히 줄어듦)

용도: 음수 간선 가능(환율 차익 거래, 일부 네트워크), 음수 사이클 판정.

## 플로이드-워셜 (Floyd-Warshall)

**모든 쌍** 최단 경로. DP ([[dp-fundamentals]]):

```
for k in 정점:        # k를 경유지로 허용
  for i, j:
    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

- **O(V³)**, O(V²) 공간
- "k까지 경유해서 i→j 최단" DP. k 루프가 바깥
- 작은 밀집 그래프의 모든 쌍에 간결. 음수 간선 OK (음수 사이클은 대각선으로 탐지)

## 선택 가이드

| 상황 | 알고리즘 | 복잡도 |
|---|---|---|
| 단일 출발, 음수 없음 | 다익스트라 | O((V+E) log V) |
| 단일 출발, 음수 있음 | 벨만-포드 | O(VE) |
| 모든 쌍, 작은 그래프 | 플로이드-워셜 | O(V³) |
| 무가중 | BFS | O(V+E) |
| DAG | 위상정렬 + 완화 | O(V+E) |

## relaxation (완화) - 공통 원리

셋 다 **완화(relaxation)**가 핵심: "u를 거쳐 v로 가는 게 더 짧으면 갱신":

```
if dist[u] + weight(u,v) < dist[v]:
    dist[v] = dist[u] + weight(u,v)
```

차이는 **완화 순서**: 다익스트라는 최소 거리 순(그리디), 벨만-포드는 모든 간선 반복, 플로이드는 경유지 순(DP).

## A* (실전 확장)

다익스트라 + **휴리스틱**(목표까지 추정 거리)으로 유망한 방향 우선 탐색:

- 목표가 정해진 경로 탐색(GPS, 게임)에 다익스트라보다 빠름
- 휴리스틱이 admissible(과대평가 안 함)하면 최적 보장
- 실전 길찾기의 표준

## 셀프 체크

> [!question]- 다익스트라가 음수 간선에서 왜 안 되나?
> 다익스트라는 "확정한 정점은 더 짧아질 수 없다"는 그리디 가정에 기댄다. 음수 간선이 있으면 나중에 음수 경로로 이미 확정한 정점이 더 짧아질 수 있어 가정이 붕괴한다.

> [!question]- 벨만-포드가 왜 V-1번 반복하나?
> 최단 경로는 사이클이 없으니 최대 V-1개의 간선을 가진다. 모든 간선을 V-1번 완화하면 최단 거리가 수렴한다. V-1번 후에도 갱신되면 음수 사이클이 존재한다는 뜻이다.

> [!question]- 세 알고리즘의 공통 원리와 차이는?
> 공통은 relaxation(u를 거쳐 v가 더 짧으면 갱신)이다. 차이는 완화 순서: 다익스트라는 최소 거리 순(그리디), 벨만-포드는 모든 간선 반복, 플로이드-워셜은 경유지 순(DP)이다.

> [!question]- 플로이드-워셜의 k 루프가 왜 가장 바깥에 있나?
> "k까지 경유해서 i→j 최단"이라는 DP 정의 때문이다. 경유 가능한 정점 집합을 k 하나씩 넓혀가며 모든 (i,j)를 갱신하려면 k가 가장 바깥 루프여야 한다.

> [!question]- A*가 다익스트라보다 빠른 원리와 최적성 조건은?
> 목표까지의 추정 거리(휴리스틱)를 더해 유망한 방향을 우선 탐색해 목표가 정해진 경로에서 더 빠르다. 휴리스틱이 admissible(실제보다 과대평가 안 함)하면 최적 경로를 보장한다.

## 연습문제

> [!example]- 문제: 다익스트라가 틀리는 음수 간선 그래프의 반례를 구성하라.
> **풀이**
> 정점 A→B(가중 1), A→C(가중 4), B→C(가중 -3). 출발 A.
> 다익스트라는 B(1) 확정 후 C를 min(4, 1+... )로 갱신하는데, A→C를 먼저 확정하는 순서에서 C를 4로 조기 확정하면 A→B→C=1+(-3)=-2를 놓친다.
> 음수 간선 B→C로 확정 정점이 더 짧아질 수 있어 그리디가 깨진다.

> [!example]- 문제: 벨만-포드 의사코드를 쓰고 음수 사이클 탐지 방법과 복잡도를 설명하라.
> **풀이**
> ```
> dist[start]=0, 나머지 ∞
> repeat V-1 times:
>   for each edge (u,v,w):
>     if dist[u]+w < dist[v]: dist[v]=dist[u]+w
> # 탐지: 한 번 더 돌려서 갱신되면 음수 사이클
> for each edge (u,v,w):
>   if dist[u]+w < dist[v]: report 음수 사이클
> ```
> 간선 E개를 V-1번 완화하므로 O(VE). V-1번이면 최단 경로(간선 ≤V-1개)가 수렴하는데 그 후에도 줄면 음수 사이클이다.

> [!example]- 문제: 플로이드-워셜 의사코드를 쓰고 O(V³)을 유도하라.
> **풀이**
> ```
> for k in 정점:
>   for i in 정점:
>     for j in 정점:
>       dist[i][j] = min(dist[i][j], dist[i][k]+dist[k][j])
> ```
> 삼중 루프가 각각 V번 도니 V·V·V = O(V³). 공간은 dist 행렬 O(V²).

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: 이 개념을 이해했다면 답할 수 있어야 하는 것들 - (1) relaxation이 무엇이고 세 알고리즘이 완화 순서로 어떻게 갈리는지, (2) 음수 간선의 존재가 왜 다익스트라를 깨고 벨만-포드가 어떻게 감당하는지, (3) 상황별(단일/모든 쌍, 음수 유무, DAG) 알고리즘 선택과 복잡도.
