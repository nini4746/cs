# 최단 경로 (Shortest Paths)

## 한 줄 요약

가중 그래프에서 최소 비용 경로. 다익스트라(음수 간선 없음, 빠름), 벨만-포드(음수 허용, 느림), 플로이드-워셜(모든 쌍). 음수 간선의 존재가 알고리즘 선택을 가른다.

## 왜 필요한가

- GPS 경로, 네트워크 라우팅([[routing]])의 핵심
- 왜 음수 간선이 문제인가
- 어떤 상황에 어떤 알고리즘인가

## 다익스트라 (Dijkstra)

**음수 간선이 없을 때** 단일 출발점 최단 경로. 그리디 ([[greedy]]):

```
1. 시작점 거리 0, 나머지 ∞
2. 미확정 중 거리 최소 정점 선택 (우선순위 큐 → [[heaps]])
3. 그 정점의 이웃 거리 갱신 (relaxation)
4. 모든 정점 확정까지 반복
```

- **우선순위 큐(힙)** 사용 시 **O((V+E) log V)**
- 그리디: "가장 가까운 미확정 정점은 이미 최단" - 음수 없으면 성립
- BFS의 가중 버전 (BFS는 모든 간선 가중치 1인 특수 경우)

### 왜 음수 간선이 안 되나

다익스트라는 "확정한 정점은 더 짧아질 수 없다"고 가정. 음수 간선이 있으면 나중에 음수 경로로 더 짧아질 수 있어 **그리디 가정 붕괴**. 확정을 못 함.

## 벨만-포드 (Bellman-Ford)

**음수 간선 허용**. 모든 간선을 V-1번 완화:

```
V-1번 반복:
  모든 간선 (u,v,w): if dist[u]+w < dist[v]: dist[v] = dist[u]+w
```

- **O(VE)** - 다익스트라보다 느림
- 최단 경로는 최대 V-1개 간선 → V-1번이면 수렴
- **음수 사이클 탐지**: V-1번 후에도 갱신되면 음수 사이클 존재 (최단경로 정의 안 됨 - 무한히 줄어듦)

용도: 음수 간선 가능(환율 차익 거래, 일부 네트워크), 음수 사이클 판정.

## 플로이드-워셜 (Floyd-Warshall)

**모든 쌍** 최단 경로. DP ([[dp-fundamentals]]):

```
for k in 정점:        # k를 경유지로 허용
  for i, j:
    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

- **O(V³)**, O(V²) 공간
- "k까지 경유해서 i→j 최단" DP. k 루프가 바깥
- 작은 밀집 그래프의 모든 쌍에 간결. 음수 간선 OK (음수 사이클은 대각선으로 탐지)

## 선택 가이드

| 상황 | 알고리즘 | 복잡도 |
|---|---|---|
| 단일 출발, 음수 없음 | 다익스트라 | O((V+E) log V) |
| 단일 출발, 음수 있음 | 벨만-포드 | O(VE) |
| 모든 쌍, 작은 그래프 | 플로이드-워셜 | O(V³) |
| 무가중 | BFS | O(V+E) |
| DAG | 위상정렬 + 완화 | O(V+E) |

## relaxation (완화) - 공통 원리

셋 다 **완화(relaxation)**가 핵심: "u를 거쳐 v로 가는 게 더 짧으면 갱신":

```
if dist[u] + weight(u,v) < dist[v]:
    dist[v] = dist[u] + weight(u,v)
```

차이는 **완화 순서**: 다익스트라는 최소 거리 순(그리디), 벨만-포드는 모든 간선 반복, 플로이드는 경유지 순(DP).

## A* (실전 확장)

다익스트라 + **휴리스틱**(목표까지 추정 거리)으로 유망한 방향 우선 탐색:

- 목표가 정해진 경로 탐색(GPS, 게임)에 다익스트라보다 빠름
- 휴리스틱이 admissible(과대평가 안 함)하면 최적 보장
- 실전 길찾기의 표준

## 연결

- 우선순위 큐 → data-structures/[[heaps]]
- 그리디 정당성 → [[greedy]]
- DP (플로이드) → [[dp-fundamentals]]
- BFS (무가중) → [[graph-traversal]]
- 네트워크 라우팅 → network/[[routing]]
