# DP 최적화 (DP Optimization)

## 한 줄 요약

DP의 시간·공간을 줄이는 기법들. 공간 압축(직전 행만 유지), 비트마스크 상태, 그리고 전이를 빠르게 하는 고급 기법(단조 큐, 분할 정복 최적화). 상태 수와 전이 비용을 각각 공략한다.

## 왜 필요한가

- DP 표가 메모리 초과할 때
- 전이가 O(n)이라 전체가 느릴 때
- 상태를 집합으로 표현해야 할 때 (비트마스크)

## 공간 압축 (rolling array)

많은 DP가 **직전 몇 행만** 참조. 전체 표 대신 그만큼만 유지:

```
// 0/1 배낭: dp[i]는 dp[i-1]만 참조 → 1차원으로
for (물건 i)
    for (w = W; w >= wt[i]; w--)   // 역순! (덮어쓰기 방지)
        dp[w] = max(dp[w], dp[w-wt[i]] + val[i]);
```

- O(nW) 공간 → **O(W)**. [[dp-patterns]]의 배낭이 실전에서 1차원인 이유
- 역순 순회가 핵심: 정순이면 같은 물건을 여러 번 쓰게 됨 (그건 무한 배낭)
- 피보나치도 O(n)→O(1) ([[dp-fundamentals]])

주의: 경로 복원(실제 선택 추적)이 필요하면 전체 표 유지 필요. 값만 필요할 때 압축.

## 비트마스크 DP

상태가 **집합**일 때 정수의 비트로 표현:

```
상태: dp[mask] = 방문 집합이 mask일 때의 답
     mask의 i번째 비트 = i가 집합에 있나
```

- **외판원 TSP**: dp[mask][i] = mask 도시들을 방문하고 i에 있을 때 최소 비용. O(2ⁿ·n²)
- n≤20 정도까지 (2ⁿ 상태)
- 비트 연산으로 부분집합 순회 ([[bits-and-integers]])

지수지만 순진한 순열(n!)보다 훨씬 나음 (TSP: 20! vs 2²⁰·20²).

## 전이 최적화 (고급)

DP 복잡도 = 상태 수 × 전이 비용. 전이가 비싸면 그걸 공략:

### 단조 큐 최적화 (monotonic queue)

전이가 "구간 내 최소/최대"면 단조 큐(덱)로 O(1) 상각:

```
dp[i] = min(dp[j]) for j in [i-k, i-1]  + cost
→ 슬라이딩 윈도우 최소를 덱으로 O(1) → 전체 O(n)
```

- 슬라이딩 윈도우 최댓값과 같은 기법 ([[stacks-and-queues]]의 덱)

### 분할 정복 최적화 / Knuth 최적화

전이가 특정 단조성(quadrangle inequality)을 만족하면 O(n²k)→O(nk log n) 등으로. 최적 분할점이 단조로 이동하는 성질 이용. 고급 경쟁 프로그래밍.

### 볼록성 (Convex Hull Trick)

전이가 선형 함수의 최솟값이면 선분들의 하한 포락선으로 O(n) 또는 O(n log n). DP 전이를 기하 문제로.

## 최적화 선택 가이드

| 문제 | 기법 |
|---|---|
| 메모리 초과, 직전 행만 참조 | 공간 압축 |
| 상태가 집합 (n 작음) | 비트마스크 |
| 전이 = 구간 min/max | 단조 큐 |
| 전이 = 선형함수 min | CHT |
| 최적 분할점 단조 | 분할정복/Knuth |

먼저 기본 DP를 세우고([[dp-fundamentals]]), 복잡도가 부족하면 병목(상태 수? 전이?)을 보고 해당 기법 적용.

## 실무 관점

- 대부분 실무 DP는 공간 압축이면 충분
- 고급 전이 최적화는 경쟁 프로그래밍/특수 상황
- 상태 정의가 잘못되면 어떤 최적화도 소용없음 - 상태 설계가 먼저

## 셀프 체크

> [!question]- 0/1 배낭을 1차원으로 압축할 때 무게를 역순으로 순회해야 하는 이유는?
> 정순으로 돌면 방금 갱신한 dp[w-wt[i]](같은 물건을 이미 쓴 상태)를 다시 참조해 한 물건을 여러 번 넣게 되는데, 그건 무한 배낭이 된다. 역순으로 돌면 dp[w-wt[i]]가 아직 이전 물건 기준(dp[i-1]) 값이라 각 물건을 최대 한 번만 쓰는 0/1 배낭이 유지된다.

> [!question]- 공간 압축을 항상 쓸 수 없는 경우는 언제인가?
> 실제로 어떤 선택을 했는지 경로 복원이 필요할 때다. 직전 행만 남기면 지나온 결정 이력이 사라지므로 값만 필요할 때 압축할 수 있고, 경로를 추적하려면 전체 표를 유지해야 한다.

> [!question]- 비트마스크 DP가 TSP를 순열 완전탐색보다 낫게 만드는 이유는?
> 방문 집합을 정수 비트로 표현해 dp[mask][i]로 상태를 정의하면 같은 집합·같은 끝점의 부분 문제를 재사용할 수 있기 때문이다. n! 순열을 다 세는 대신 2ⁿ·n² 상태로 줄어든다(예: 20! vs 2²⁰·20²). 여전히 지수지만 훨씬 작다.

> [!question]- 전이 최적화 기법을 고를 때 무엇을 먼저 파악해야 하는가?
> 병목이 상태 수인지 전이 비용인지다. DP 복잡도는 상태 수 × 전이 비용이므로, 전이가 "구간 min/max"면 단조 큐, "선형 함수의 min"이면 CHT, "최적 분할점이 단조"면 분할정복/Knuth를 쓴다. 기본 DP를 먼저 세우고 병목을 확인한 뒤 해당 기법을 붙인다.

## 연습문제

> [!example]- 문제: 0/1 배낭의 1차원 압축 루프를 직접 쓰고 순회 방향이 정확성에 미치는 영향을 설명하라
> **풀이**
> ```
> for (물건 i)
>     for (w = W; w >= wt[i]; w--)   // 역순
>         dp[w] = max(dp[w], dp[w-wt[i]] + val[i]);
> ```
> 역순이라 dp[w-wt[i]]는 아직 물건 i를 반영하지 않은 값(이전 행)이라 물건 i를 최대 한 번만 쓴다.
> 만약 w를 정순으로 돌리면 dp[w-wt[i]]가 이미 물건 i를 쓴 값이 되어 같은 물건을 반복 사용, 무한 배낭 답이 나온다.
> 공간은 O(nW)에서 O(W)로 줄어든다.

> [!example]- 문제: dp[i]=min(dp[j]) (j∈[i-k,i-1]) + cost 전이의 복잡도를 단조 큐로 개선하고 복잡도를 분석하라
> **풀이**
> 순진하게는 각 i마다 폭 k 구간의 최소를 훑어 O(nk).
> 슬라이딩 윈도우 최소를 덱(단조 큐)으로 유지한다: 새 원소를 넣을 때 뒤쪽에서 자신보다 큰 값을 제거하고, 창을 벗어난 앞쪽 인덱스를 제거한다. 덱의 앞이 항상 구간 최소.
> 각 인덱스가 덱에 최대 한 번 들어가고 한 번 나오므로 상각 O(1), 전체 전이가 O(n).
> 결과적으로 O(nk) → O(n).

> [!example]- 문제: 상태가 "방문한 도시 집합"인 TSP의 비트마스크 상태와 복잡도를 설계하라
> **풀이**
> 상태: dp[mask][i] = mask에 속한 도시들을 방문했고 현재 i에 있을 때의 최소 비용. mask의 i번째 비트가 도시 i 방문 여부.
> 전이: dp[mask][i] = min over j∈mask, j≠i ( dp[mask without i][j] + dist[j][i] ).
> 상태 수는 2ⁿ개 mask × n개 끝점 = 2ⁿ·n, 전이마다 이전 도시 j를 O(n) 훑음 → 전체 O(2ⁿ·n²).
> n≤20 정도까지 실용적이며, 순열 완전탐색 n!보다 훨씬 작다.
