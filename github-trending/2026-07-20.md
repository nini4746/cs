오늘 트렌딩은 "AI 코딩 도구 생태계 자체"가 주제인 하루입니다. 대형 코드베이스를 통째로 읽지 않고도 AI가 필요한 부분만 보게 만드는 컨텍스트 그래프(code-review-graph), 여러 LLM 제공자를 하나의 엔드포인트로 묶는 게이트웨이(OmniRoute), 코딩 에이전트 자체를 표방하는 하네스(jcode), 페르소나별로 역할을 나눈 에이전트 팀(agency-agents)까지, 에이전트를 "어떻게 더 잘 쓰게 만들 것인가"에 대한 인프라 경쟁이 상위권을 채웠습니다. 그 사이에 실전 AI 엔지니어링을 처음부터 가르치는 학습 자료(ai-engineering-from-scratch)가 함께 오른 것도, 도구는 빠르게 늘어나는데 이를 제대로 다루는 법을 배우려는 수요가 그만큼 크다는 신호로 보입니다.

### swapnil404/mineSQL

- <https://github.com/swapnil404/mineSQL>
- 언어: Go
- 스타: 33 (2026-06-04 생성)

마인크래프트 월드를 실제 저장 매체로 쓰는 PostgreSQL 와이어 호환 관계형 데이터베이스 엔진으로, 농담처럼 보이지만 Go로 SQL 파서·쿼리 실행기·WAL·MVCC를 직접 구현한 진짜 DB 엔진입니다. 테이블은 Z좌표에, 행은 잔디 위 블록 줄무늬에 매핑되고 배너 문양으로 정수·불리언을, 표지판으로 텍스트를 인코딩하며, WAL은 서쪽으로 갈수록 오래된 트랜잭션이 쌓이는 식으로 시각화됩니다. Wire Protocol → Parser → Executor → Storage → WAL → (Paper 플러그인으로 연결되는) HAL 계층 구조를 갖추고 있어 `psql` 같은 표준 Postgres 클라이언트가 그대로 붙는다는 점에서, MVCC와 스토리지 엔진의 원리를 기발한 방식으로 체감해볼 수 있는 프로젝트입니다. 다만 테스트 커버리지나 벤치마크 근거는 README에 나와 있지 않아, 진지한 DB 학습보다는 개념을 재미있게 익히는 보조 자료로 보는 편이 맞습니다.
