오늘 트렌딩은 "에이전트를 어떻게 더 잘 다룰 것인가"라는 주제가 여러 방향에서 동시에 터진 하루입니다. AI 에이전트 설계 원리를 정리한 오픈소스 교재(ai-agent-book)가 하루 만에 4천 개 넘는 스타를 얻으며 압도적 1위로 올라섰고, 코딩 에이전트가 전체 코드베이스를 다시 읽지 않도록 구조 그래프를 제공하는 도구(code-review-graph), 에이전트의 답변 스타일 자체를 바꾸는 스킬(i-have-adhd), 배포 인프라를 통째로 자동화하는 플랫폼(openship)까지 모두 "에이전트를 얼마나 신뢰하고 얼마나 도와줄 것인가"라는 질문으로 수렴합니다. 여기에 500개 넘는 뉴스 피드를 하나의 대시보드로 모으는 지정학 모니터링 프로젝트(worldmonitor)가 함께 오르면서, 정보 과부하를 다루는 문제의식이 코드뿐 아니라 세계 정세 파악에도 번지고 있음을 보여줍니다.

## bojieli/ai-agent-book

- <https://github.com/bojieli/ai-agent-book>
- 언어: Python
- 오늘 스타: 4,434 (총 13,778)

"AI Agent = LLM + 컨텍스트 + 도구"라는 공식을 중심으로 컨텍스트 엔지니어링, 메모리 시스템, 도구 통합, 코딩 에이전트, 평가, 멀티 에이전트 협업까지 다루는 오픈소스 교재입니다. 10개 장과 88개의 실습(그중 70여 개는 독립 실행 가능)으로 구성되어 있고 중국어·영어·베트남어 등 5개 언어판을 함께 제공합니다. 하루 만에 4천 개가 넘는 스타가 붙은 것은 "에이전트를 만드는 법"에 대한 체계적인 학습 자료 수요가 그만큼 크다는 뜻으로 보이며, 개념을 이론으로만 두지 않고 실행 가능한 실습으로 짝지은 구성이 특히 공부하는 입장에서 참고할 만합니다.

## tirth8205/code-review-graph

- <https://github.com/tirth8205/code-review-graph>

- 오늘 스타: 1,921 (총 24,299)

SQLite에 로컬로 그래프를 저장하고 Leiden 알고리즘으로 커뮤니티(모듈) 구조를 탐지하며, MCP 서버로 30개의 툴을 노출해 Cursor·Claude Code 등에서 바로 쓸 수 있게 만든 구조가 핵심입니다. 질문당 토큰 소비를 평균 82배 줄였다고 밝히는데, "에이전트에게 코드베이스를 어떻게 압축해서 보여줄 것인가"라는 문제를 그래프 데이터베이스와 정적 분석으로 푼 사례로 공부할 가치가 있습니다.

- 언어: Rust
- 스타: 88
- 생성일: 2026-03-11

AI가 생성한 TypeScript 코드를 안전하게 실행하기 위한 인터프리터로, oxc_parser로 타입을 벗겨낸 뒤 자체 IR(ZapcodeIR)을 거쳐 약 50개 명령어짜리 스택 기반 바이트코드로 컴파일하고, 외부 호출 시점마다 실행 상태를 스냅샷으로 저장했다가 나중에 재개할 수 있는 VM으로 실행합니다. ### syunnPC/qemu-system-ia64

- <https://github.com/syunnPC/qemu-system-ia64>
- 언어: C
- 스타: 39
- 생성일: 2026-05-28

이제는 단종된 IA-64(Itanium) 아키텍처를 위한 실험적 QEMU 풀시스템 에뮬레이터로, TCG 변환·레지스터 스택 엔진·TLB를 갖춘 Itanium 2 CPU 모델과 SCSI·AHCI·e1000 이더넷 등 주변장치를 갖춘 `ia64-vpc` 머신 프로파일을 구현했습니다. 900개의 아키텍처 마이크로프로그램으로 구성된 TCG 레지스트리와 유닛·머신·기능 테스트 스위트를 갖추고 있고, 실제로 Itanium용 Windows Server 2008을 부팅시키는 데 성공했다고 밝히고 있어 장난감 수준이 아니라 진지한 저수준 구현임을 보여줍니다. LLM의 도움을 받아 만들어졌다고 명시하면서도 명령어 커버리지나 부동소수점 예외 처리는 아직 검증이 더 필요하다고 솔직히 밝히는 점도, CPU 에뮬레이터를 밑바닥부터 만드는 과정을 공부하기에 좋은 참고 자료입니다.

### Tako-Research/TakoVM

- <https://github.com/Tako-Research/TakoVM>
- 언어: Python
- 스타: 82
- 생성일: 2026-01-25

AI 에이전트가 생성한 코드를 격리된 환경에서 실행하기 위한 보안 실행 플랫폼으로, gVisor(유저스페이스 커널)·잡 단위 Docker 컨테이너·seccomp 필터링·`--network=none` 네트워크 격리·capability 제거를 층층이 쌓은 구조가 특징입니다. Redis나 Celery 같은 외부 의존성 없이 자체 워커 풀로 비동기 잡 큐를 구현했고, PostgreSQL에 실행 기록과 stdout/stderr을 남겨 잡을 재현(replay)할 수 있게 해둔 점도 실무적입니다. "에이전트가 짠 코드를 어떤 격리 계층으로 실행할 것인가"라는 질문에 다층 방어(defense in depth)로 답한 사례로, 오늘 트렌딩에 오른 zapcode와 나란히 놓고 비교하며 보면 좋은 프로젝트입니다.
