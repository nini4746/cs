오늘 트렌딩 상위권은 확실히 "AI 에이전트" 쪽으로 쏠려 있습니다. 에이전트가 참고할 스킬 저장소(agent-skills, superpowers, last30days-skill), 에이전트가 실무 도구를 다루게 해주는 어댑터(OfficeCLI), 에이전트의 장기 기억을 담당하는 인프라(TencentDB-Agent-Memory)까지, "LLM 에이전트를 더 똑똑하고 쓸모 있게 만드는 레이어"가 오늘 트렌딩의 공통 테마로 보입니다. 그 사이사이에 시스템 프롬프트 리버스 엔지니어링, 물리 신호(WiFi) 기반 센싱, 경량 벡터DB처럼 성격이 다른 프로젝트들도 섞여 있어서, AI 도구 자체보다 그 주변 생태계(메모리, 스킬, 통합)가 빠르게 채워지고 있다는 인상을 받았습니다.

## addyosmani/agent-skills

- <https://github.com/addyosmani/agent-skills>
- 언어: JavaScript
- 오늘 스타: 1,322

Google Chrome 팀의 Addy Osmani가 만든, AI 코딩 에이전트용 "프로덕션급 엔지니어링 스킬" 모음입니다. Claude Code 같은 에이전트에 특정 작업 방식(코드 리뷰 기준, 리팩토링 절차 등)을 스킬 형태로 주입해 결과물의 품질을 실무 수준으로 끌어올리는 접근으로 보입니다. 이런 "스킬 파일로 에이전트 행동을 표준화한다"는 패턴 자체가 최근 에이전트 생태계에서 빠르게 자리잡고 있는 관용구라서, 팀 단위로 에이전트를 쓸 때 프롬프트를 어떻게 구조화하면 좋은지 참고할 만합니다.

## iOfficeAI/OfficeCLI

- <https://github.com/iOfficeAI/OfficeCLI>
- 언어: C#
- 오늘 스타: 1,712

AI 에이전트가 Word, Excel, PowerPoint 파일을 직접 다룰 수 있게 해주는 오피스 스위트/CLI입니다. LLM이 문서 조작 API를 텍스트 명령으로 호출하도록 감싸는 구조로 추정되며, "에이전트가 실제 오피스 문서를 읽고 편집하게 만든다"는 실용적인 니즈를 정면으로 겨냥합니다. 오늘 트렌딩 중 스타 증가폭이 가장 큰 저장소로, 사무 자동화 영역에서 에이전트 활용 수요가 크다는 걸 보여줍니다.

## TencentCloud/TencentDB-Agent-Memory

- <https://github.com/TencentCloud/TencentDB-Agent-Memory>
- 언어: TypeScript
- 오늘 스타: 351

AI 에이전트를 위한 완전 로컬 장기 기억(long-term memory) 시스템으로, 4단계(4-tier) 점진적 파이프라인을 통해 메모리를 관리한다고 소개합니다. 에이전트가 세션을 넘어 정보를 기억하게 하려면 결국 별도의 메모리 저장/검색 계층이 필요한데, 이를 계층화된 파이프라인으로 설계한 사례로 볼 수 있습니다. RAG나 벡터DB 기반 메모리와 비교해 어떤 트레이드오프를 택했는지 뜯어보면 공부가 될 것 같습니다.

### ajitpratap0/GoSQLX

- <https://github.com/ajitpratap0/GoSQLX>
- 언어: Go
- 스타: 105
- 생성일: 2025-02-17

PostgreSQL, MySQL, MariaDB, SQL Server, Oracle, SQLite, Snowflake, ClickHouse 등 여러 SQL 방언을 하나의 엔진으로 토크나이즈하고 AST까지 생성해주는 Go SDK입니다. 제로카피 토크나이징과 sync.Pool 기반 객체 재사용, 재귀 하강(recursive descent) 파서를 조합해 초당 138만 건 이상의 처리량을 낸다고 밝히고 있으며, ORM이 아니라 파싱된 AST 자체를 반환해 상위 도구가 자유롭게 활용하도록 설계했습니다. SQL-99 문법의 약 85%를 커버하고, 레이스 컨디션 검증을 포함한 테스트, 30개의 린트 규칙과 20개의 옵티마이저 규칙, VS Code 확장과 MCP 서버까지 갖춰놓은 걸 보면 개인 프로젝트치고 상당히 프로덕션 지향적입니다.
