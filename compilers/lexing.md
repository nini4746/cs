# 렉싱 (Lexing / Tokenization)

## 한 줄 요약

소스 문자 스트림을 의미 있는 토큰(숫자, 연산자, 키워드)으로 쪼개는 컴파일러 첫 단계. 토큰은 정규 언어라서 렉서는 본질적으로 DFA다. 파서가 문자가 아닌 토큰을 다루게 해준다.

## 왜 필요한가

- 컴파일러/인터프리터의 첫 단계
- 왜 렉서가 오토마타(DFA)인가 → automata/[[dfa-nfa]]
- 파싱을 단순하게 만드는 전처리

## 역할: 문자 → 토큰

렉서(scanner/tokenizer)는 **원시 문자열을 토큰 열로**:

```
"12 + 34 * (5 - 6)"
→ [NUM(12), OP(+), NUM(34), OP(*), OP((), NUM(5), OP(-), NUM(6), OP())]
```

실측 (간단한 산술식 렉서):
```
[('NUM','12'), ('OP','+'), ('NUM','34'), ('OP','*'), ('OP','('),
 ('NUM','5'), ('OP','-'), ('NUM','6'), ('OP',')')]
```

- 공백·주석 버림
- 숫자 여러 글자를 하나의 NUM 토큰으로 묶음
- 각 토큰 = (종류, 값) - 파서가 다루기 쉬운 단위

## 토큰 종류

- **리터럴**: 숫자(123, 3.14), 문자열("hi"), 불리언
- **식별자**: 변수·함수 이름 (foo, count)
- **키워드**: 예약어 (if, while, return) - 식별자와 구분
- **연산자/구두점**: `+ - * / ( ) { } ; ,`
- 각 토큰에 위치 정보(줄, 열) 첨부 → 오류 메시지용

## 왜 렉서는 DFA인가

토큰의 패턴은 **정규 언어** (automata/[[dfa-nfa]], [[regex-and-finite-automata]]):

- 숫자 = `[0-9]+`, 식별자 = `[a-zA-Z_][a-zA-Z0-9_]*`, 연산자 = 고정 문자
- 이들은 전부 정규식으로 표현 → DFA로 인식
- 렉서 = 각 문자를 읽으며 상태 전이하는 유한 오토마타

```
숫자 인식 DFA: 시작 →(digit)→ 숫자상태 →(digit, 루프)→ ... →(non-digit)→ 토큰 완성
```

렉서 생성기(lex/flex)는 정규식 명세를 받아 **DFA를 자동 생성** → automata/[[regex-and-finite-automata]]의 정규식→NFA→DFA. 계산 이론이 실무 도구가 되는 지점.

## 최장 일치 (maximal munch)

렉서의 핵심 규칙: **가능한 가장 긴 토큰을 취함**:

```
"123" → NUM(123)  (1, 12, 123 중 가장 긴 것)
">=" → GTE 하나  (> 와 = 로 안 쪼갬)
"foreach" → 식별자 하나  (for + each 아님)
```

DFA가 계속 진행하다 더 못 가면(non-accepting) 마지막 수락 지점까지가 토큰. 이 규칙이 대부분 애매함을 해결.

## 키워드 vs 식별자

`if`, `while`은 식별자 패턴에도 맞음. 처리:
- 식별자로 렉싱한 뒤 **키워드 테이블 조회** → 키워드면 키워드 토큰으로
- 또는 렉서 DFA에 키워드를 직접 인코딩 (상태 폭발)
- 보통 전자 (식별자 인식 후 해시 조회 → data-structures/[[hash-tables]])

## 렉서가 처리하는 성가신 것들

- **공백·주석**: 버림 (하지만 위치 추적은 유지)
- **문자열 이스케이프**: `"a\"b"`의 `\"` 처리
- **숫자 형식**: 정수, 실수(computer-architecture/[[floating-point]]), 16진수, 지수
- **오류**: 알 수 없는 문자를 위치와 함께 보고
- **문맥 의존**: 일부 언어는 렉서가 파서 정보 필요 (C의 typedef, 나머지는 렉서/파서 분리 깔끔)

## 파이프라인에서의 위치

```
소스 문자열
  → [렉서: DFA]        ← 여기 (문자 → 토큰)
  → [파서: PDA]        → compilers/[[parsing]]
  → AST
  → ...
```

렉서(정규/DFA)와 파서(문맥자유/PDA)의 분리 = automata 계층의 실무 반영. 렉서가 문자 잡음을 제거해 파서가 구조에 집중.

## 연결

- 렉서 = DFA → automata/[[dfa-nfa]], [[regex-and-finite-automata]]
- 다음 단계 파싱 → [[parsing]]
- 키워드 조회 → data-structures/[[hash-tables]]
- 숫자 리터럴 → computer-architecture/[[floating-point]], [[bits-and-integers]]
