# 파싱 (Parsing)

## 한 줄 요약

토큰 열을 문법에 따라 구조(파스 트리/AST)로 조립하는 단계. 재귀 하강 파서는 문법 규칙을 함수로 직접 번역해 손으로 짜기 쉽고, 문법 계층으로 연산자 우선순위를 자연스럽게 표현한다.

## 왜 필요한가

- 토큰([[lexing]])을 구조로 - 컴파일러의 뼈대
- 연산자 우선순위·결합을 어떻게 처리하나
- 파서가 왜 PDA인가 → automata/[[pushdown-automata]]

## 역할: 토큰 → 구조

렉서가 준 토큰 열을 **문법에 맞는 트리**로 ([[lexing]] 다음 단계):

```
[NUM(2), OP(+), NUM(3), OP(*), NUM(4)]
→        +
        / \
       2   *
          / \
         3   4
```

이 트리 구조가 "곱셈 먼저"를 표현. 파서 = 문맥 자유 문법([[cfg-to-parsing]]) 인식 = PDA ([[pushdown-automata]]).

## 재귀 하강 파서 (recursive descent)

각 문법 규칙([[context-free-grammars]])을 **함수로**:

```
expr   → term (('+' | '-') term)*      # 덧셈
term   → factor (('*' | '/') factor)*  # 곱셈
factor → '(' expr ')' | NUM            # 괄호 or 숫자
```

각 비단말이 함수. 하향식 LL 파싱 ([[cfg-to-parsing]]). 손으로 짜기 직관적.

## 연산자 우선순위 - 문법 계층으로

핵심 기법: **우선순위를 문법의 깊이로** 인코딩:

- `expr`(덧셈)이 `term`(곱셈)을 호출 → 곱셈이 **더 깊음** → 먼저 묶임
- 깊은 규칙 = 높은 우선순위

실측 (재귀 하강 파서 + 평가):
```
2+3*4    = 14    ← 곱셈 먼저 (3*4=12, +2)
(2+3)*4  = 20    ← 괄호가 뒤집음
10-2-3   = 5     ← 좌결합 ((10-2)-3, 5 아님)
2*3+4*5  = 26    ← 6+20
```

우선순위, 괄호, 좌결합 전부 정확. 문법 계층 하나로 표현.

### 좌결합 처리

`10-2-3`은 `(10-2)-3`(좌결합)이어야 함. `while` 루프로:
```
while (다음이 +/-): 왼쪽에 계속 붙임   # 좌결합
```
재귀로 하면 우결합이 되므로 반복 루프 사용. 우결합(거듭제곱 `**`)은 재귀로.

## 좌재귀 문제

재귀 하강(LL)은 **좌재귀 문법을 못 씀**:
```
expr → expr '+' term    # 무한 재귀! expr() 부르면 또 expr() 첫 줄
```
- expr()이 즉시 expr()을 호출 → 스택 오버플로
- 해결: 반복(위 while)이나 문법 변형 → automata/[[cfg-to-parsing]]
- LR 파서는 좌재귀 OK (상향식)

## 파서 종류 (복습)

automata/[[cfg-to-parsing]]에서 다룬 것:

- **재귀 하강 (LL)**: 손으로 짜기 쉬움, 좌재귀 불가. 많은 실전 파서
- **파서 생성기 (LR/LALR)**: yacc/bison, 강력한 문법, 좌재귀 OK
- **파서 콤비네이터**: 함수형, 작은 파서 조합 ([[functional-programming]])
- **PEG/packrat**: 순서 있는 선택, 메모이제이션

## Pratt 파싱 (연산자 우선순위 파싱)

재귀 하강 + 우선순위를 우아하게: 각 토큰에 **바인딩 파워**(우선순위)를 부여해 우선순위 테이블로 처리. 많은 우선순위 레벨을 문법 계층 없이 깔끔하게. 실전 파서(많은 언어)가 채택.

## 에러 처리

- **에러 복구**: 하나 틀려도 계속 파싱해 여러 오류 보고 (동기화 지점까지 스킵)
- **좋은 메시지**: 위치([[lexing]]의 줄/열) + "무엇을 기대했나"
- 컴파일러 UX의 큰 부분

## 파이프라인

```
소스 → [렉서: DFA] → 토큰 → [파서: PDA] → AST → compilers/[[ast-and-interpretation]]
```

파서 출력 = **AST**(추상 구문 트리) - 파스 트리에서 불필요한 것(괄호 노드 등) 제거한 핵심 구조 → [[ast-and-interpretation]].

## 셀프 체크

> [!question]- 재귀 하강 파서는 문법 규칙을 어떻게 코드로 옮기나?
> 각 비단말(문법 규칙)을 하나의 함수로 번역한다. `expr → term (('+'|'-') term)*` 같은 규칙은 `expr()` 함수가 되고, 그 안에서 `term()`을 호출한다. 하향식 LL 파싱이며 손으로 짜기 직관적이다.

> [!question]- 연산자 우선순위를 재귀 하강 파서에서 어떻게 표현하나? 왜 곱셈이 덧셈보다 먼저 묶이나?
> 우선순위를 문법의 깊이로 인코딩한다. `expr`(덧셈)이 `term`(곱셈)을 호출하고 `term`이 `factor`를 호출하는 계층에서, 더 깊은 규칙일수록 높은 우선순위다. 곱셈 규칙이 더 깊이 있어 먼저 트리로 묶이므로 곱셈이 먼저 계산된다.

> [!question]- 좌결합(`10-2-3` = `(10-2)-3`)은 왜 재귀가 아니라 while 루프로 처리하나?
> 재귀로 하위 표현식을 오른쪽으로 계속 붙이면 우결합(`10-(2-3)`)이 되어버린다. while 루프로 왼쪽 결과에 다음 항을 계속 붙이면 좌결합이 된다. 반대로 거듭제곱 `**` 같은 우결합 연산자는 재귀로 처리한다.

> [!question]- 재귀 하강(LL) 파서가 좌재귀 문법을 못 쓰는 이유는?
> `expr → expr '+' term`처럼 규칙 첫 심볼이 자기 자신이면, `expr()`이 즉시 아무 입력도 소비하지 않고 `expr()`을 다시 호출해 무한 재귀(스택 오버플로)에 빠진다. 반복 루프로 바꾸거나 문법을 변형해야 한다. LR 파서는 상향식이라 좌재귀가 문제없다.

> [!question]- Pratt 파싱은 재귀 하강과 무엇이 다른가?
> 각 토큰에 바인딩 파워(우선순위 값)를 부여해 우선순위 테이블로 처리한다. 많은 우선순위 레벨을 문법 계층을 깊게 쌓지 않고 깔끔하게 다룰 수 있어 실전 파서가 많이 채택한다.

## 연습문제

> [!example]- 문제: 아래 문법으로 `2+3*4`를 재귀 하강 파서가 파싱하는 과정을 함수 호출 순서로 추적하고, 결과 AST를 그려라. `expr → term (('+'|'-') term)*`, `term → factor (('*'|'/') factor)*`, `factor → '(' expr ')' | NUM`.
> **풀이**
> - `expr()` 진입 → `term()` 호출
> - `term()` → `factor()` 호출 → NUM(2) 반환. 다음 토큰이 `+`(곱셈 아님) → `term()`은 2만 반환
> - `expr()`: 다음이 `+` → 루프 진입, 다시 `term()` 호출
> - `term()` → `factor()` → NUM(3). 다음이 `*` → 루프 진입, `factor()` → NUM(4). `3*4`를 `*`(3,4)로 묶어 반환
> - `expr()`: 왼쪽 2에 `+`(3*4) 결과를 붙임
> AST:
> ```
>     +
>    / \
>   2   *
>      / \
>     3   4
> ```
> `term()`이 `*`를 자기 안에서 먼저 소비하므로 곱셈이 더 깊이 묶인다.

> [!example]- 문제: `10-2-3`이 좌결합으로 파싱되도록 `expr()`의 while 루프를 의사코드로 작성하고, 실행을 추적해 결과가 5임을 보여라.
> **풀이**
> ```
> def expr():
>     left = term()                 # 10
>     while 다음이 '+' 또는 '-':
>         op = 소비()
>         right = term()
>         left = BinOp(op, left, right)  # 왼쪽에 누적
>     return left
> ```
> 추적: `left=10` → 루프1: op=`-`, right=`2`, `left = (10-2)` → 루프2: op=`-`, right=`3`, `left = ((10-2)-3)`. 평가 = `(8)-3 = 5`. 재귀였다면 `10-(2-3)=11`이 됐을 것 - while 누적이 좌결합을 보장한다.

## 파인만

> [!note]- 백지에 "우선순위와 결합성을 재귀 하강 파서 하나로 어떻게 다 표현하는가"를 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 우선순위 = 문법 계층의 깊이(호출 관계)로 인코딩된다는 것, (2) 좌결합은 while 루프, 우결합은 재귀라는 것, (3) 좌재귀가 왜 무한 재귀를 일으키는지 - 이 셋을 예시와 함께 설명할 수 있어야 한다.
