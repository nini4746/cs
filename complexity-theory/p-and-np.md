# P와 NP (P and NP)

## 한 줄 요약

P는 결정적 튜링 머신이 다항 시간에 **푸는** 문제, NP는 답(증명서)이 주어지면 다항 시간에 **검증**하는 문제. NP는 비결정적 다항 시간과 검증자(verifier) 두 관점이 동치이며, "찾기가 확인만큼 쉬운가"라는 P=NP 질문이 이론 컴퓨터과학의 핵심 미해결 문제다. automata/[[complexity-classes]]의 맛보기를 여기서 정식으로 파고든다.

## 왜 필요한가

- "효율적으로 풀림"의 수학적 정의가 필요 (P)
- 탐색 문제 대부분은 "검증은 쉬운데 찾기는 어려움" 구조 → NP로 포착
- 최적화·조합 문제의 본질적 난이도를 한 틀로 분류
- P=NP 질문은 암호·최적화·수학 증명 자동화 전부에 연결

## P의 정의

결정적 튜링 머신([[turing-machines]] 관점은 automata/)이 입력 길이 n에 대해 어떤 상수 k로 `O(nᵏ)` 단계 안에 판정하는 결정 문제들의 집합.

```
P = ⋃ₖ TIME(nᵏ)
```

- 예: 정렬, 최단경로(Dijkstra), 최대유량(algorithms/[[network-flow]]), 소수판정(AKS)
- "다항이면 효율적"은 관례 - n¹⁰⁰도 이론상 P지만 실전 알고리즘은 대개 낮은 차수
- 모델 무관성: 합리적 결정적 모델끼리는 다항 시간 차이만 (확장 처치-튜링 명제)

## NP의 두 정의

**정의 1 (비결정적 머신)**: 비결정적 TM(automata/[[dfa-nfa]]의 비결정성을 계산 모델로 확장)이 다항 시간에 판정. 매 단계 여러 갈래를 "동시에" 추측, 하나라도 수용하면 수용.

**정의 2 (검증자)**: 다항 시간 검증자 V와 다항 길이 증명서(certificate) w가 존재해서

```
x ∈ L  ⇔  ∃w (|w| ≤ poly(|x|))  V(x, w)가 수용
```

두 정의는 동치. 비결정적 추측 = 증명서를 하나 찍는 것, 그 갈래 검사 = 검증자 실행.

| 문제 | 증명서 w | 검증 |
|---|---|---|
| SAT | 만족시키는 변수 할당 | 식에 대입해 참인지 |
| 해밀턴 경로 | 정점 순열 | 인접·전부방문 확인 |
| 부분집합 합 | 합이 목표인 부분집합 | 더해서 비교 |
| 합성수 판정 | 약수 하나 | 나눠떨어지는지 |

핵심: 증명서는 짧고(다항 길이), 검증은 빠르다(다항 시간). **찾기**는 지수 후보 중 하나를 고르는 문제라 어려울 수 있음.

## P ⊆ NP

풀 수 있으면 검증도 됨 - 증명서를 무시하고 그냥 풀면 되니까. 빈 증명서 w=ε로 검증자가 직접 판정. 따라서 P ⊆ NP는 자명. 진짜 질문은 **반대 방향**.

```mermaid
graph LR
    P[P: 다항에 푼다] -->|자명 포함| NP[NP: 다항에 검증한다]
    NP -.->|P = NP?| P
```

## P vs NP 질문의 의미

"검증이 다항이면 해결도 다항인가?" = **찾기와 확인이 같은 난이도인가?**

- P = NP라면: 검증 가능한 모든 문제를 효율적으로 풀 수 있음 → 최적화·정리증명·설계 자동화 폭발, 대칭키 외 대부분 공개키 암호 붕괴(security/[[crypto-basics]])
- P ≠ NP라면(다수 예상): 본질적으로 어려운 문제가 존재, 근사·휴리스틱이 필요(algorithms/[[approximation-and-heuristics]])
- 대다수 전문가는 P ≠ NP를 믿지만 **증명이 없음** - Clay 밀레니엄 문제, 상금 100만 달러
- 상세한 실전 함의는 algorithms/[[p-vs-np]]

## co-NP와의 대칭

- **co-NP**: "아니오"를 다항 검증하는 문제 (NP의 여집합 클래스). 예: 식이 **불만족**임을 검증(반례 없음을 어떻게 짧게 보이나?)
- NP = co-NP인지 미해결. P ⊆ NP ∩ co-NP는 확실
- 소수판정은 원래 NP ∩ co-NP에 있음이 먼저 알려졌고, 나중에 P로 판명

## 왜 NP가 특별히 중요한가

- 현실의 조합 최적화 결정판이 거의 다 NP에 들어감 (스케줄링, 배치, 라우팅)
- NP 안에서 "가장 어려운" 문제 = NP-완전([[np-completeness]]), 하나 풀면 NP 전부 풀림
- "지수 탐색 공간 + 다항 검증"이라는 구조가 실무 난제의 공통 뼈대

## 셀프 체크

> [!question]- NP의 비결정적 머신 정의와 검증자 정의는 왜 동치인가?
> 비결정적 추측 = 증명서 w를 하나 찍는 것, 그 갈래 검사 = 검증자 V(x,w) 실행. 비결정 TM의 수용 경로가 존재하면 그 경로를 증명서로 삼아 검증자가 수용하고, 역으로 증명서가 있으면 비결정 머신이 그것을 추측해 수용한다. 증명서 길이가 다항으로 제한되는 것이 핵심.

> [!question]- P ⊆ NP는 왜 자명하고, 진짜 어려운 질문은 어느 방향인가?
> 풀 수 있으면 증명서를 무시하고 그냥 풀면 되므로(빈 증명서 w=ε로 검증자가 직접 판정) P ⊆ NP는 자명하다. 진짜 질문은 반대 방향 NP ⊆ P, 즉 "검증이 다항이면 해결도 다항인가"이다.

> [!question]- co-NP는 어떻게 정의되며 NP와의 관계 중 무엇이 미해결인가?
> co-NP는 "아니오"를 다항 검증하는 문제들(NP 언어의 여집합 클래스)이다. 예: 불리언 식이 불만족임을 짧게 증명하기. NP = co-NP인지는 미해결이고, P ⊆ NP ∩ co-NP만 확실하다.

> [!question]- 소수판정이 P와 NP∩co-NP 이야기에서 특별한 이유는?
> 소수판정은 먼저 NP ∩ co-NP에 있음이 알려졌고(NP-중간처럼 보였음), 이후 AKS 알고리즘으로 결국 P에 속함이 증명되었다. NP∩co-NP에 있던 문제가 P로 내려온 대표 사례.

## 연습문제

> [!example]- HAM-PATH(해밀턴 경로 존재)가 NP에 속함을 증명서·검증자로 엄밀히 보여라
> **풀이**
> 증명서 w = 정점들의 순열 (v₁,v₂,…,vₙ).
> 검증자 V(G,w): (1) w가 G의 모든 정점을 정확히 한 번씩 포함하는지(순열인지) 확인, (2) 연속한 각 쌍 (vᵢ,vᵢ₊₁)이 G의 간선인지 확인.
> 두 검사 모두 O(n²) 안에 끝나므로 다항 시간이고, |w| = O(n log n)로 다항 길이. G에 해밀턴 경로가 있으면 그 경로를 w로 주었을 때 V가 수용하고, 없으면 어떤 w로도 수용 안 됨. 따라서 `G ∈ HAM-PATH ⇔ ∃w. V(G,w) 수용`, 즉 HAM-PATH ∈ NP. ∎

> [!example]- 문제의 클래스를 판정하라: TAUTOLOGY("주어진 불리언 식이 모든 할당에서 참인가")는 NP와 co-NP 중 어디에 자연히 속하는가? 근거를 대라
> **풀이**
> TAUTOLOGY의 부정은 "어떤 할당에서 거짓" = 반례 할당이 존재, 즉 ¬φ가 만족 가능(SAT). SAT는 NP에 있다.
> TAUTOLOGY 자체는 "아니오"(항진식 아님)를 반례 할당 하나로 다항 검증할 수 있으므로 co-NP에 자연히 속한다. 실제로 TAUTOLOGY는 co-NP-완전이다. "∀ 할당에 대해 성립"이라는 전칭 구조는 co-NP의 전형이고, "∃ 증명서"로 짧게 확인되는 SAT류의 NP와 대칭을 이룬다. ∎

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) P의 정의와 NP의 두 정의(비결정/검증자)가 동치인 이유, (2) P⊆NP가 자명하고 P vs NP가 "찾기 vs 확인"이라는 의미, (3) co-NP의 정의와 NP=co-NP 미해결 여부.
