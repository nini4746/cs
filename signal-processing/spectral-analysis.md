# 스펙트럼 분석 (Spectral Analysis)

## 한 줄 요약

실제 신호에서 "어떤 주파수가 얼마나, 언제" 들어있는지 추정하는 실전 기술. 유한 길이로 잘라 DFT를 쓰면 스펙트럼 누설(leakage)이 생기므로 윈도우(window) 함수로 완화하고, 시간에 따라 변하는 신호는 STFT로 잘라 스펙트로그램(spectrogram)을 만든다. 파워 스펙트럼으로 잡음 속 신호 세기를 정량화한다.

## 왜 필요한가

- 이론 DFT([[fourier-transform]])와 달리 실제 신호는 유한·잡음 포함·시변
- 누설·해상도·분산을 다뤄야 쓸모 있는 스펙트럼이 나옴
- 오디오(음정·포먼트), 통신(채널·간섭), 진동 진단, 뇌파 분석의 핵심 도구

## 스펙트럼 누설

무한 신호를 유한 창으로 자르는 것 = 사각창(rectangular window)을 곱하는 것.

- 시간 영역 곱셈 = 주파수 영역 합성곱 → 스펙트럼이 창의 스펙트럼(sinc)과 번짐
- 신호 주파수가 DFT 빈(bin) 중심과 안 맞으면 에너지가 이웃 빈으로 새어 나감(leakage)
- 사각창의 sinc 부엽(sidelobe)이 크고 느리게 감쇠 → 약한 성분이 강한 성분에 묻힘

## 윈도우 함수

경계에서 부드럽게 0으로 가는 창을 곱해 부엽을 억제:

| 창 | 주엽 폭 | 부엽 억제 | 특징 |
|---|---|---|---|
| 사각(rectangular) | 가장 좁음 | ~13 dB | 해상도 최고, 누설 최악 |
| 해닝(Hann) | 중간 | ~31 dB | 범용 기본값 |
| 해밍(Hamming) | 중간 | ~43 dB | 첫 부엽 억제 우수 |
| 블랙만(Blackman) | 넓음 | ~58 dB | 누설 최소, 해상도 희생 |
| 카이저(Kaiser) | 조절 가능 | β로 조절 | 유연한 트레이드오프 |

- **근본 트레이드오프**: 주엽 폭(해상도) ↔ 부엽 높이(누설). 동시에 좋게 못 함 (불확정성 원리)
- 가까운 두 톤 구별 → 좁은 주엽(사각·해닝), 약한 톤 검출 → 낮은 부엽(블랙만)

## 주파수 해상도

- 해상도 `Δf = fs/N` (관측 길이가 길수록 미세 구별)
- **영 채움(zero-padding)**은 빈 간격을 촘촘히 보간할 뿐 진짜 해상도는 안 올림
- 진짜 해상도는 오직 더 긴 관측(N↑)으로만
- 여기서 시간-주파수 트레이드오프: 길게 볼수록 주파수는 정밀, 시간 국소성은 상실

## STFT와 스펙트로그램

시변 신호(음성, 음악)는 전체 DFT가 무의미 → 짧은 창으로 옮겨가며 DFT:

```
STFT(m, k) = Σ_n x[n]·w[n − mR]·e^(−j·2πkn/N)
```
- w: 윈도우, R: 홉 크기(hop), m: 프레임 인덱스
- 프레임마다 [[fft]] → 시간축 × 주파수축 2D

**스펙트로그램** = |STFT|²을 색으로 표시 (가로 시간, 세로 주파수, 밝기 세기)

```mermaid
flowchart LR
    X["x[n]"] --> W["프레임 분할 + 윈도우"]
    W --> F["프레임별 FFT"]
    F --> S["|STFT|² → 스펙트로그램"]
```

| 창 길이 | 시간 해상도 | 주파수 해상도 |
|---|---|---|
| 짧음 | 좋음(빠른 변화 포착) | 나쁨 |
| 긺 | 나쁨(뭉개짐) | 좋음(정밀 음정) |

- 겹침(overlap, 예 50~75%)으로 시간 연속성 확보

## 파워 스펙트럼 추정

랜덤·잡음 신호는 결정적 스펙트럼이 없음 → 파워 스펙트럼 밀도(PSD)를 **추정**.

| 방법 | 아이디어 | 특징 |
|---|---|---|
| 피리오도그램 | \|DFT\|²/N | 간단하나 분산 큼(들쭉날쭉) |
| Bartlett | 구간 나눠 평균 | 분산↓, 해상도↓ |
| **Welch** | 겹친 구간 + 윈도우 + 평균 | 실전 표준, 분산·해상도 균형 |
| 파라메트릭(AR) | 모델 적합 | 짧은 데이터에 유리 |

- 피리오도그램은 N을 키워도 분산이 안 줄어듦(불일치 추정량) → 평균화가 필수
- 통계적 관점 → math/[[random-variables]], math/[[statistics-basics]]

## 실전 응용

- **오디오**: 음정 검출, 포먼트 분석, 노이즈 게이트, 음악 시각화
- **통신**: 점유 대역·간섭 탐지, 반송파 주파수 오프셋 추정
- **진동/기계**: 베어링 결함 주파수로 고장 진단
- **생체신호**: EEG 대역별(알파·베타) 파워

## 셀프 체크

> [!question]- 스펙트럼 누설(leakage)은 왜 발생하는가?
> 무한 신호를 유한 창으로 자르는 것은 사각창을 곱하는 것과 같고, 시간 영역 곱셈은 주파수 영역 합성곱이므로 스펙트럼이 창의 스펙트럼(sinc)과 번진다. 신호 주파수가 DFT 빈 중심과 맞지 않으면 에너지가 이웃 빈으로 새어 나간다.

> [!question]- 윈도우 함수 선택의 근본 트레이드오프는 무엇인가?
> 주엽 폭(주파수 해상도) ↔ 부엽 높이(누설 억제). 사각창은 주엽이 좁아 해상도가 좋지만 부엽이 커서 누설이 심하고, 블랙만은 반대다. 불확정성 원리 때문에 둘을 동시에 좋게 할 수 없다.

> [!question]- 영 채움은 주파수 해상도를 실제로 높이는가?
> 아니다. 영 채움은 빈 간격을 촘촘히 보간(표시)할 뿐 실제 분해능은 올리지 못한다. 진짜 해상도 Δf = fs/N은 오직 관측 길이 N을 늘려야 좋아진다.

> [!question]- 피리오도그램에 평균화가 필수인 이유는?
> 피리오도그램은 불일치(inconsistent) 추정량이라 N을 키워도 분산이 줄지 않아 스펙트럼이 들쭉날쭉하다. Bartlett/Welch처럼 구간을 나눠 여러 피리오도그램을 평균해야 분산이 줄어든다.

## 연습문제

> [!example]- 문제: fs = 8000 Hz, N = 1024점 DFT일 때 주파수 해상도 Δf와, 1000 Hz 성분이 놓이는 빈 번호 k를 구하라.
> **풀이**
> 해상도 `Δf = fs/N = 8000/1024 ≈ 7.81 Hz`.
> 빈 주파수 = k·fs/N 이므로 `k = f·N/fs = 1000·1024/8000 = 128`.
> k=128이 정수라 1000 Hz가 빈 중심에 정확히 놓임 → 이 주파수에 대해서는 누설이 없다.

> [!example]- 문제: 100 Hz와 105 Hz 두 톤을 구별하려면 최소 관측 시간(길이)이 얼마여야 하는가? (해상도 기준 Δf ≤ 5 Hz)
> **풀이**
> 해상도 `Δf = fs/N = 1/T_obs` (T_obs = N/fs = 총 관측 시간).
> 두 톤 간격 5 Hz를 구별하려면 Δf ≤ 5 Hz 필요 → `T_obs = 1/Δf ≥ 1/5 = 0.2초`.
> 즉 최소 0.2초 관측해야 하며, 실제로는 윈도우 주엽 폭 때문에 여유를 더 둬야 한다. 영 채움으로는 이 한계를 못 넘는다.

> [!example]- 문제: STFT에서 창 길이를 늘리면 시간·주파수 해상도가 어떻게 변하는지, 홉 크기 R과 겹침의 역할과 함께 설명하라.
> **풀이**
> STFT: `STFT(m,k) = Σ_n x[n]·w[n − mR]·e^(−j·2πkn/N)`.
> 창을 길게 하면 주파수 해상도↑(정밀 음정)이지만 시간 해상도↓(빠른 변화가 뭉개짐). 짧게 하면 반대.
> 이는 불확정성 원리에 따른 시간-주파수 트레이드오프다.
> 홉 크기 R은 프레임 간 이동 간격이고, R < 창길이면 겹침(예 50~75%)이 생겨 프레임 간 시간 연속성을 확보하고 경계 아티팩트를 줄인다.

## 파인만

> [!note]- 백지에 이 노트 핵심을 남에게 설명하듯 써보라. 막히면 그 부분만 다시.
> **점검 포인트**: (1) 누설이 유한 창(사각창) 곱셈 → 주파수 영역 합성곱에서 나온다는 유도, (2) 윈도우의 주엽-부엽 트레이드오프와 상황별 선택, (3) 해상도 Δf=fs/N의 의미와 영 채움·평균화(Welch)가 각각 무엇을/무엇을 못 하는지.
