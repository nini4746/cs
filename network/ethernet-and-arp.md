# 이더넷과 ARP (Ethernet and ARP)

## 한 줄 요약

링크 계층은 같은 로컬 네트워크 안의 인접 노드 간 전달을 담당한다. MAC 주소로 식별하고, ARP가 IP 주소를 MAC 주소로 변환한다. 스위치는 MAC 기반, 라우터는 IP 기반으로 동작한다.

## 왜 필요한가

- IP 주소가 있는데 왜 MAC 주소가 또 필요한가
- 로컬 네트워크에서 실제 전달이 어떻게 되나
- 스위치와 라우터의 차이

## 링크 계층의 역할

네트워크 계층(IP, [[ip-addressing]])이 "전체 경로"라면, 링크 계층은 **한 홉** - 같은 로컬 네트워크의 인접 노드 간 전달:

```
IP: 서울 → ... → 부산 (전체 경로, [[routing]])
링크: 각 홉마다 "이 라우터 → 다음 라우터" (한 구간)
```

각 홉이 링크 계층으로 실제 프레임을 전달. 이더넷이 가장 흔한 링크 기술.

## MAC 주소

**MAC(Media Access Control) 주소**: 네트워크 인터페이스의 하드웨어 식별자:

```
92:4e:2a:22:08:55   (48비트, 16진수 6쌍)   ← 실측한 내 인터페이스
```

- **하드웨어에 고정**(제조 시), 앞 3바이트 = 제조사
- IP는 **논리적**(바뀜, 위치 기반), MAC은 **물리적**(고정, 기기 기반)
- 로컬 네트워크 안에서 프레임은 MAC으로 전달

## IP vs MAC: 왜 둘 다

| | IP 주소 | MAC 주소 |
|---|---|---|
| 계층 | 네트워크 | 링크 |
| 범위 | 전 세계 (라우팅) | 로컬 네트워크 |
| 성격 | 논리적 (바뀜) | 물리적 (고정) |
| 용도 | 전체 경로 | 한 홉 전달 |

비유: IP = 최종 목적지 주소(계속 유효), MAC = 다음 갈아탈 곳(홉마다 바뀜). 라우터를 지날 때마다 목적 IP는 그대로, 목적 MAC은 다음 홉으로 바뀜.

## ARP: IP → MAC 변환

같은 로컬 네트워크에서 IP는 아는데 MAC을 모를 때 → **ARP(Address Resolution Protocol)**:

```
1. "10.18.0.1의 MAC 누구?" 를 로컬 네트워크에 브로드캐스트
2. 해당 IP를 가진 기기가 "나야, 내 MAC은 ..." 응답
3. IP→MAC 매핑을 ARP 캐시에 저장
```

실측 (`arp -a`):
```
10.18.0.1     → 0:0:5e:0:1:1
10.18.0.254   → b0:26:28:f5:3d:ac
10.18.108.210 → 9a:9b:6:1b:99:b7
```

IP→MAC 매핑 테이블 = ARP 캐시. os/[[page-cache]]처럼 캐싱으로 매번 브로드캐스트 회피. TTL로 만료.

### ARP 스푸핑

ARP도 검증 없음(신뢰 기반) → **ARP 스푸핑**: 공격자가 "게이트웨이 IP = 내 MAC"이라 거짓 응답 → 트래픽 가로챔(중간자 공격) → security/[[authn-authz-failures]].

## 스위치 vs 라우터

로컬 네트워크의 두 장비:

- **스위치(switch)**: **MAC 기반**, 링크 계층. MAC 주소 테이블로 프레임을 해당 포트로만 전달 (학습으로 테이블 구성). 같은 네트워크 내
- **라우터(router)**: **IP 기반**, 네트워크 계층. 다른 네트워크 간 전달, 경로 결정 → [[routing]]

```
같은 서브넷 안 → 스위치 (MAC)
다른 서브넷으로 → 라우터 (IP)
```

전달 과정: 목적이 같은 서브넷이면 ARP로 MAC 찾아 직접, 다른 서브넷이면 게이트웨이(라우터) MAC으로 보냄 → 라우터가 다음 홉으로.

## 이더넷 프레임

```
[목적 MAC][출발 MAC][타입][페이로드(IP 패킷)][CRC]
```

- **CRC**: 오류 검출 (손상 프레임 폐기)
- 페이로드에 IP 패킷 → 캡슐화 ([[internet-overview]])
- 타입 필드로 상위 프로토콜 (IPv4/IPv6/ARP)

## 충돌과 CSMA/CD (역사)

옛 이더넷(공유 매체)은 충돌 처리 필요 (CSMA/CD - 들어보고 보내고 충돌 감지). 현대 스위치 이더넷은 **전이중**(각 포트 전용) → 충돌 없음. 무선은 여전히 충돌 회피 필요 → [[wifi]].
