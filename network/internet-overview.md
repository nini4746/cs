# 인터넷 개요 (Internet Overview)

## 한 줄 요약

인터넷은 패킷 교환 방식으로 데이터를 잘게 쪼개 보내고, 계층 모델(각 계층이 아래 계층 위에 쌓임)로 복잡도를 관리한다. 지연은 처리·큐잉·전송·전파 네 종류로 나뉜다.

## 왜 필요한가

- 네트워크 전체 그림 (이후 모든 것의 틀)
- 패킷 교환이 왜 회선 교환을 이겼나
- 지연·처리량의 정확한 구성

## 패킷 교환 vs 회선 교환

### 회선 교환 (circuit switching)

통신 전 **전용 경로를 예약** (옛 전화망):
- 통화 내내 대역폭 독점 → 지연 일정, 품질 보장
- **낭비**: 말 안 하는 동안도 회선 점유. 예약에 시간

### 패킷 교환 (packet switching) - 인터넷

데이터를 **패킷으로 쪼개 각자 보냄**:
- 각 패킷이 헤더(목적지 등) + 데이터
- 라우터가 패킷마다 다음 홉 결정 (경로 예약 없음)
- **통계적 다중화**: 여러 통신이 링크를 공유 → 안 쓸 때 남이 씀 → 효율↑
- 대가: 혼잡 시 큐잉 지연·손실 가능 (예약 안 하니까)

패킷 교환이 이긴 이유: **버스트한 데이터 트래픽**에 효율적 (전화처럼 일정하지 않음). 인터넷 규모의 공유가 가능.

## 계층 모델

복잡도를 **계층**으로 분할. 각 계층이 아래 계층 서비스를 쓰고 위 계층에 서비스 제공:

### TCP/IP 5계층 (실용)

| 계층 | 역할 | 예 | 노트 |
|---|---|---|---|
| **응용(application)** | 앱 프로토콜 | HTTP, DNS | [[http]], [[dns]] |
| **전송(transport)** | 종단 간 전달 | TCP, UDP | [[tcp-basics]], [[udp]] |
| **네트워크(network)** | 라우팅 | IP | [[ip-addressing]], [[routing]] |
| **링크(link)** | 인접 노드 전달 | 이더넷, WiFi | [[ethernet-and-arp]], [[wifi]] |
| **물리(physical)** | 비트 전송 | 전기/광 신호 | - |

(OSI 7계층은 여기에 세션·표현을 추가한 이론 모델. 실무는 TCP/IP 5계층)

### 캡슐화 (encapsulation)

각 계층이 위 계층 데이터에 **자기 헤더를 붙임**:

```
응용: [HTTP 데이터]
전송: [TCP헤더][HTTP 데이터]
네트워크: [IP헤더][TCP헤더][HTTP 데이터]
링크: [이더넷헤더][IP헤더][TCP헤더][HTTP 데이터][이더넷꼬리]
```

- 보낼 때 아래로 내려가며 헤더 추가, 받을 때 위로 올라가며 제거
- 각 계층은 자기 헤더만 봄 (아래 데이터는 불투명) → 계층 독립

## 왜 계층화인가

- **모듈성**: 한 계층 바꿔도 나머지 무관 (WiFi→이더넷 바꿔도 HTTP 그대로)
- **추상화**: 응용은 라우팅·물리를 몰라도 됨
- programming-languages의 추상화, os의 계층과 같은 원리
- 대가: 헤더 오버헤드, 계층 경계의 비효율 (계층 위반 최적화도 존재)

## 네트워크 지연 4종

패킷 하나가 링크를 건너는 총 지연:

```
총 지연 = 처리 + 큐잉 + 전송 + 전파
```

| 지연 | 원인 | 결정 요인 |
|---|---|---|
| **처리(processing)** | 헤더 검사, 경로 결정 | 라우터 속도 |
| **큐잉(queuing)** | 버퍼에서 대기 | 혼잡도 (가변!) |
| **전송(transmission)** | 패킷을 링크에 밀어넣기 | 패킷 크기 / 대역폭 |
| **전파(propagation)** | 신호가 물리적으로 이동 | 거리 / 광속 |

- **전송 vs 전파 혼동 주의**: 전송 = 패킷을 링크에 다 올리는 시간(크기÷속도), 전파 = 첫 비트가 반대편 도착(거리÷광속). 서울-부산은 전파 짧고, 대용량은 전송 김
- **큐잉이 유일하게 가변**: 혼잡하면 폭증 → 지연 스파이크의 주범

## 대역폭 vs 지연시간

- **대역폭(throughput)**: 초당 얼마나 (파이프 굵기). 병목 링크가 결정
- **지연시간(latency)**: 한 번 왕복 시간 (RTT). 거리·홉 수
- 둘은 독립: 위성은 대역폭 커도 지연 큼 (거리). 이 구분이 성능 튜닝의 핵심
