# WiFi (무선 링크)

## 한 줄 요약

무선은 유선과 근본적으로 다르다 - 매체를 공유하고, 충돌을 감지할 수 없어 회피(CSMA/CA)하며, 신호가 거리·간섭에 약하다. 이 제약들이 무선 성능의 예측 불가능함을 만든다.

## 왜 필요한가

- 무선이 유선과 왜 다른가 (같은 이더넷 아님)
- WiFi가 왜 느리거나 불안정하나
- 충돌 회피가 왜 필요한가

## 무선의 근본적 차이

유선 이더넷([[ethernet-and-arp]])과 무선의 핵심 차이:

1. **공유 매체**: 공기는 모두가 공유 → 동시에 보내면 충돌
2. **충돌 감지 불가**: 무선 기기는 보내면서 동시에 들을 수 없음 (자기 신호가 너무 큼) → CSMA/CD 불가
3. **신호 감쇠**: 거리·장애물로 신호 약해짐, 간섭(전자레인지, 다른 WiFi)
4. **숨은 노드 문제**: A와 C가 서로 안 들리지만 둘 다 B에 보내면 B에서 충돌

이 제약들 때문에 무선은 유선과 다른 프로토콜 필요.

## CSMA/CA (충돌 회피)

유선은 CSMA/CD(충돌 **감지**)지만, 무선은 감지 불가 → **CSMA/CA(충돌 회피)**:

```
1. 채널이 비었나 확인 (carrier sense)
2. 비었으면 랜덤 시간 대기 후 전송 (동시 전송 확률 낮춤)
3. 수신자가 ACK 보냄 → ACK 없으면 충돌로 간주, 재전송
```

- 충돌을 **미리 피함** (감지가 아니라)
- **ACK 필수**: 유선은 충돌 감지로 알지만, 무선은 ACK로 성공 확인
- **RTS/CTS** (선택): 보내기 전 "보낼게(RTS)"/"보내(CTS)" 교환 → 숨은 노드 문제 완화

## 왜 무선이 느리고 불안정한가

- **충돌 회피 오버헤드**: 대기·ACK가 처리량 깎음
- **재전송**: 신호 약하면 손실 → 재전송 (TCP는 이걸 혼잡으로 오해할 수도 → [[congestion-control]])
- **공유**: 같은 AP의 여러 기기가 대역폭 나눔 (반이중 - 한 번에 하나)
- **거리·간섭**: 멀어지거나 벽 있으면 속도 급락 (적응형 변조 - 나쁘면 느린 인코딩)
- **비결정적**: 유선보다 지연·처리량 변동 큼

## WiFi 구조

- **AP(Access Point)**: 무선 기기를 유선 네트워크에 연결. 기기들이 AP를 통해 통신
- **SSID**: 네트워크 이름
- **채널**: 주파수 대역 분할 (2.4GHz는 겹침 많음, 5GHz/6GHz는 채널 많음)
- **연결**: 스캔 → 인증 → 결합(association) → (암호화 협상)

## 주파수 대역

| 대역 | 특성 |
|---|---|
| **2.4GHz** | 멀리 가고 벽 통과 잘함, 하지만 느리고 혼잡(전자레인지, 블루투스) |
| **5GHz** | 빠르고 채널 많음, 하지만 도달 거리 짧고 벽에 약함 |
| **6GHz** (WiFi 6E) | 더 넓고 깨끗, 최신 |

거리-속도 트레이드오프: 낮은 주파수 = 멀리/느리게, 높은 주파수 = 가까이/빠르게.

## WiFi 세대

- WiFi 4 (802.11n), 5 (ac), **6 (ax)**, 6E, **7 (be)**
- 개선: MIMO(다중 안테나), OFDMA(주파수 효율), 더 넓은 채널
- 각 세대가 처리량·다중 기기 처리 개선

## 보안

무선은 **공기로 전파** → 누구나 들을 수 있음 → 암호화 필수:

- **WPA2/WPA3**: 무선 암호화 (WEP는 깨짐, 사용 금지)
- 개방 WiFi는 도청 위험 → HTTPS([[http]], security/[[tls-deep-dive]])가 앱 레벨 보호
- WPA3는 개별 암호화(개방망에서도), 오프라인 사전 공격 방어

## 셀프 체크

> [!question]- 유선 이더넷은 CSMA/CD를 쓰는데 무선은 왜 CSMA/CA를 써야 하는가?
> 무선 기기는 보내면서 동시에 들을 수 없다. 자기 송신 신호가 너무 커서 다른 신호와의 충돌을 감지(CD)할 수 없기 때문이다. 그래서 충돌을 감지하는 대신 미리 회피(CA)한다. 채널이 비었는지 확인하고 랜덤 시간 대기 후 전송하며, 성공 여부는 수신자의 ACK로 확인한다.

> [!question]- 숨은 노드 문제란 무엇이고 RTS/CTS가 이를 어떻게 완화하는가?
> A와 C가 서로의 신호를 못 듣지만 둘 다 중간의 B에 보내면 B에서 충돌이 난다. carrier sense만으로는 서로를 감지 못 해 막을 수 없다. RTS/CTS는 전송 전에 "보낼게(RTS)"와 "보내(CTS)"를 교환하는데, B가 보내는 CTS는 A와 C 모두 들을 수 있어 다른 노드가 채널이 예약됐음을 알고 대기하게 된다.

> [!question]- WiFi가 유선보다 느리고 불안정한 이유를 몇 가지 대라.
> 충돌 회피의 대기와 ACK 오버헤드가 처리량을 깎고, 신호가 약하면 손실돼 재전송이 늘며(TCP가 이를 혼잡으로 오해할 수도 있음), 같은 AP의 여러 기기가 반이중으로 대역폭을 나눠 쓴다. 거리와 장애물, 간섭으로 적응형 변조가 느린 인코딩을 택하면 속도가 급락하고, 전반적으로 유선보다 지연/처리량 변동이 크다.

> [!question]- 2.4GHz와 5GHz의 트레이드오프는 무엇인가?
> 2.4GHz는 낮은 주파수라 멀리 가고 벽을 잘 통과하지만 느리고 전자레인지/블루투스 등으로 혼잡하다. 5GHz는 빠르고 채널이 많지만 도달 거리가 짧고 벽에 약하다. 낮은 주파수는 멀리/느리게, 높은 주파수는 가까이/빠르게라는 거리-속도 트레이드오프가 있다.

## 연습문제

> [!example]- 문제: 하나의 AP에 5개 기기가 붙어 있고 AP의 실효 처리량이 100Mbps다. 반이중 공유 매체 특성을 고려해 기기당 기대 처리량을 논하고, 유선 스위치와 어떻게 다른지 설명하라.
> **풀이**
> WiFi는 반이중 공유 매체라 한 순간 하나의 기기만 전송할 수 있고, 채널 접근 시간을 나눠 쓴다. 이상적으로 균등 분배하면 기기당 100/5 = 20Mbps지만, 실제로는 CSMA/CA의 대기/backoff와 ACK 오버헤드, 충돌 재시도로 총 실효 처리량 자체가 명목 대비 더 깎여 기기당 20Mbps보다 낮다. 반면 유선 스위치는 각 포트가 독립된 전이중 링크라 기기끼리 대역폭을 나누지 않고 각자 전용 속도를 낼 수 있다.

> [!example]- 문제: 클라이언트가 특정 WiFi에 처음 붙는 과정을 순서대로 나열하고, 각 단계에서 무엇이 오가는지 설명하라.
> **풀이**
> 1. 스캔: 클라이언트가 채널을 돌며 주변 AP의 비컨을 듣거나 프로브 요청을 보내 SSID 목록을 수집한다.
> 2. 인증(authentication): 대상 AP와 개방/공유 방식으로 초기 인증 프레임을 교환한다.
> 3. 결합(association): AP에 결합 요청을 보내 이 AP를 통해 통신하도록 등록되고, association ID를 받는다.
> 4. 암호화 협상: WPA2/WPA3면 4-way 핸드셰이크(WPA3는 SAE)로 세션 키를 유도한다.
> 이후 클라이언트는 AP를 거쳐 유선 네트워크로 나간다. 개방망이면 4단계 없이 평문이라 앱 레벨 HTTPS 보호가 필수다.
