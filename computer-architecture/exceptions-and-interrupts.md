# 예외 제어 흐름 (Exceptions and Interrupts)

## 한 줄 요약

프로그램의 정상 흐름을 하드웨어/OS가 가로채는 메커니즘. 인터럽트(외부 신호), 트랩(의도적 - 시스템 콜), 폴트(오류)로 나뉘며, 시스템 콜이 실제로 어떻게 커널로 넘어가는지가 여기 있다.

## 왜 필요한가

- `read()` 한 줄이 실제로 커널에서 어떻게 실행되나
- 사용자 프로그램이 어떻게 하드웨어(디스크, 네트워크)에 접근하나
- OS가 어떻게 제어권을 되찾아 멀티태스킹하나 → [[cpu-scheduling]]
- 컴퓨터 구조와 OS의 경계면 (이 노트가 그 다리)

## 예외 제어 흐름 (ECF)

보통 명령어는 순차 또는 분기로 흐른다. **예외(exception)**는 이 흐름을 강제로 바꿔 미리 정해진 핸들러로 점프시킴. 각 예외 종류마다 **예외 테이블**(exception vector)에 핸들러 주소가 등록돼 있고, CPU가 그리로 점프.

핵심: 예외가 나면 **유저 모드 → 커널 모드**로 전환되고 핸들러(커널 코드)가 실행됨.

## 유저 모드 vs 커널 모드

CPU는 특권 레벨을 가짐:

- **유저 모드**: 일반 명령만. 하드웨어 직접 접근, 특권 명령 금지
- **커널 모드**: 전부 가능. OS 커널이 여기서 돎

일반 프로그램은 유저 모드. 하드웨어가 필요하면(파일 읽기 등) 직접 못 하고 **커널에 요청**해야 함 → 그 요청 수단이 예외(트랩). 이 격리가 보안·안정성의 기반 ([[limited-direct-execution]]).

## 예외 4종

| 종류 | 원인 | 비동기? | 예 | 복귀 |
|---|---|---|---|---|
| **인터럽트 (interrupt)** | 외부 하드웨어 신호 | 비동기 | 키보드 입력, 타이머, 네트워크 패킷 | 다음 명령 |
| **트랩 (trap)** | 의도적 명령 | 동기 | **시스템 콜**, 브레이크포인트 | 다음 명령 |
| **폴트 (fault)** | 복구 가능한 오류 | 동기 | 페이지 폴트, 0 나누기 | 같은 명령 재시도 or 종료 |
| **어보트 (abort)** | 복구 불가 오류 | 동기 | 하드웨어 고장(ECC) | 종료 |

- **비동기(인터럽트)**: 현재 명령과 무관하게 아무 때나. 외부 세계가 CPU를 부름
- **동기(나머지)**: 특정 명령 실행의 결과

## 시스템 콜: 트랩의 대표

`read()`, `write()`, `fork()` 같은 시스템 콜이 실제로 커널로 넘어가는 과정:

```
1. 유저 코드가 시스템 콜 번호를 레지스터에 넣음 (예: read = x86-64 Linux 0, ARM64 Linux 63)
2. 인자를 레지스터에 세팅
3. 트랩 명령 실행: ARM64 `svc #0`, x86-64 `syscall`
4. → CPU가 커널 모드로 전환, 예외 테이블의 시스템 콜 핸들러로 점프
5. 커널이 번호로 핸들러 찾아 실행 (실제 파일 읽기)
6. 결과를 레지스터에 넣고 `eret`(exception return)
7. → 유저 모드로 복귀, 다음 명령 계속
```

`printf` → `write` 시스템 콜 → `svc` → 커널 → 실제 출력. 우리가 짠 코드와 하드웨어 사이에 항상 이 트랩이 있다.

### 왜 함수 호출이 아니라 트랩인가

시스템 콜을 일반 함수 호출로 하면 유저 모드에서 커널 코드를 직접 부르는 것 → 격리 붕괴. 트랩은 **정해진 진입점(예외 테이블)으로만** 커널에 들어가게 강제 → 커널이 요청을 검증할 수 있음. 이게 보안 경계.

## 컨텍스트 스위치도 예외 기반

**타이머 인터럽트**가 주기적으로 발생 → 커널이 제어권 획득 → 다른 프로세스로 전환 가능. 프로그램이 자발적으로 양보 안 해도 OS가 CPU를 되찾는 메커니즘. 선점형 멀티태스킹의 하드웨어 기반 → [[cpu-scheduling]], [[limited-direct-execution]].

## 페이지 폴트: 폴트의 대표

프로그램이 아직 물리 메모리에 없는 페이지를 접근 → **페이지 폴트**(fault) → 커널이 디스크에서 페이지를 로드 → **같은 명령 재시도**. 가상 메모리가 동작하는 핵심 → [[virtual-memory]], [[swapping]].
