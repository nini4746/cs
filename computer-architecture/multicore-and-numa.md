# 멀티코어와 NUMA (Multicore and NUMA)

## 한 줄 요약

코어를 늘려도 왜 선형으로 안 빨라지나. 캐시 일관성 트래픽, 메모리 대역폭, 그리고 큰 시스템에선 "코어마다 메모리 접근 비용이 다른"(NUMA) 현상이 확장성의 벽이다.

## 왜 필요한가

- 클럭이 안 오르니 코어를 늘렸는데(2005년경~), 그 병렬성을 어떻게 쓰나
- 스레드를 2배로 늘렸는데 왜 2배가 안 되나
- 서버에서 스레드/메모리 배치가 왜 성능을 좌우하나
- OS 동시성([[threads-and-races]])의 하드웨어 배경

## 왜 멀티코어로 갔나

2000년대 중반 단일 코어 클럭이 벽에 부딪힘:

- **전력 벽**: 클럭 ∝ 전력, 발열이 냉각 한계 초과 (Pentium 4의 실패)
- **ILP 벽**: 단일 스레드에서 뽑을 병렬성([[instruction-level-parallelism]])의 한계
- → 클럭 대신 **코어 개수**를 늘림. 대신 병렬 프로그래밍의 짐을 소프트웨어로 넘김

## 메모리 계층 공유 구조

전형적 멀티코어 ([[memory-hierarchy]]):

```
코어0        코어1        코어2        코어3
L1/L2(전용)  L1/L2       L1/L2       L1/L2
   └────────────┴─── L3 (공유) ───┴────────────┘
                     메인 메모리 (공유)
```

- L1/L2는 코어 전용, L3와 RAM은 공유
- 공유 지점이 경합 지점: 여러 코어가 RAM을 동시에 원하면 **메모리 대역폭**을 나눠 씀

## 확장성의 벽 3가지

### 1. 캐시 일관성 트래픽

코어들이 공유 데이터를 쓰면 MESI 무효화가 오감 ([[cache-coherence]]). 공유 카운터·락 경합이 심하면 라인이 코어 사이 핑퐁 → 코어를 늘릴수록 오히려 느려짐. false sharing도 여기.

### 2. 메모리 대역폭 포화

SIMD([[simd]]) 예제가 이론 4배 안 난 이유와 같음. 코어가 많아도 RAM 대역폭은 유한 → 대역폭 집약 작업은 코어 몇 개에서 이미 포화. "코어는 노는데 메모리가 병목".

### 3. 암달의 법칙 (Amdahl's Law)

프로그램의 직렬 부분 비율이 s면, 아무리 코어가 많아도 최대 속도향상은 1/s로 제한.

```
speedup ≤ 1 / (s + (1-s)/p)     p = 코어 수
```

s=5%면 코어 무한대라도 최대 20배. **직렬 부분이 확장성을 지배**. 병렬화보다 직렬 병목 제거가 먼저인 이유.

## NUMA (Non-Uniform Memory Access)

큰 시스템(멀티소켓 서버)에선 메모리가 소켓별로 분산:

```
소켓0 ── 로컬 RAM0          소켓1 ── 로컬 RAM1
  코어들                      코어들
   └──────── 인터커넥트 ────────┘
```

- 코어가 **자기 소켓의 로컬 메모리**에 접근하면 빠름
- **다른 소켓의 메모리**(원격)에 접근하면 인터커넥트를 건너 느림 (수 배)
- 그래서 "메모리 접근 비용이 코어 위치마다 다름" = Non-Uniform

### NUMA 대응

- **first-touch 정책**: 메모리를 처음 만진 코어의 로컬에 할당. 그래서 초기화를 쓸 스레드가 직접 해야 함
- **스레드 고정(affinity)**: 스레드를 특정 코어/소켓에 묶어 로컬 접근 유지
- **데이터 분할**: 각 스레드가 자기 로컬 데이터만 만지게
- DB/HPC에서 NUMA 인식 배치가 성능을 크게 좌우

(참고: Apple Silicon 같은 단일 소켓은 NUMA 아님. 서버·워크스테이션 이슈)

## 메모리 순서 모델 (재확인)

멀티코어에서 "한 코어의 쓰기가 다른 코어에 어떤 순서로 보이나"는 ISA마다 다름 ([[cache-coherence]], [[isa-design]]):

- **x86 (TSO)**: 강한 순서
- **ARM**: 약한 순서 → 명시적 배리어 필요
- 락프리 코드([[lock-free-basics]])가 아키텍처마다 다르게 동작하는 근원
