# 파이프라이닝 (Pipelining)

## 한 줄 요약

명령어 하나를 여러 단계로 쪼개고, 세탁기-건조기처럼 단계를 겹쳐 실행한다. 명령어 하나의 지연시간은 그대로지만 처리량(throughput)이 단계 수만큼 오른다.

## 왜 필요한가

- CPU가 클럭을 어떻게 올릴 수 있었나의 핵심 (한 사이클에 할 일을 줄이면 클럭 ↑)
- 분기 예측([[branch-prediction]])·해저드([[hazards]])가 왜 존재하는지의 전제
- "명령어 하나가 여러 사이클 걸리는데 어떻게 IPC가 1에 가깝나"의 답

## 비유: 빨래

빨래 4더미, 각 더미가 세탁(30분)→건조(30분)→개기(30분):

- **순차**: 한 더미 끝내고 다음 → 4 × 90분 = 6시간
- **파이프라인**: 1더미가 건조 중일 때 2더미 세탁 시작 → 겹침. 4더미 = 90분 + 3×30분 = 3시간

단계를 겹치면 각 더미의 소요는 그대로(90분)인데 전체 처리량이 오른다.

## 고전 5단계 파이프라인 (RISC 교과서 모델)

```
IF  → ID  → EX  → MEM → WB
```

1. **IF** (Instruction Fetch): 명령어 메모리에서 가져오기
2. **ID** (Instruction Decode): 해독 + 레지스터 읽기
3. **EX** (Execute): ALU 연산 / 주소 계산
4. **MEM** (Memory): 메모리 접근 (load/store만)
5. **WB** (Write Back): 결과를 레지스터에 쓰기

각 단계 사이에 **파이프라인 레지스터**가 있어 중간 상태를 다음 사이클로 전달.

## 겹침의 그림

```
사이클:    1    2    3    4    5    6    7    8
명령1:    IF   ID   EX   MEM  WB
명령2:         IF   ID   EX   MEM  WB
명령3:              IF   ID   EX   MEM  WB
명령4:                   IF   ID   EX   MEM  WB
```

사이클 5부터는 매 사이클 명령어 하나씩 완료 → 정상 상태 처리량 = 명령어/사이클 = 1 (이상적).

## 성능: 지연시간 vs 처리량

- **지연시간(latency)**: 명령어 하나가 5단계 다 거치는 시간 = 5 사이클. **안 줄어듦** (오히려 단계 레지스터 오버헤드로 살짝 늘 수도)
- **처리량(throughput)**: 이상적으로 5배. 실제로는 해저드 때문에 덜

왜 클럭이 오르나: 5단계로 쪼개면 한 단계가 하는 일이 1/5 → 사이클 시간을 1/5로 줄일 수 있음 → 클럭 5배. 이게 파이프라이닝의 진짜 이득.

### 더 깊은 파이프라인의 함정

단계를 더 쪼개면(예: 20단계, Pentium 4) 클럭은 더 오르지만:

- 분기 예측 실패 시 버리는 명령어가 많아짐 (파이프라인이 길수록 손실 큼)
- 단계 레지스터 오버헤드 증가
- Pentium 4가 31단계까지 갔다가 전력·발열로 실패 → 현대는 14~19단계 수준에서 균형

## 해저드: 이상이 깨지는 지점

파이프라인이 항상 1 IPC를 못 내는 이유. 세 종류 (상세는 [[hazards]]):

- **구조 해저드**: 하드웨어 자원 부족 (한 사이클에 IF와 MEM이 같은 메모리를 원함)
- **데이터 해저드**: 앞 명령의 결과를 뒷 명령이 아직 못 받음 (`add x1,...; sub x2,x1,...`)
- **제어 해저드**: 분기 결과가 나오기 전에 다음 명령을 가져와야 함 → [[branch-prediction]]

해결책 맛보기: 포워딩(결과를 WB 전에 앞당겨 전달), 스톨(거품 삽입), 분기 예측. 전부 [[hazards]]에서.

## 현대 CPU와의 거리

교과서 5단계는 개념 모델. 실제는:

- 14~19단계 파이프라인
- **슈퍼스칼라**: 한 사이클에 여러 명령어 발행 (IPC > 1) → [[instruction-level-parallelism]]
- **비순차 실행**: 순서 바꿔 실행 후 재정렬
- 하지만 이 모든 것의 기반 개념이 파이프라이닝

## 연결

- 파이프라인을 막는 해저드와 해결 → [[hazards]]
- 제어 해저드를 줄이는 분기 예측 → [[branch-prediction]]
- 파이프라인 위에 병렬성을 더 쌓기 → [[instruction-level-parallelism]]
- 파이프라인이 구현하는 ISA → [[isa-design]]
